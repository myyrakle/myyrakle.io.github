enum은 열거형 값을 지원하는 메타데이터성 타입이다.
C/C++의 enum과 비슷하다.
Rust와도 비슷한가 싶었는데 Rust의 enum에 비하면 기능이 부족하다. 다형적인 타입은 표현할 수 없다.

기본적으로 다음과 같은 형태로 정의된다.
아래는 색상을 나타내는 enum을 정의한 예시다.

다만 여기에서는 enum이 무언가 의미있는 값을 나타내지는 않는다.




enum에 타입 지정

만약 숫자값 등을 부여하고 싶다면, 다음과 같이 정수 타입을 내부에 지정해주고, 값을 할당해주면 된다.

기본적으로 정수 타입만 지정 가능하다.

그러면

@intFromEnum 같은 변환함수를 통해 값을 추출하거나 할 수 있다.

근데 이게 최선인건가? 별로 깔끔한 구조 같진 않다.




C ABI 호환

enum은 기본적으로 c와 호환되지 않는다.
c abi에 호환되게 하려면 다음과 같이 c 전용 정수타입을 넣어줘야 한다.




메서드

enum도 객체처럼 다룰 수 있다. 구조체처럼 내부 정의에 메서드를 갖다 박을 수 있다.




switch

enum은 특성상 switch case를 통해 분기 처리를 하는 경우가 많다.
대충 이런 느낌으로 사용될 것이다.

zig는 신택스 체크가 꽤 빡빡한 편이라서,

처리하지 않은 케이스가 있으면 오류를 던진다.
