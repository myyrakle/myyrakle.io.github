#CockroachDB
[원본 링크]

CockroachDB는 대형 분산시스템에 최적화된 관계형 데이터베이스다.
전세계에 지리적으로 분포된 글로벌 서비스에 적합하다.

기존의 관계형 데이터베이스들은 대부분 초대형-고성능 스케일아웃에 한계가 대단히 많고 불편하기 때문에, 나름 독자적인 위치를 차지하고 있다.

GCP의 초대형 관계형 데이터베이스인 Cloud Spanner의 영향을 많이 받았으며, 실제로 이걸 Cloud Spanner의 개발자들이 만들기도 했다. 지향점도 거의 동일하다.

관계형 데이터베이스답게 ACID를 잘 지원하며, PostgreSQL과 거의 유사한 구조를 갖고있다. 프로토콜도 postgres 걸 그대로 갖다쓴다.

돈이 많다면 Cloud Spanner를 쓰는게 가장 편리하고 좋지만, 돈이 없다면 그 다음 대안으로 쓸만하다. 실제로도 그렇게 채택되고 있고.




Raft 알고리즘

CockroachDB는 Raft라는 알고리즘을 통해 노드들 간의 replication, 합의를 구현한다.
따지자면 multi-master긴 한데, 모든 노드가 마스터가 되지는 않는다.

Raft 구조에서 노드는 총 3가지로 분류된다. 리더(Leader), 팔로워(Follower), 후보자(Candidate)다.


리더는 전통적인 master replica의 역할을 한다고 보면 된다. 읽기와 쓰기를 다 처리할 수 있다.
팔로워는 slave replica고, read only로 사용된다.

대충 이러한 구조를 통해서 리더<>팔로워 간 복제가 수행된다.

  1. 리더가 write를 받으면, 그 변경사항을 "로그"로 저장해서 팔로워들에게 공유해준다.
  2. 팔로워가 리더에게서 변경사항을 공유받고 문제없이 처리했다면, 리더에게 성공했다고 답신을 준다.
  3. 이때 팔로워 중 일부가 실패했더라도, 리더는 팔로워의 과반수가 성공했다고 알려오면, 실패한 팔로워들에게 성공할때까지 로그를 재전송해준다.

그리고 Candidate는 말 그대로 리더가 될 수 있는 후보다.
만약 리더에 문제가 생긴다면,

  1. 임의의 팔로워가 후보로 선출이 된다. 이건 election timeout이라는 무작위 대기시간을 기반으로 결정된다.
  2. 후보는 다른 팔로워들에게 투표를 요구한다.
  3. 후보가 정상 응답을 과반 얻는다면 바로 리더가 된다.

과반을 기준으로 장애를 처리하는 이러한 구조 탓에, Raft 알고리즘에서는 노드의 개수를 3개 이상의 홀수로 유지하는 것을 강력하게 권장한다.




설치하기

Linux에서는 다음과 같이 설치할 수 있다.
파일 받아서, 압축만 푼다.

curl -o cockroach.tgz https://binaries.cockroachdb.com/cockroach-v22.2.7.linux-amd64.tgz
tar zxvf cockroach.tgz

그리고 풀어놓은걸 다음과 같이 적절히 재배치하면 끝이다.

sudo mkdir -p /usr/local/lib/cockroach
sudo cp -i  cockroach-v22.2.7.linux-amd64/lib/libgeos.so /usr/local/lib/cockroach/
sudo cp -i  cockroach-v22.2.7.linux-amd64/lib/libgeos_c.so /usr/local/lib/cockroach/

sudo cp -i cockroach-v22.2.7.linux-amd64/cockroach /usr/local/bin/cockroach
