#CassandraDB
[원본 링크]

카산드라DB는 분산시스템에 최적화된 NoSQL 데이터베이스 중 하나다.

신속한 스케일아웃-확장이 가능하다는 것을 가장 큰 장점으로 내세운다. 다운타임 없이 노드를 수십수백개씩 자유롭게 늘리거나 줄일 수 있다는 것이다.

그리고 write가 read보다 더 빠르다. 분산 환경에서의 빠르고 안전한 write 작업을 위해 read 성능을 희생한 형태인 것 같다.

2008년에 페이스북에 의해서 처음 만들어졌고, 현재는 아파치 재단에서 관리하고 있다.

NoSQL이라고는 하지만, SQL 구문을 약간 제공하고 ACID를 보장해서 RDB처럼 사용할 수는 있다.
다만 JOIN 같은건 없다.

대규모로 쓰기에는 꽤 적절한 것 같다. 넷플릭스나 애플 같은데서도 수천개 단위의 노드를 클러스터링해서 쓰고 있다고 한다.

다만 자바로 만들어진거라서 GC로 인한 문제가 조금은 있는 것 같다.
디스코드는 카산드라로 수십억개 단위의 메세지 데이터를 관리하고 있다가 GC로 인한 문제를 겪고 ScyllaDB로 마이그레이션을 했다고 한다.




구조

카산드라는 multi master 레플리케이션을 통해 분산구조를 제공한다.

각 노드가 전부 읽기/쓰기를 할 수 있으며, 한 노드에 쓰기가 발생했을 경우 다른 노드에도 변경사항을 전파하는 것이다.




gossip 프로토콜

카산드라는 노드 간 데이터 복제에 gossip이라는 프로토콜을 사용한다.
블록체인이나 기타 P2P에서도 자주 사용되는데, 꽤나 독특한 호출 구조를 갖는다.

가십 프로토콜 (Gossip Protocol) 이란? :: Jins' Dev Inside (tistory.com)
4개의 노드가 있더라도 서로를 다 바라보고 다 쏘는게 아니라, 무작위의 노드에게 데이터를 연속적으로 날리면서 주고받을 뿐이다.
영희가 철수에게, 철수가 스미스에게, 소문(Gossip)을 퍼트리듯이 정보가 전달된다고 해서 가십 프로토콜이라 한다. 결국에는 소문이 모든 사람을 거쳐가므로 데이터 동기화는 똑같이 된다.

그리고 송신자와 수신자가 명확하게 결합되어있지 않은 이러한 통신 구조 덕에 노드의 추가나 삭제가 매우 유연한 편이다.




사용해보기

docker를 통해서 테스트환경을 한번 만들어보겠다.

이미지를 받고, 네트워크 세팅하고, 바로 실행하면 된다.

docker pull cassandra:latest
docker network create cassandra
docker run --rm -d --name cassandra --hostname cassandra --network cassandra cassandra

그리고 저 컨테이너에 접속해서 cqlsh를 실행하면, 바로 대화형 쉘이 열린다.

docker exec -it 컨테이너해시 bash 

cqlsh

여기서 쿼리 쳐보면서 테스트를 하면 된다.




키스페이스 (keyspace)

키스페이스는 다른 RDB의 데이터베이스나 스키마와 비슷한 단위다.
하나의 키스페이스에는 여러개의 테이블이 들어갈 수 있다.

만드는 법은 어렵지 않다.
아래는 키스페이스 test_space를 만드는 간단한 쿼리다.

CREATE KEYSPACE IF NOT EXISTS test_space WITH REPLICATION = { 'class' : 'SimpleStrategy', 'replication_factor' : '1' };




테이블

테이블은 여러분이 알던 그 테이블 단위가 맞다.
아래와 같은 형태로 만들 수 있다.

-- Create a table
CREATE TABLE IF NOT EXISTS test_space.item(
    uuid text PRIMARY KEY,
    item_name text,
    item_price double,
    last_update_timestamp timestamp
);

그리고 익숙한 SQL 구문을 통해 CRUD 작업을 처리할 수 있다.
데이터를 적절히 4개만 넣고 조회해보자.

insert into test_space.item(uuid, item_name, item_price, last_update_timestamp)
values
('c714d095-0a63-4c1f-921d-42abd954e2c1', 'buldak', 100.5, toTimeStamp(now()));
insert into test_space.item(uuid, item_name, item_price, last_update_timestamp)
values
('02773666-f483-4749-a3dd-c526e62daca2', 'eojuk', 4000, toTimeStamp(now()));
insert into test_space.item(uuid, item_name, item_price, last_update_timestamp)
values
('1f81911a-9a90-451c-8104-d6c9ff742588', 'gejang', 100.5, toTimeStamp(now()));
insert into test_space.item(uuid, item_name, item_price, last_update_timestamp)
values
('4c2fff64-a727-48a6-aa87-5af6079f2ead', 'john doe', 100.5, toTimeStamp(now()));

select * from test_space.item;

잘 되는 것 같다.
