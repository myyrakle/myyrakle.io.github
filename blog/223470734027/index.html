[Rust] Cargo: check와 lint, clippy

  • 2024.06.06
  • ::
  • 2 min read
#Rust
[원본 링크]

Rust는 컴파일 오류로만 빡빡한게 아니라 컨벤션도 꽤 빡빡한 편이다.
그래서 lint 수준도 2가지로 나뉘어있고, 종류나 지원도 꽤 적극적인 편이다.

관련 명령을 다루는 방법을 간단히 정리해본다.




cargo check

check는 기본적인 종속성 설치나 컴파일 검사까지만 수행하는 명령어다.
체크는 돌려보고 싶은데 실행하고는 싶지 않을때 가볍게 사용하기 좋다.

check나 run, build를 사용하면 unused import 같은 자잘한 경고들이 뜰 수 있다.

이렇게 말이다.

그리고 저렇게 발생한 기본 경고는 cargo fix 명령을 통해 정정할 수 있다.

그럼 이렇게 고쳐질 것이다.

경고를 임의로 무시하는 것도 가능하다. 별도 포스트를 참조한다.
https://blog.naver.com/sssang97/222828143796?trackingCode=blog_bloghome_searchlist



cargo clippy

clippy는 기본 체크에서 수행하지 않는 컨벤션 수준의 강력한 린트 기능을 제공한다.
원래는 서드파티였으나 이제는 기본 도구로 편입되었다.

이런 식으로 코드가 간결하고 명확해질 수 있는 형태의 개선 방향을 늘 제시해준다.


한땀한땀 수정하기 귀찮다면 이것도 fix 명령을 통해 일괄 수정이 가능하다.
fix로 커버되지 않는 것도 있긴 하다.

이것도 기본 경고와 마찬가지로 allow 매크로를 사용하면 무시 처리할 수 있다.

솔직히 너무 잔소리가 과한 것도 있어서 다 들어주는건 어렵다.
