diesel에서 테이블의 값을 select하는 방법에 대해서 다시 개괄적으로 정리해본다.

테스트할 테이블 구조는 다음과 같이 간단하다.

아래는 데이터 매핑용 구조체 타입이다.



실행하는 방법

일단 select는 table! 매크로로 자동 생성된 test::dsl의 테이블 값 test에서 다음의 메서드들을 호출하는 것으로 수행할 수 있다.

load와 get_results는 완전히 동일한 메서드인데, 배열 형태로 select값들을 가져와준다.
get_result와 first는 기능상 거의 동일하게 하나만 가져오는데, first는 limit 1를 달아줘서 대체로 성능은 조금 더 빠를 것이다.

그리고 당연히 이런 전체 select만으로는 제대로 데이터를 사용할 수가 없다.
저 테이블 객체는 QueryDsl이란 트레잇을 구현하는데, 여기에는 filter, limit, order 등의 select에 부가적으로 다는 문장들을 지원한다.
https://docs.diesel.rs/diesel/query_dsl/trait.QueryDsl.html
그리고 이 메서드들은 Self를 다시 반환하므로 체이닝 형식으로 연달아 사용할 수 있다.
이런 식으로 말이다.

test::dsl::test.fiter(...).order_by(...).limit(3).load();


filter: 조건절

filter는 sql에서 정확하게 where절에 해당한다.
보통 dsl의 컬럼으로 표현식 생성을 위한 함수를 호출하고 그 값을 인자로 전달한다.
동등 연산자인 =의 경우에는 eq 함수를 사용한다.
아래는 id=3인 값을 읽어오는 쿼리다.

여러 조건을 달려면 filter를 연속해서 호출하면 된다.
그리고 아래에서는 < 연산자의 lt, > 연산자의 gt를 사용했다.

order: 순서 부여

이건 SQL의 order by에 대응된다.
인자로는 dsl 컬럼에 asc나 desc를 호출한 값을 넘겨주면 된다.
아래는 id로 내림차순 정렬을 수행한 쿼리다.

여러 정렬조건을 주고 싶다면 튜플로 묶어서 주면 된다.




offset&limit: 자르기

이것도 SQL의 offset, limit와 완전 대응된다.
특별한게 있지는 않다. offset은 조회할 목록 중에서 자르기 시작할 인덱스를 받고, limit는 총 몇개까지 잘라올지를 선택한다.




select: 컬럼 지정

따로 뭘 걸지 않고 데이터를 읽어오려면 위처럼 그냥 컬럼 전체를 다 가져와버린다.
일부 컬럼만 가져오고 싶다면 select로 dsl 컬럼을 지정하면 된다.

여러개의 컬럼을 가져오고 싶다면 컬럼들을 튜플로 묶어서 던진다.

그리고 select를 할때 어느정도 값의 가공을 할 수도 있다.
그럴때는 컬럼 dsl 값의 메서드들을 사용할 수가 있겠다.
아래의 concat은 문자열을 연결해주는 함수다. ||로 변환될 것이다.
