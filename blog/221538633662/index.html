[C++] MySQL 연동 in Visual Studio

  • 2019.05.16
  • ::
  • 1 min read
#C++
[원본 링크]

보통 mysql에 기본적으로 깔려있는건 그냥 C버전 API다.
그냥 C버전을 써도 되긴 하는데, 이왕 C++을 쓸거면 제대로 써야하지 않겠는가?

일단 여기서 C++버전 커넥터를 다운받으면 된다.
https://dev.mysql.com/downloads/connector/cpp/8.0.html
나는 64 버전으로 깔고 그냥 기본설정으로 깔았다.
그럼 양쪽 프로그램 파일즈의 MySQL 디렉터리에 이렇게 생길거다.

여길 대충 찝어놓자.
바로가기를 만들어서 어디에 모아놓든 간에.

아 그리고, 이 C++ 커넥터는 boost 라이브러리에 종속적이다.
boost를 먼저 설치하고 세팅할 것을 알린다.

나는 비주얼 스튜디오에서 작업을 했는데.. 원한다면 다른걸 써도 된다.

여튼 나는 dll 달고다니는게 싫어서 정적으로 링크를 할 것이다.
근데 정적 버전은 디버깅 모드에서는 안 도니까, 릴리즈 모드로 해야한다.
sfml은 static 버전도 디버깅 모드가 있었는데... 정성이 부족한 것 같다.

여튼 프로젝트 속성에 들어가서 릴리즈 64비트로 바꾸고,
포함 디렉터리에 boost와 커넥터 헤더 위치를 추가해준다.

추가 라이브러리 디렉토리에도 커넥터와 부스트의 라이브러리 경로를 넣어준다.

추가 라이브러리 종속성에는 저 두놈을 넣어준다.

만약 dll이랑 연결하고 싶다면, 저기서 static만 없는 버전이 있다. 그걸 달면 된다.


코드는 이렇게 짜면 된다.
아래 코드는 그냥 디비에 있는걸 select로 읽기만 한다. 데이터는 미리 넣어뒀다.

#include "pch.h"
#include <iostream>
#include <memory>

#define STATIC_CONCPP //static 버전이라는 걸 알림!
#include <mysql/jdbc.h>

int main()
{
	constexpr auto host_name = "localhost";
	constexpr auto username = "admin";
	constexpr auto password = "1234";

	try {
		auto driver = get_driver_instance(); //얜 해제하면 안됨
		auto connector = std::unique_ptr<sql::Connection>(driver->connect(host_name, username, password)); //연결
		connector->setSchema("TestDB"); //특정 DB 접속

		auto statement = std::unique_ptr<sql::Statement>(connector->createStatement());

		//쿼리 보내고 받아옴
		auto result_set = std::unique_ptr<sql::ResultSet>(statement->executeQuery("SELECT * from PeopleTable"));

		//계속 출력
		while (result_set->next())
			std::cout << result_set->getString("id") << ' ' << result_set->getString("name") << ' ' << result_set->getString("age") << std::endl;
	}
	catch (sql::SQLException& e) {
		std::cout << e.what() << std::endl
			<< "error code: " << e.getErrorCode() << std::endl
			<< "sqlstate:" << e.getSQLState() << std::endl;
	}
}
