[AWS] QLDB

  • 2022.11.17
  • ::
  • 4 min read
#AWS
[원본 링크]

QLDB는 Amazon Quantum Ledger Database의 축약으로, 블록체인 기반의 데이터베이스 시스템이다.
금융 거래의 기록처럼 정확성과 영구성, 히스토리, 보안 등이 중요한 부분에 최적화되어있다.

mongoDB처럼 비정형의 JSON 형태 데이터를 관리한다.




원리

QLDB는 놀랍게도 블록체인과 같은 구조로 데이터를 관리한다.
블록체인과 다른 점이라면, 분산형이 아니라 AWS에서 모든걸 통제하는 중앙집중형이라는 것 뿐이다.

이걸 통해 각 트랜잭션의 무결성을 완벽하게 보장한다고 하는데.. 난 잘 모르겠다.
이게 그만큼이나 대단한 이점이 있나?

아무튼 삭제가 불가능하고 모든 히스토리가 보존된다는 점에서는 꽤 괜찮아보이긴 한다.




비용

완전관리형 서버리스 시스템이라, 기본비용같은건 따로 없다.
사용한 만큼만 비용을 낸다.


딱히 막 그렇게 비싼 편은 아니다.

전송에도 요금을 좀 문다. 동일 리전 내에서만 사용할 경우에는 공짜다.

더 자세한 것은 문서를 참조하길 바란다.
https://aws.amazon.com/ko/qldb/pricing/?pg=ln&sec=hs




RDB와의 비교

구조는 RDB들과 비교하면 대충 이런 형태를 가진다.





원장(ledger) 생성

원장은 RDB의 데이터베이스와 동등한 개념이다. 하나의 원장에는 여러개의 테이블이 들어갈 수 있다.
먼저 만들어주자

난 대충 다 기본값으로 줬다.


그럼 이렇게 만들어진다.




테이블 생성

이제 저기다 테이블을 추가해보겠다.

따로 뭘 해야하는게 있지는 않다.
이름만 잘 지어준다.

그럼 이렇게 만들어질 것이다.




쿼리 작성해보기 (PartiQL)

쿼리 버튼을 누르면 쿼리를 실행시킬 수 있는 창으로 이동한다.

QLDB는 SQL은 아닌데, 그거랑 비슷한 PartiQL이란 고유의 신택스를 제공한다.




INSERT 문

만약 insert를 하고 싶다면, 이런식으로 할 수 있다.
values를 JSON으로 받는 기기묘묘한 형태다.

INSERT INTO transactions VALUE
{
    'id' : 1, 
    'from': 1232324,
    'to': 44444, 
    'money': 10000
}

그럼 잘 들어가질 것이다.
그리고 "documentId"라는 기본키가 암시적으로 만들어진다.




SELECT 문

이건 그냥 비슷하게 쓸 수 있다.

이런식으로 조회가 잘 된다.

where 절도 사용 가능하다.

보통 이런식으로 동등 조건이나(=) In 표현 정도로만 필터를 걸고, 아래 탭에서 설명할 인덱스가 필수적으로 존재해야 한다. 안그러면 성능에 심각한 문제가 생길 수 있다.

그리고 뭐 order by나 group by 같은 복잡한 신택스는 없고, join 정도만 지원한다.

가급적이면 이런 1차원적이면서 단순한 쿼리만 사용하는게 좋다. 나머지는 그냥 프로그램에서 처리하자..




UPDATE 문

update문도 SQL과 거의 같다.




DELETE 문

이것도 다를건 없다.
다만, 겉으로만 삭제해서 없는것처럼 보이는 것 뿐이지, 내역에는 다 남아있다.




인덱스

RDB들과 비슷하게 인덱스도 지원한다. QLDB에서 인덱스는 항상 필수다.

이게 없으면 SELECT는 항상 풀스캔을 돌리는데, QLDB는 구조상 하나의 쿼리가 무조건 30초 이내에 완료가 되어야만 한다. 그렇지 않으면 그 쿼리는 버려진다.
게다가 너무 느린 쿼리는 동시성에 영향을 준다고 한다.

이런식으로 쓰면 된다.

create index on 테이블명(필드명)

그래서 위의 예제에서 id에 인덱스를 걸고 싶다면, 이런식으로 하면 된다.



신택스에 대한 더 자세한 정보를 원한다면, 문서를 참조해보길 바란다.
https://docs.aws.amazon.com/qldb/latest/developerguide/console_QLDB.html
