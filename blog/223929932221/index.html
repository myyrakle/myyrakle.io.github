clickhouse에서 SELECT문으로 데이터를 조회하는 방법을 대강 다뤄보겠다.




쿼리 기본

SQL들이 통상적으로 제공하는 기본 구문들은 다 있다.

SELECT *
FROM 테이블
JOIN ... ON ...
WHERE ...
OFFSET ... LIMIT ...
GROUP BY ...
ORDER BY ...

WHERE절, OFFSET LIMIT 절, ORDER BY 절 등은 다른 DB들과 거의 동등하다.

Group by와 Count 등의 집계 함수를 사용하면 다른 DB들과 마찬가지로 집계 연산을 할 수 있다.




Limit By 절

clickhouse에서 유용한 기능 중 하나다.
그냥 Limit와는 키워드만 같을뿐 완전히 다른 기능이다.

알다시피 clickhouse에서는 Update나 Unique Key 제약이 잘 안되는 편인데, 그래서 ReplcaingMergeTree 같은걸 쓰더라도 키가 같은 값이 항상 노출될 수 있다.

그래서 이렇게 키가 같은 값이 보존되는 경우가 빈번한데,

ReplacingMergeTree의 Optimize 기능을 사용해서 병합하거나, 직접 삭제하는 것도 방법이지만 그건 대형 테이블에서는 그다지 효율적이지 않다.
권장되는 방법은 Limit 1 by id를 걸어서, 해당 id를 기준으로 최신 값만 가져오도록 하는 것이다.

문서 참조
https://clickhouse.com/docs/sql-reference/statements/select/limit-by





Sample 절

데이터 샘플링을 위해 제공되는 기능이다.
대형 데이터셋에 대해서 대략적인 수치만 빠르게 뽑아내고 싶을때 사용할 수 있다 .

비율 기반이나 행 개수 기반으로 데이터를 대강 스캔해서 처리하도록 할 수 있다.

SELECT *
FROM table
SAMPLE 0.1 -- 10%만 샘플링

SELECT *
FROM table
SAMPLE 1000 -- 행 1000개만 샘플링

참고로 이건 테이블 Setting에 샘플링 키 설정이 되어있어야 사용할 수 있다.

설정 방법은 다음 문서를 참조한다.
https://clickhouse.com/docs/engines/table-engines/mergetree-family/mergetree#sample-by




WIth 절

clickhorse도 복잡한 쿼리를 단순화하기 위한 용도로 with절을 제공한다.

이를 통해 임시 쿼리 결과값을 with으로 바인딩한 이후에 재사용하거나 할 수 있다.
