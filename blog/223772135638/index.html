[Grafana Tempo] 캐싱 설정 (with Memcached)

  • 2025.02.24
  • ::
  • 1 min read
#Monitoring
[원본 링크]

Tempo는 자체적으로 TraceQL 등을 이용한 쿼리 기능을 제공하지만, 읽기 성능 부분에서는 아쉬운 부분이 있을 수 있다.

Tempo는 쿼리 성능 향상을 위해서 인덱스 잡고 그런건 없고, 그냥 캐싱을 구성하는 것을 권장한다.
캐싱 백엔드로는 Redis, Memcached가 지원된다.

캐시는 모든 데이터를 저장한다거나 하는건 아니고, 쿼리에서 조회된 블록의 블룸 필터를 저장하는 형식이다.

Docker compose 기준으로 다룬다.

  memcached:
    image: memcached:1.6.29
    ports:
      - "11211:11211"
    environment:
      - MEMCACHED_MAX_MEMORY=64m # Set the maximum memory usage
      - MEMCACHED_THREADS=1 # Number of threads to use
    restart: always

이런 식으로 memcached DB가 있다고 하면

tempo config yaml에서 이런 식으로 구성하면 된다.

cache:
  background:
    writeback_goroutines: 5
  caches:
    - roles:
        - frontend-search
      memcached:
        addresses: dns+memcached:11211

여기선 docker 자체 dns로 내부 통신을 타도록 했는데, 다른 영역에 있다면 주소를 쓰거나 도메인을 넣거나 하면 된다.
