[원본 링크]

loco.rs는 Rust판 Rails를 표방하면서 나온 백엔드 프레임워크다.

기본적인 특징은 이 정도다.

  • 기존 Rust ORM인 SeaORM과 통합되어있다.

  • 내장 서버는 axum이다. 이건 자체 서버 구현이 아니고 axum의 래퍼다.

  • 전용 CLI를 통해 편의성을 제공한다.

  • 프로젝트 구조를 사전 정의해서 제공한다.

  • 전용 CLI를 통해 보일러플레이트들의 생성이 지원된다.






설치

loco는 실행이나 코드 생성 등을 하려면 전용 CLI를 설치하는 것이 요구된다.

cargo install loco

이렇게 뜨면 잘 설치된 것이다.




프로젝트 구성

new 명령을 통해서 프로젝트 초기화를 시도할 수 있다.

loco new

소감

계속 바이럴이 떠서 뭔가 싶어서 한번 실행하면서 봤는데, 실망스러운 부분이 너무 많았다.

서버는 자체 구현이 아니고 그냥 axum 실행기에 불과한데다, ORM도 다른거 갖다가 짜깁기해둔거다.
전반적으로 그냥 엉성하게 엮어둔 누더기골렘으로밖에 보이지 않는다. 프로젝트 레포 가서 Cargo.toml 디펜던시 보면 느낄 수 있을 것이다.

보일러플레이트 만드는 것도 너무 1차원적이고 단순하다. 서버를 레이어로 분리하는 것은 전혀 고려하지 않은 것 같다.

당연히 Swagger 자동 생성 기능 같은것도 들어있을 것이라 생각했는데, 없다. 할 생각도 없고 알아서 하라는 입장이다.
https://github.com/loco-rs/loco/issues/632
