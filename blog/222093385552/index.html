[Rust] 필수 유틸: cargo-edit

  • 2020.09.18
  • ::
  • 1 min read
#Rust
[원본 링크]

cargo는 pip나 npm 등과는 다르게 일반 라이브러리는 직접 설치할 수가 없다.

cargo install로 라이브러리를 설치하려고 하면

바이너리가 없다고 에러를 띄운다.
애초부터 cargo install은 바이너리, 완성된 하나의 프로그램만을 설치하는 명령이기 때문이다.

cargo 프로젝트에 라이브러리를 붙이려면 근본적으로 cargo.toml을 수정하는 것이 최선이다.

그런데 이 얼마나 귀찮은 짓인가.
그냥 커맨드로 바로바로 설치할 수는 없을까?

당연히 있다!

cargo-edit이라는 끝내주는 녀석을 사용하면 이를 손쉽게 처리할 수 있다.
cargo install로 설치하면 된다.

시간이 좀 걸린다.

설치가 잘 됐다면, 프로젝트 위치에서 cargo add 명령으로 바로 종속성을 추가할 수 있다.

그럼 자동으로 toml이 변경된다.

rm 치면 또 지워지고

그리고 종속성 버전이 낮은 상태에서

upgrade를 치면

최신버전으로 업그레이드된다.
