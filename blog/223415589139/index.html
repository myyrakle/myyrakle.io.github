[Pyroscope] Grafana - Pyroscope

  • 2024.04.15
  • ::
  • 2 min read
#Monitoring
[원본 링크]

Pyroscope는 Grafana에서 제공하는 지속적 프로파일링(Continuous Profiling)을 위한 모니터링 도구다.
프로파일링을 실시간으로 한다는 소리다. 성능/사용량 모니터링에 있어서는 가장 집착스런 방법론이다.

계속해서 프로파일 데이터를 쏴줘야 하기 때문에, 당연히 리소스를 좀 소모할 것이다.

그래도 한번 세팅해두면 문제의 해결 추적은 굉장히 편리해질 수 있다.


이런 식으로 쓴다. Pyroscope 서버를 띄워넣고 그냥 거기다가 주기적으로 보고를 보내는 것이다.




Docker로 실행

Pyroscope는 여러가지 설치/실행 방법을 제공한다. linux라면 바로 실행파일을 받아서 사용할 수도 있고, docker에서 띄울 수도 있고, k8s에서 구성할 수도 있다.

일단 여기에서는 Docker로 실행해보겠다.

다음 명령을 사용한다.

sudo docker pull grafana/pyroscope:latest
sudo docker network create pyroscope-demo
sudo docker run --rm --name pyroscope --network=pyroscope-demo -p 4040:4040 grafana/pyroscope:latest

그래서 이렇게 배너가 뜨면 잘 실행된 것이다.

헬스체크는 다음 경로로 할 수 있다.




페이지 접속

4040 포트로 접속을 하면 바로 제어 페이지가 뜬다.

별다른 세팅을 하지 않았다면 pyroscope가 실행된 환경의 프로파일만을 보여줄 것이다.
그냥 기본으로 보여주는거긴 한데, 애초에 실제 실행환경에서 이걸 띄울 일은 잘 없을테니 거의 장식이다.

Client가 설정되었을 경우엔, 저 최상단 탭을 골라서 수치를 확인하면 된다.

이런 식으로 시각화된다. 이건 소모한 CPU time에 대한 프로파일이다. 함수 단위로 측정된다.


우상단 탭을 조작하면 조회 범위를 조정할 수 있다.
