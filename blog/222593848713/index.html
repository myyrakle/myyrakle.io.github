[Elasticsearch] 설치 in Linux

  • 2021.12.13
  • ::
  • 2 min read
#Elasticsearch
[원본 링크]

본 포스트에서는 Elasticsearch를 Linux 환경에서 직접 설치하고 구축하는 법에 대해 다뤄본다.

테스트 환경은 우분투 18이다.




설치

설치를 하려면 다음 명령들을 차례대로 실행한다.

curl -fsSL https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
echo "deb https://artifacts.elastic.co/packages/7.x/apt stable main" | sudo tee -a /etc/apt/sources.list.d/elastic-7.x.list
sudo apt-get update
sudo apt install elasticsearch




설정파일 수정

설치가 다 됐으면 설정파일을 열어서 기본적인 설정을 좀 건드려준다.
가장 먼저 해야할 것은 네트워크를 로컬호스트로 설정하는 것이다.

sudo vi /etc/elasticsearch/elasticsearch.yml

그래야 내부에서 로컬호스트로 접근이 가능하다.




엘라스틱서치 가동

다 됐으면 다음 명령을 사용해 엘라스틱서치가 시작되도록 한다.

sudo systemctl start elasticsearch
sudo systemctl enable elasticsearch

그럼 다음과 같이 요청을 날릴 수 있을 것이다.




외부 개방

아예 외부에서도 접근을 해야 한다면, 일단 방화벽 9200 포트를 열어준다.

그리고 설정파일을 열어서 아까 로컬호스트로 해뒀던걸 다시 0.0.0.0로 깔아준다.

그리고 discovery.seed_hosts라는 항목도 주석 해제해서 똑같이 추가해준다.

discovery.seed_hosts: ["0.0.0.0"]

그리고 systemctl로 재시작을 하면 된다.

그럼 이제 외부에서도 접근이 가능해질 것이다.




보안 설정

근데 저 상태로만 둔다면, 아무나 막 인덱스를 생성하거나 읽어올 수가 있다.

당연히 이러면 안되고, 패스워드 설정해서 권한을 걸어야 한다.

아까의 설정파일을 열어서 아래 문구를 추가해준다.

xpack.security.enabled: true
xpack.security.transport.ssl.enabled: true

그리고 재부팅하면

이제 막 들어가는건 막힐 것이다.




비밀번호 설정

그러면 비밀번호를 설정해보자.
다음 명령을 실행한 다음, 비밀번호를 다 깔아주면 된다.

cd /usr/share/elasticsearch/
sudo ./bin/elasticsearch-setup-passwords interactive

그럼 이제 다음과 같은 식으로만 로그인이 되어야 접근이 가능해질 것이다.

여기서는 주소에 사용자:패스워드를 때려박았지만, 토큰 등을 통해서도 접근이 가능하다.
그건 차후 정리해두겠다.
