[Rust] fltk GUI

  • 2022.12.18
  • ::
  • 1 min read
#Rust
[원본 링크]

fltk는 경량형의 크로스플랫폼 GUI 라이브러리다.
원래는 C++ 기반 라이브러리인데, Rust에도 바인딩이 잘 제공되는 편인 것 같다.

무엇보다 문서화나 사용법이 간편해서 한번 사용해보게 되었다.

이것저것 뭐 깔고 dll 우르르 받을 필요 없이 그냥 디펜던시 하나면 사용이 가능하다.
toml에 종속성을 추가해준다.

fltk = { version = "^1.3", features = ["fltk-bundled"] }

그리고 이런식으로 짜면 기본적인 동작은 확인해볼 수 있을 것이다.
간단한 창 하나만 띄우는 코드다.

use fltk::{app, prelude::*, window::Window};

#[tokio::main]
async fn main() {
    let app = app::App::default();
    let mut wind = Window::new(100, 100, 400, 300, "Hello from rust");
    wind.end();
    wind.show();
    app.run().unwrap();
}

잘 된다.




버튼 추가해보기

이번에는 버튼을 추가해서 뭔가 동작은 하게 만들어보자.

기존 윈도우에 버튼을 추가하는 법은 꽤나 간단하다.
end 메서드를 찍기 전에 객체를 생성만 하면 자동으로 바인딩이 된다. 그리고 버튼에 콜백 메서드만 찍어주면 된다.

use fltk::{app, button::Button, prelude::*, window::Window};

fn main() {
    let app = app::App::default();
    let mut wind = Window::new(100, 100, 400, 300, "Hello from rust");
    let mut but = Button::new(160, 210, 80, 40, "Click me!");
    wind.end();
    wind.show();

    but.set_callback(move |_| println!("눌림!")); // the closure capture is mutable borrow to our button
    app.run().unwrap();
}

누를때마다 콘솔로그가 잘 찍힐 것이다.

이번에는 콘솔만 찍는게 아니라 버튼을 누를떄 뭔가 상호작용을 하도록 해보겠다.
프레임이라는 컴포넌트를 하나 깔고, 버튼을 눌렀을 때만 텍스트가 보이도록 할 것이다.

use fltk::{app, button::Button, frame::Frame, prelude::*, window::Window};

fn main() {
    let app = app::App::default();
    let mut wind = Window::new(100, 100, 400, 300, "Hello from rust");
    let mut frame = Frame::new(0, 0, 400, 200, "");
    let mut but = Button::new(160, 210, 80, 40, "Click me!");
    wind.end();
    wind.show();
    but.set_callback(move |_| frame.set_label("Hello World!")); // the closure capture is mutable borrow to our button
    app.run().unwrap();
}

그럼 처음에는 비어있겠지만

버튼을 누르면 레이블이 깔리면서 텍스트가 노출된다.

구조는 그럭저럭 나쁘지 않은 것 같다.
