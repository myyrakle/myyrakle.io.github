[Go] lint: golangci-lint

  • 2023.09.26
  • ::
  • 2 min read
#Go
[원본 링크]

golangci-lint는 아마 go 환경에서 그나마 쓸만한 수준의 lint인 것 같다.

golint라고 go 그룹에서 관리한게 있었는데, 유기당했더라




설치

먼저 cli를 설치해야 한다.

go install github.com/golangci/golangci-lint/cmd/golangci-lint@v1.54.2

이렇게 실행이 되면 잘 깔린 것이다.




린터 기본 사용법

linters 명령을 사용하면 사용 가능한 린터 속성을 조회할 수 있는데

아래 링크에 들어가면 더 상세한 옵션과 정보를 볼 수 있다.
https://golangci-lint.run/usage/linters/

아무튼, run을 사용하면 바로 린터를 돌려볼 수 있다.
자기들만의 기준으로 오류는 아니지만 뭔가 이상한 코드를 찾아준다.




린터 커스텀하기

이런걸 쓰면 보통 기본값을 쓰지는 않고, 린트 옵션들을 설정해서 쓰는게 보통이다.
프로젝트 루트 경로에 .golangci.yml 라는 파일을 만들어서 옵션을 적당히 정의해준다.

아래 설정은 다른걸 다 끄고 지나치게 긴 코드를 경고하는 lll 옵션을 켠다.

linters:
  disable-all: true
  enable: 
    - lll

linters-settings:
  lll: 
    line-length: 120
    tab-width: 1

저걸 해둔상태에서 돌리면

한줄에 120자가 넘는 기다란 코드를 다 찾아줄 것이다.
대충 이런 느낌으로 쓰는 것이다.




Visual Studio Code과 통합

vscode에서도 간단하게 lint에 대한 기능을 사용할 수 있다.
setting.json에 아래 옵션을 추가하고, 프로젝트 루트경로에 .golangci.yml를 적당히 깔아주면 된다.

 "go.lintTool": "golangci-lint",
 "go.lintFlags": [
    "--fast"
  ]

그럼 run 명령을 사용할 것도 없이 편집기에서 경고를 바로 표시해준다.




Jetbrain Goland와 통합
