rupring v0.14.0 배포

  • 2025.09.22
  • ::
  • 2 min read
#라이브러리
[원본 링크]

찔끔찔끔 좀 치다가 간만에 배포 한번 말았다.
주요 변경사항들은 다음과 같다.




1. HTTP2 지원 제거

생각해보니까, 사실 이거 쓸 일이 딱히 없어서 걷어냈다.

대부분의 서버 인프라 환경에서는 HTTP1.1로 충분한 효율성이 나오는데다, 서버마다 TLS 붙이는 건 거의 뇌절에 가깝기 때문이다.

게다가 이거 하나때문에 코드베이스 관리가 너무 복잡해지는 면도 있었다. 나중에 다시 추가할 일이 생길지도 모르지만, 일단은 버린다.




2. Streaming 및 SSE 지원

이제 SSE가 바로 지원된다.
사용 구조는 최대한 단순하고 직관적인 형태로 잡아봤다.

#[rupring::Get(path = /sse)]
#[tags = [user]]
#[summary = "SSE 페이지"]
pub fn serve_sse(request: rupring::Request) -> rupring::Response {
    rupring::Response::new()
        .sse_stream(async move |stream_handler|  {
            let mut count = 0;
            loop {
                if stream_handler.is_closed() {
                    println!("Client disconnected, stopping SSE");
                    break;
                }
                let event = rupring::http::sse::Event::new()
                    .event("custom-event")
                    .id("event-id-1")
                    .retry(300)
                    .data(format!("This is custom event number {}", count));
                if let Err(e) = stream_handler.send_event(event).await {
                    eprintln!("Error sending message: {}", e);
                }
                count += 1;
                tokio::time::sleep(tokio::time::Duration::from_secs(2)).await;
            }
        })
}

그냥 이런 식으로 콜백 넣고 돌리면 된다.
커넥션 중단 감지는 직접 해야한다.


그러면 이렇게 잘 돈다.




3. 멀티파트 파싱 기능 추가

이제 멀티파트 파싱을 직접 지원한다.
바로 멀티파트 형태로 파일을 받아다가 처리할 수 있다.

#[rupring::Post(path = /multipart-upload)]
#[summary = "단순 파일 업로드 API입니다."]
#[description = "별다른 기능은 없습니다."]
#[tags = [root]]
pub fn multipart(mut request: rupring::Request) -> rupring::Response {
    for (i, file) in request.files.iter().enumerate() {
        std::fs::write(format!("{i}.foo"), &file.data).unwrap();
    }

    rupring::Response::new().text("Hello, World!")
}

#[rupring::Get(path = /multipart-upload-page)]
#[summary = "단순 파일 업로드 API입니다."]
#[description = "별다른 기능은 없습니다."]
#[tags = [root]]
pub fn multipart_page(request: rupring::Request) -> rupring::Response {
    rupring::Response::new().html(
        r#"
        <html>
            <body>
                <form action="/multipart-upload" method="post" enctype="multipart/form-data">
                    <input type="file" name="file1" />
                    <input type="file" name="file2" />
                    <input type="submit" value="Submit" />
                </form>
            </body>
        </html>
        "#,
    )
}




4. 기타등등

  1. 편의용 Response::download 함수가 추가됐다. 이걸 쓰면 바로 content_disposition 헤더를 조정해서 브라우저에서 즉시 다운로드를 하도록 동작을 정의한다.
  2. Response::html 숏컷 함수가 추가된다. 그냥 text의 한 종류인데 content-type만 알아서 잘 조정해준다.
  3. Response::cache_control 함수가 추가된다. 이를 통해 명시적으로 캐시 정책을 조절할 수 있다.
  4. 쿠키를 자동으로 파싱해서 넣어주는 플래그 "server.cookie.auto-parsing-enabled"가 추가된다. 이게 켜지면 request 컨텍스트의 header에 자동으로 쿠키 값이 파싱돼서 들어간다.
  5. URI 길이 제한 플래그 "server.request.uri.max-length"가 추가된다. 초과하면 414가 떨어진다.
  6. 헤더 크기 제한 플래그 "server.request.header.max-length"와 "server.request.header.max-number-of-headers"가 추가된다. 초과하면 400이 떨어진다.
  7. request body 크기를 제한하는 "server.request.body.max-length" 플래그가 추가된다.
  8. reqeust context에 ip, protocol 등의 값을 포함하는 metadata 필드가 추가된다.
  9. 기타 버그 수정
