[원본 링크]

Github은 수많은 Git 호스팅 서비스들 중 하나일 뿐이다. 근데 Github은 또 Github만의 확장 기능이 좀 있어서 Git CLI로는 제어하기가 불편하거나 어려운 부분들이 존재한다.
대표적으로 PR이나 Issue 제어 같은거 말이다.

Github CLI는 그런 Github 특화 기능들에 대한 편리한 제어 기능을 제공하는 도구다.




설치

Linux, 데비안 계열은 아래와 같이 설치할 수 있다. 좀 번잡스럽다.

sudo mkdir -p -m 755 /etc/apt/keyrings && wget -qO- https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null \
&& sudo chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
&& echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
&& sudo apt update \
&& sudo apt install gh -y

mac은 brew로 설치할 수 있다.

brew install gh

설치가 잘 됐다면 gh 명령을 사용할 수 있게 될 것이다.




로그인하기

gh cli는 git와 인증 정보를 공유하지 않는다. 따로 로그인을 해줘야 한다.

gh auto login

로그인에는 토큰 방식과 브라우저 로그인이 있다. 난 브라우저로 했다.


브라우저 띄우고 인증토큰 입력해주면 된다.


끝




레포지토리 확인

repo 명령어들을 사용하면 레포지토리들에 대한 제어를 수행할 수 있다.
list는 리스트를 조회한다.

view를 사용하면 레포지토리 정보를 바로 확인할 수 있다.
리드미 정도만 보여준다.

set-default를 사용하면 앞으로의 액션들에 대해서 기본 레포지토리로 설정이 된다.

깔아주는게 편하다.




이슈

이슈에 대한 제어는 issue의 서브커맨드로 처리할 수 있다.




PR 확인

내가 이걸 설치한 이유가 사실 이거다.
pr list로 PR 목록을 조회할 수 있고

checkout을 사용하면 외부 repo에 대한 pr도 바로 체크아웃이 가능하다.

이걸 git CLI로만 하려면 상당히 지리해진다.
