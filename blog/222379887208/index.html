[AWS] elastic beanstalk: 시작

  • 2021.06.01
  • ::
  • 2 min read
#AWS
[원본 링크]

elastic beanstalk는 웹서버를 올리기 위한 종합 패키지라고 보면 된다.

이게 뭐 독립적인 기능은 아닌데, 기본적인 서버 구축부터 스펙 자동조절, 로드밸런싱까지 자동으로 세팅을 해주기 때문에 상당히 편리하다 할 수 있다.
기본 소스만 올리면 나머지는 대부분 알아서 해준다.

본 포스트에서는 간단하게 시작만 한번 해보겠다.




앱 만들기

먼저 앱을 만들어보자.
이름을 적어주고,

사용할 플랫폼을 고른다.
Node.js, Java, Ruby 등의 메이저한 환경은 대부분 지원하며, 그게 아니라도 Docker로 직접 올릴 수도 있다.

이대로 생성을 하면

몇분정도 걸릴 것이다.

따단




소스 업로드하기

이번엔 서버 소스를 작성해서 저기에 배포해보자.

아래에서 사용할 언어의 샘플 코드를 받아서 서버를 작성하자.
단순 서버만 작성하면 되는 게 아니라 빈즈톡 전용 설정파일같은 것도 좀 넣어줘야 하기 때문이다.
https://docs.aws.amazon.com/ko_kr/elasticbeanstalk/latest/dg/GettingStarted.DeployApp.html

그리고 여기서 필요한 만큼 수정해서 사용하면 되겠다.
저기서 빈스톡이 실행할 메인 소스는 app.js다 저것만 적절히 작성하면 된다.

나는 대충 2개의 경로만 가지는 단순한 서버를 구현했다.

const port = process.env.PORT || 3000;
const fs = require("fs");

const log = function (entry) {
    fs.appendFileSync(
        "/tmp/sample-app.log",
        new Date().toISOString() + " - " + entry + "\n"
    );
};

const express = require("express");
const app = express();

app.get("/", async (req, res) => {
    log("/ 탐");
    res.json({ foo: "bar", text: "으악" });
});

app.get("/test", async (req, res) => {
    log("/test 탐");
    res.json({ foo: "test", text: "테스트" });
});

app.listen(port, () => {
    console.log("## 서버 실행");
});

취향껏 구성하면 된다.

그리고 빈스톡 관리페이지로 가서

업로드 및 배포

방금 소스를 nodejs라는 이름의 파일로 압축해서 업로드하면 된다.

그러면 약간의 처리시간이 소요되는데,

다 되고나서 도메인에 들어가보면

작성한대로 잘 반영이 되어있을 것이다.

대충 이런식으로 써먹으면 된다.

다만 데이터베이스같은건 RDS나 다이나모 등으로 별도 구축해야 할 것이다.
