[PostgreSQL] pg_cron

  • 2025.02.11
  • ::
  • 2 min read
#PostgreSQL
[원본 링크]

pg_cron은 postgres 자체에서 일정 기반 스케줄러를 구현할 수 있는 강력한 확장 도구다.

서비스의 규모가 대단한게 아니라면, 스케줄러 서버 띄워서 관리포인트 늘리는 것보단, 이게 훨씬 간편하고 비용
효율적일 수 있다.




설치

관리형 PostgreSQL을 쓴다면 pg_cron이 기본적으로 설치되어있는 경우가 많다. AWS와 GCP를 포함한 상당수의 클라우드들이 이 정도는 다 해준다.
그런 경우엔 다음 절차를 다 생략하고 CREATE Extension만 해도 된다.

일반적인 리눅스 환경에서는 pg_cron 모듈을 설치하고, 설정도 조금은 해줘야 사용할 수 있다.

sudo apt-get -y install postgresql-16-cron

현재 사용중인 psql 버전에 맞는 cron 모듈을 먼저 설치한다.
psql 버전이 너무 낮다면 설치하기 매우 불편하거나 어려울 수 있다.

패키지 매니저에서 버전에 맞는걸 지원하지 않는다면, 직접 받아서 빌드해야 한다.

git clone https://github.com/citusdata/pg_cron.git
cd pg_cron
export PATH=/usr/pgsql-10/bin:$PATH
make && sudo PATH=$PATH make install

설치가 잘 되었다면 postgres 설정파일을 찾아서 열고

다음 2줄을 추가한 뒤에

shared_preload_libraries = 'pg_cron'
cron.database_name = 'postgres'

리스타트를 때리면 된다.

sudo systemctl restart postgresql

그리고 psql에 들어가서 확장을 활성화하면 기본 설정은 끝이다.

CREATE EXTENSION pg_cron;




기본 사용법

초기화가 잘 되었다면 cron.job, cron.job_run_detail과 같은 기본 테이블들이 생성이 될 것이다.

한번 테이블 하나 만들고, 거기에 1분마다 데이터를 집어넣는 cron을 정의해보겠다.

create table cron_test (
  id serial primary key,
  time timestamp default now()
);

테이블은 대충 만들었고


SELECT cron.schedule(
  'test_cron_job',  
  '* * * * *',             
  $$INSERT INTO cron_test (time) VALUES (now());$$
);

이런 식으로 정의하면 된다.
첫번째 인자가 job 이름, 두번째 인자가 cron식, 세번째 인자가 실행될 쿼리다.


그렇게 생성하면 job 목록에 항목이 추가된다.


그리고 좀 기다리면 의도대로 쌓이는걸 볼 수 있을 것이다.




Connection Failed 문제

클라우드를 쓰지 않고 직접 구성했는데, 추가적인 네트워크 설정을 하지 않았다면 cron job들이 줄줄이 실패하는 것을 볼 확률이 높다.

이렇게 말이다.

pg_cron 확장은 생각보다 postgres 자체와 강하게 결합되어있는 것은 아니다.
별도의 프로세스로 떠서 postgres에 TCP 요청을 때리는 식으로 동작하는데, 그래서 TCP 통신을 위한 보안 수준 설정이 필요하다.

일반적으로 권장되고 편리한 방법은, 그냥 로컬호스트 내에서는 자유롭게 통신이 가능하도록 열어주는 것이다.
pb_hba.conf 파일을 열어서 trust로 허용해주고

postgresql.conf에도 다음 한줄을 추가해준 뒤에

cron.host = ''

리스타트를 때리면 된다.

sudo systemctl restart postgresql
