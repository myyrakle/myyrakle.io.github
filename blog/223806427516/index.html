[LLM] Cline (in VSCode)

  • 2025.03.23
  • ::
  • 3 min read
#LLM
[원본 링크]

Cline은 LLM 기반으로 코드 생성을 보조해주는 VSCode 전용 확장이다. 코파일럿의 강화형이라고 보면 된다.
오픈소스이고, 코드 생성에 사용하는 LLM은 필요에 따라 교체 가능하다.

개략적인 부분만 다뤄본다.




설치 및 기본 설정

그냥 확장 검색 들어가서 찾고, 깔면 된다.

기본 세팅은 어려울게 없다.

그러면 왼쪽에 패널이 하나 뜰 것이고

들어가면 일단 LLM부터 설정하라고 한다.
실사용하려면 LLM 하나 구독하거나 직접 서버 띄운 다음에 API Key를 넣어서 써야 하는데, 일단은 Free 모드로 돌려보겠다.
기본 LLM은 Claude다.




Task 시작해보기

LLM 설정이 대충 되었다면, 이제 한번 코드 생성을 시켜보자.

Cline도 Copilot Chatting처럼 채팅을 통해서 원하는 결과를 유도해내는 형태인데, 그냥 답변만 알려주는게 아니라 전체 프로젝트를 탐색하고 실제로 코드 생성 결과를 파일로까지 만들어준다는 것이 다르다.
그리고 이런 작업 자체를 "Task"라고 지칭한다. 하나의 Task는 여러번의 피드백을 통해서 완성된다.

내 경우에는 도커 컨테이너를 강제 중단하는 함수가 필요해서 그걸 한번 시켜봤다.

그러면, 일단 처음부터 원하는 결과가 나오지는 않는다.
Task가 완료될때까지 계속해서 답변을 하고, Approve/Reject 선택지를 통해서 그게 맞는지를 내가 통제할 수 있다. 물론 이녀석을 충분히 신뢰한다면 Auto Approve를 켜도 된다.

아무튼 처음에는 프로젝트 구조를 파악하고, 어느 위치에 코드를 작성해야하며, 어떤 컨텍스트를 참조할지를 생각해서 그게 맞는지를 물어봤다.
대충 맞았다.


그러다보면 질답이 계속 이어지고, 충분한 컨텍스트를 얻었다고 판단하면


이제 코드 생성을 하기 시작한다.
실시간으로 파일을 만들어서 코드 생성을 하기도 하며, 이에 대해서도 파일로 저장을 할지 말지를 결정할 수 있다.
이번 경우에는 퀄리티가 꽤 괜찮았다.

Task 상태는 언제나 상단에서 확인하고, 강제로 종료를 할 수도 있다.

현재 Token이 얼마나 사용되었는지 비용이 얼마인지도 대충 보여준다.




API 설정

패널에서 우상단 톱니바퀴를 누르면 어떤 API를 사용할지 정할 수 있다.

내 경우에는 지금 Cline Free 계정을 통해서 Claude를 쓰고 있는데


Cline은 Claude와 Llama 정도만 모델을 지원한다.
그리고 구독제 같은데 아니라 크레딧을 구매해서 쓰는 형태로 제공된다.

근데 내가 보기엔 그냥 다른거 직접 찌르는게 나은거같다.

아무튼 굳이 Cline API로 중개하지 않아도 선택지는 매우 많다. 어지간한건 전부 된다.

제일 저렴한걸 원한다면 GCP Gemini도 좋고

AWS Bedlock, 미스트랄 등 선택지는 많다.
본인이 필요로 하는 수준과 비용을 고려해서 잘 선택하면 될 것 같다.




브라우저 시뮬레이션 가능 (Computer Use)

웹-브라우저 개발 환경이라면, 실제로 실행해서 자체적인 테스트와 피드백을 수행하게 할 수도 있다.
Computer Use라고 부르는데, Claude를 비롯한 일부 모델에서만 가능하다.




MCP와 다양한 데이터소스 활용

코드 생성에 필요한 여러가지 부가데이터를 전달해서 사용하게 할 수도 있다.
MCP Server 덕에 확장이 좀 된 부분이다.

예를 들어, Figma 플러그인을 활용하면 피그마에 있는 디자인을 기반으로 코드를 생성하게 할 수도 있는 것이다.

MCP 설치는 완전 자동형은 아니고 가이드만 해준다.




기타 설정: 언어 등

기본 설정은 영어인데, 원한다면 한국어 등으로 바꿔서 쓸 수도 있다.
