  • ::

[원본 링크]

제너릭에 대한 이론적인 설명은 구차히 하지 않겠다.
이 글을 보는 여러분이 이미 자바나 C++에 통달해서 일반화에 대한 개념을 완벽히 이해하고 있다고, 굳건히 믿고있기 때문이다.

제너릭을 적용한 Swap 함수는 이렇게 된다.
static void Swap (ref T a, ref T b) { ... }

제너릭을 적용한 클래스는 이렇게 된다.
class My
{
T value;
...
}

제너릭을 적용한 딜리게이트는 이렇게 된다. 이건 System.에서 기본제공되는 대리자다.
delegate int Comparision(T a, T b);


여기까지는 자바와 동일하다.

이제 타입파라미터에 제약을 걸어보자.


where
where은 시그너처, 그러니까 대개 중괄호가 시작되기 바로 전 부분에 붙여서 타입파라미터에 조건을 부여한다.
이런 타입이 아니면 못 넣는다는 거다.


그 조건에 위배되면 빨간줄을 긋는다.

어쨌든 이 제약조건에는 5가지가 있다.

타입파라미터명 : class //참조타입만 가능!
: struct //값타입만 가능!
: new() //매개변수 없는 생성자를 갖고있는 타입이라야 가능!
: 타입명 //이 타입이거나 이 타입을 상속받은 타입만!
: 다른타입파라미터명 //이 타입파라미터와 동일하거나 상속받았을 때만!
