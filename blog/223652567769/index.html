[MongoDB] Index: Partial Index

  • 2024.11.08
  • ::
  • 1 min read
#MongoDB
[원본 링크]

때로는, 인덱스를 부분적으로만 만들고 싶을 수 있다.

인덱스도 쌓이고 쌓이면 메모리나 디스크를 점유하는 양이 크게 늘 수 있는데, Partial Index는 그걸 좀 한정해서 줄이게 해주는 기능이다.

문법은 다음과 같다. partialFilterExpression에 조건식을 지정하면 된다.

db.컬렉션.createIndex(
  { 인덱스 대상 필드... },
  {
    partialFilterExpression: { 조건식... },
  }
);

그러면 partialFilterExpression 조건식에 부합하는 데이터만 인덱스를 만들고, 부합하지 않으면 인덱스를 만들지 않는다. 또 조회할때도 당연히 partialFilterExpression에 부합해야만 인덱스를 타고, 부합하지 않으면 인덱스 스캔을 타지 못한다.

이 기능은 단순히 인덱스 사이즈를 줄이는 것에만 강점이 있는 것은 아니다.
unique와 결합하면 강력하면서도 유연한 DB 수준 제약을 가할 수 있다.

예를 들면, user 데이터에 email 기준으로 유일성을 보장하되, 삭제된 회원은 제외하는 요구사항을 인덱스 하나만으로 만족시킬 수 있다.

db.users.createIndex(
  { email: 1 },
  {
    unique: true,
    partialFilterExpression: { deletedAt: { $eq: null } },
  }
);

이러면 삭제되어서 deletedAt!=null 데이터는 유일성 신경쓰지 않고 일단 쌓이고, 삭제되지 않아서 deletedAt=null인 것에 대해서만 이메일 중복을 거부할 것이다.
