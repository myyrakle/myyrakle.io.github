[Spring] JPA(Java Persistance API) 기본

  • 2019.12.29
  • ::
  • 3 min read
#개발노트
[원본 링크]

JPA는 ORM(Object Realation Mapping) 도구 중 하나다.
이게 뭔 소리냐면, OOP 언어의 객체와, SQL DB를 연결해준다는 것이다.

그리고 JPA 자체는 그냥 인터페이스라서 혼자 작동하지는 못한다.
그래서 구현체를 사용해야하는데, Hibernate, EclipseLink 등이 있다.
보통 하이버네이트를 더 쓰는것같다.

먼저 사용할 데이터베이스를 고르고, 의존성을 이래저래 추가해준다.
난 mysql을 사용한다.

잘 세팅해놓고


속성 파일에, 데이터베이스의 uri과, 사용자명, 비밀번호, 드라이버 클래스를 넣어준다.

안하면 실행도 안된다.



엔티티

그리고 @Entity 어노테이션을 사용해 엔티티 타입을 만든다.
이게 바로 테이블의 각 행(Row) 정보를 담고 관리하는 타입이다.
때문에 가장 기본이 되고, 가장 중요하다.

내가 사용할 테이블은 이렇게 되어있으므로

이렇게 정의할 수 있다.

각 필드의 이름을 일치시키고, 디폴트 생성자를 꼭 둬야한단걸 잊지 말자.
Getter나 Setter는 뭐 둬고 되고, 안 둬도 되고.

그리고 Entity의 인자로 테이블명을 줬는데, 이름을 주지 않으면 클래스명을 테이블 이름으로 간주한다.
Table 어노테이션으로 이름을 지정할 수도 있다.


Column 어노테이션

각 속성(열:column)의 이름을 명시적으로 지정하고 싶다면, Column 어노테이션을 사용할 수 있다.

이러면 실제 속성명과 필드의 이름이 일치하지 않아도 된다.



Jpa 레포지토리

그다음에 할 건 접근용 레포지토리 인터페이스를 정의하는 것이다.
레포지토리는 특정 테이블에 대한 접근을 제어한다.

JpaRepository에 정의한 엔티티타입과 Long만 집어넣으면 된다!
구현은 할 필요가 없다. 스프링이 알아서 마술을 부려서 채워준다.

그럼 이제 저걸 한번 돌려보자
일단은 코드의 단순함을 위해 main 함수에서만 돌려볼 것이다.
run 메서드는 context 객체를 반환하고, 여기서 빈 객체들을 꺼내올 수 있다.
위의 NameRepository 클래스는 @Repository를 달았으므로 당연히 빈즈 객체로 생성된다.
JpaRepository엔 이런저런 기본 메서드가 존재하는데, 그 중에서도 findAll은 select *의 효과를 낸다.
엔티티를 전부 가져온단 것이다.

데이터 삽입

데이터를 저장하려면 레포지토리의 save 메서드를 사용하면 된다.

URI로 name 파라미터를 받아서 DB에 삽입하도록 했다.


잘 들어간다.

그런데 삽입 시도 시에 아래와 같은 예외가 터질 수도 있다.

jpa jdbc4.MySQLSyntaxErrorException: Table 'test.hibernate_sequence' doesn't exist

이게... 데이터베이스와의 자동 매핑 기능이 제대로 기능하지 않아서 그렇다.
그럼 엔티티 정의에서 strategy를 AUTO에서 IDENTITY로 바꿔준다. 해당 식별자 그대로 따라가겠다는 뜻이다.

아니면 컬럼명을 명시해도 좋다.
