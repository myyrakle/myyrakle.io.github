[k8s] 파드(pod)

  • 2023.02.13
  • ::
  • 2 min read
#Kubernetes
[원본 링크]

파드(Pod)는 쿠버에서 관리하는 가장 기본적인 리소스 단위다.
쿠버에서 실질적으로 기능하는 대부분의 리소스는 Pod나 Pod의 변종이라고 보면 된다.

Docker의 컨테이너와 비슷하다고 볼 수 있다. 마찬가지로 도커 이미지(OCI)를 받고, 이미지를 그대로 실행하는 식으로 동작한다.

k8s의 Pod가 Docker의 컨테이너와 다른 점은, 하나의 Pod에 여러개의 컨테이너가 들어갈 수 있다는 것이다.
하나의 Pod는 각자가 고유한 IP를 가지며, 동일 Pod 내에 공존하는 컨테이너들도 같은 IP를 가진다.




기본 사용법

만만한 nginx 이미지를 가져다가 잘 동작하는지 돌려보도록 하자

먼저 배포를 위한 yaml 파일을 하나 작성한다.
쿠버에서 리소스의 관리는 항상 yaml로 시작해서 yaml로 끝난다.

apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod # 파드 이름
spec:
  containers:
  - name: container-name 
    image: nginx # 도커 이미지

이런식으로 쳐주면 된다.
이미지로는 nginx를 가져가게 했는데, 이러면 기본적으로 nginx라는 이미지를 도커허브에서 꺼내올 것이다.




파드 생성

create/apply 명령을 통해 생성할 수 있다.

kubectl create -f 설정파일
kubectl apply -f 설정파일

create는 생성 전용이고, 좀 더 권장하는 명령은 apply다. apply는 생성/수정이 다 한방에 된다.

문제가 없다면 이렇게 PENDING 상태로 추가되었다가

문제가 없다면 실행이 될 것이다.




파드 로그 보기

파드 내부에서 돌아간 것에 대한 로그를 보고싶다면 logs 명령을 사용하면 된다.

kubectl logs 파드명

그럼 실제로 실행된 내역을 볼 수 있다.




파드 접속하기

파드에 직접 들어가서 뭔가 확인하고 싶을 수도 있다.
그럴때는 exec 명령을 사용하면 되는데, 도커 컨테이너에 들어가는 것과 거의 같다.

kubectl exec -it 파드명 /bin/bash




파드 정보 조회하기

파드 자체에 대한 전체 정보를 보고 싶다면, descibe pods 명령을 사용한다.

kubectl describe pods 파드명

이런저런 전체적인 정보와, IP 같은 것도 확인할 수 있다.

파드 삭제

파드 삭제는 delete pods 명령으로 처리할 수 있다.

kubectl delete pods 파드명

근데 보통 프로덕션 레벨에서 시스템을 구축할 때는 Pod만 사용하는 일은 거의 없다.

보통 deployment나 statefulset, daemonset 같은 그룹 단위를 통해 재배포까지 통합해서 관리하곤 하는데, 그건 이후 포스트에서 하나씩 다뤄보겠다.
여기서는 Pod라는게 있고 대충 이렇게 쓴다는 것만 알고 넘어가면 된다.
