[AWS] EC2: ENI

  • 2024.07.09
  • ::
  • 2 min read
#AWS
[원본 링크]

ENI는 Elastic network interfaces의 축약으로, 네트워크 단위를 리소스에 바인딩해줄 수 있게 하는 단위다. 그냥 네트워크 인터페이스라고도 부른다.

디스크를 EBS 기반으로 관리하면서 EC2에 붙일 수 있는 것과 비슷하게, ENI를 통해서 EC2에 여러개의 네트워크, IP 주소 등을 할당할 수 있다.

https://crishantha.medium.com/handling-elastic-network-interface-s-enis-in-aws-part-01-9696fe6f6df0

ENI 자체는 만든다고 해서 비용이 발생하는 부분은 없다.
ENI에 public IP를 붙인다면 그 IP 개수에 대해서 IP 사용료를 낼 뿐이다.




EC2 만들어보기

ENI는 사실 안쓴 것 같아도 은근슬쩍 다 쓰고 있던 기능요소다.
그냥 EC2 인스턴스만 하나 만들어도, EBS가 자동으로 만들어지고 매핑되듯이, ENI 또한 자동으로 구성된다.

이런 식으로 eni가 자동으로 만들어지고 붙었다.


만들어진 ENI 자체는 EC2의 네트워크 인터페이스 탭에서 확인할 수 있다.




고정 IP 연결해보기

Elastic API를 사용하면 EC2에 고정 IP를 할당할 수 있다.

사실 인스턴스로 할당하더라도 ENI로 할당하게 되는 셈이다. 동작은 같다.




추가 ENI 만들기

네트워크 인터페이스는 EC2와 별개로도 얼마든지 직접 만들고 제어할 수 있다.

적당히 서브넷 깔고, 보안그룹 골라서 만들면 되는 것이다.


그럼 미사용 상태로 붕 뜬 채로 만들어진다.




ENI 연결/해제

EBS와 마찬가지로 ENI도 EC2에서 자유롭게 떼어내거나 붙일 수 있다.

지금은 Elastic IP를 붙이지 않아서 public IP는 하나인데, 저 ENI에도 Elastic IP를 붙이면 IP가 2개가 되는 것이다.




보안 그룹

사실 EC2에 달리는 보안그룹도 EC2 자체에 바로 연결되는게 아니라, ENI 단위에 연결되는 것이다.

그래서 네트워크 인터페이스를 보면 보안그룹이 연결되어있을 것이고, EC2에 ENI를 하나 추가한다면

Fargate의 경우

ECS에서 주로 사용하는 컨테이너 단위 Fargate는, EC2와 다른 수준의 리소스긴 하지만 대전제는 얼추 공유한다.

Fargate로 프로비저닝되는 컨테이너 하나하나는 위처럼 다 고유의 ENI를 할당받는다.
