[Elasticsearch] 별칭 (alias)

  • 2025.07.29
  • ::
  • 1 min read
#Elasticsearch
[원본 링크]

별칭은 Elasticsearch의 유용한 보조기능 중 하나다.
그냥 이름 붙이는 것을 넘어서, 일종의 로드밸런서 비슷한 역할도 하기 때문이다.

별칭이 붙은 인덱스는 별칭을 통해서도 동일한 동작을 취할 수 있다.

만드는 법은 간단하다.
_aliases API에 생성 요청을 하면 된다.

이건 market_item_v7_0_0라는 인덱스에 market_item이라는 별칭을 붙여준 것이다.


그러면 이렇게 alias 목록으로 조회가 되며


alias를 통해서도 인덱스에 대해서 검색이나 write 등을 처리할 수 있다.




다운타임 없는 alias 교체

alias는 서버가 가리키는 인덱스를 교체할 때도 용이하게 사용할 수 있다.

elasticsearch를 쓰다보면 하위호환이 보장되는 상태로 인덱스를 새로 만들때가 많다. 컬럼을 추가한다거나 해도 인덱스를 새로 따야 하기 때문이다.
그러면 매번 서버가 가리키는 인덱스를 바꿔서 배포를 해야하는데, alias를 쓰면 이걸 간단하게 해결할 수 있다.

alias 처리 API가 원자성을 보장하기 때문이다.
예를 들어, market_item가 가리키는 실제 인덱스를 market_item_v5.3.0에서 market_item_v6.0.0로 바꾸고 싶다면, 이렇게 할 수 있다.

{
  "actions": [
    {
      "remove": {
        "index": "market_item_v5.3.0",
        "alias": "market_item"
      }
    },
    {
      "add": {
        "index": "market_item_v6.0.0",
        "alias": "market_item"
      }
    }
  ]
}

그러면 다운타임 없이 순식간에 alias가 가리키는 인덱스를 교체할 수 있다.
