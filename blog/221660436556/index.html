[Rust] unsafe: 포인터

  • 2019.09.26
  • ::
  • 2 min read
#Rust
[원본 링크]

unsafe는 러스트에서 일반적으로 허용되지 않는, 플랫폼 의존적인 코드나 메모리에 직접 접근하는 코드 등을 작성할때 사용한다.

자주 쓸만한건 아니지만, 종종 필요한 때가 있다.

문서에서 정의하는 unsafe의 기능에 따르면, unsafe는 아래의 행동들을 할 수 있다.

1.생(raw:날것) 포인터를 역참조하는 것.

2.unsafe 함수나 메서드를 호출하는 것

3.mutable static 변수에 접근하거나 값을 수정하는 것

4.unsafe 트레잇을 구현하는 것


1번인 포인터부터 보자.
러스트에선 기본적으로 &로 참조값을 얻어오고, &T나 &mut T로 참조타입을 표시한다.

하지만 러스트의 엄격한 규칙들을 적용하지 않는 생포인터 또한 사용할 수 있는데, 이것들은 *const T나 *mut T의 형태로 타입을 표현한다.
참조값을 얻어오는건 이것도 &를 쓴다.

간단한 예시를 보자

포인터값의 생성은 그냥도 할 수 있지만, *로 역참조하는건 unsafe 블럭에서만 할 수 있다.

잘 돈다.

하지만 unsafe 블럭을 빼면 에러가 난다.

const 포인터는 여전히 상수성을 유지한다.

그런데 상수 포인터를 변수 포인터로 변환하는 짓은 가능하다.

변수 포인터에는 소유권 규칙도 없다. 한번에 여러개의 가변 포인터를 생성할 수도 있다.

잘 된다.


그렇다.
