인덱스가 뭘까?

알고리즘을 공부해봤다면 데이터를 다루는 데 있어서 가장 중요한 것이 '정렬'이라는 것을 알 것이다.
정렬되어있지 않은 값을 가져오는 것은 몹시 비효율적인 성능을 보이기 때문이다.

데이터베이스도 이와 완전히 같지는 않지만 비슷하다. 데이터베이스의 값들을 정렬시킨 상태로 관리해서 그때그때 빨리 골라올 수 있도록 하는 것이 인덱스다.

인덱스가 걸려있는 컬럼을 조건에 걸어서 값을 가져오면, '인덱스 스캔'이란 것을 유도할 수 있다.
인덱스 스캔이 항상 빠른 것은 아니지만, 큰 값 목록에서 극소수의 값만을 꺼내올 때는 이보다 효율적인 것이 없다.
자세한 것은 아래 포스트를 참고하길 바란다.
https://blog.naver.com/sssang97/222066082736

한번 테스트를 해보자.
pk 하나 걸려있는 간단한 테이블이다.

사실 pk 자체도 인덱스지만 없는걸로 생각하고 진행하겠다.

조촐하게 값을 10만개 정도만 넣었다.

num에는 현재 아무것도 없기 때문에, num으로 값을 찾아오면 느려터진 Seq Scan으로 가져온다.
일반 코드 알고리즘과 비교하면 정렬되지 않은 배열에서 값을 하나하나 비교하면서 찾는 것과 동등하다.



인덱스의 생성

이제 인덱스를 걸어보자.
인덱스 생성은 다음과 같은 구문으로 수행할 수 있다.

create index 인덱스명 on 테이블명(컬럼명...)

그럼 이제 index scan이 유도될 것이다.



인덱스 삭제

인덱스의 삭제는 drop index로 행할 수 있다.



인덱스 타입

인덱스에도 종류가 하나만 있는 것이 아니다.
postgres의 인덱스는 b-tree, hash, Gist, Gin 등이 지원되나, 특별한 이유가 없으면 그냥 btree를 쓰면 된다.
기본값도 btree다.

btree가 가장 안정적이고 유용하다.

해시도 우리가 아는 그 해시테이블이다.

굉장히 빠르고 좋을 것 같지만, btree와 다르게 범위 검색이 되지 않는다는 단점이 있다.
딱 = 조건인 것만 인덱스를 탄다. 복합인덱스도 안되고.

그리고 이외에 자잘한 기능적 결함도 많아서 postgres에서는 사용을 비권장하는 기능이다.

나머지 둘은 조건도 참 까다로운지, 그냥은 생성이 안된다.
이건 나중에 별도로 다뤄보도록 하겠다.

결론은 굳이 타입 주지 말고 그냥 btree만 쓰면 된다.
그래서 앞으로의 설명도 다 btree를 전제한 내용들이다.



복합 인덱스

인덱스는 한 컬럼에만 지정할 수 있는 게 아니라 2개 이상의 인덱스에도 지정할 수 있다.

그런데 이렇게 되면 신경쓸게 조금 더 생긴다. 컬럼을 여러개 둔다고 각 컬럼마다 인덱스가 생기는 건 아니기 때문이다.
이러면 순서가 중요해진다. 가장 왼쪽에 있는 저 num이 선행 인덱스가 돼서, 저걸로 먼저 조건을 타는게 더 빠르다! 왼쪽에서 오른쪽으로 조건을 타서 들어가는 구조라 그렇다.

일단 둘다 인덱스 스캔을 유도할 수는 있다.

선행 컬럼인 num은 실행시간이 0.028밀리초
후행컬럼인 text 검색은 실행시간이 2.497밀리초가 걸렸다.

그럼 순서를 바꾸면 어떻게 나올까?


이번엔 반대로
num이 2.951밀리초.
text가 0.050밀리초가 나왔다.

신경쓰자.



정렬

btree 인덱스의 또 큰 장점 중 하나가 정렬이다.
btree는 기본적으로 값들을 정렬해서 관리한다고 했다. 그래서 인덱스가 걸린 컬럼을 정렬 조건으로 걸면 인덱스를 이용해서 더 빠르게 정렬 처리를 할 수가 있다!

인덱스를 걸지 않은 상태에서 order by를 걸면

비용만 19000여에,
실행시간이 66 밀리초나 걸렸다.

인덱스를 걸고 다시 돌려보자.


이번에는 비용이 10000 정도에 실행시간은 19 밀리초 정도로 단축됐다.

이것도 잘 응용하면 큰 성능 향상을 기대할 수 있는 부분이다.



고유 인덱스

인덱스를 생성할때 create index 사이에 unique를 낑겨넣으면 고유 인덱스가 된다.

이러면 이거는 인덱스로서만 동작하는게 아니라, 이 컬럼 값은 항상 중복되지 않고 고유해야 한다는 제약사항이 되기도 한다. 자동으로 CONTRAINT를 추가해주는 것이다.


그럼 이제 중복된 값이 들어가면 오류를 낸다.

그렇다.
