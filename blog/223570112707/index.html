[Rust] assert

  • 2024.09.03
  • ::
  • 2 min read
#Rust
[원본 링크]

assert는 Rust의 유용한 표준 매크로 중 하나다.
다른 언어들의 assertion 기능들과 크게 다르지는 않다.

테스트를 짜거나, 무결성을 지속적으로 추적할 필요가 있을 때는 거의 필연적으로 사용된다.

그 사용법에 대해 간략히 정리해보겠다.




기본 assert

기본 assert는 true/false 값만을 받는다.

true면 통과하고


false면 패닉이 발생한다.


그리고, 추가 인자로 패닉 메세지를 커스텀할 수 있다.
이거도 format 기반이라서 메세지에 값을 이래저래 구겨넣을 수 있다.




assert_eq

assert_eq는 비교 전용 숏컷이다.

인자 2개 받고, 값이 다르면 패닉을 던진다.


이것도 마찬가지로 패닉 메세지 커스텀이 되고


단위테스트 짤때 가장 많이 사용되는 녀석이다.
이런 느낌으로 쓰면 된다.


반대 부정버전으로 assert_ne가 있긴 한데, 딱히 쓴적은 없는 것 같다. 쓸일이 별로 없다.




debug_assert

근데 단위테스트와 별개로, 코드 사이사이에도 무결성에 대한 검증을 끼워넣고 싶을 수도 있다.
그렇다고 또 assert를 떡칠하면, 이것도 런타임에 실행되는 것이다보니 성능 손실이 발생하는게 신경쓰인다.

그럴때 사용하는 것이 debug_assert다.

이런 식으로 함수 내에서 사전조건이나 사후조건, 혹은 중간중간에 해당 조건이 보장되어야 함을 코드로 명시할 수 있다. 사실 저 코드의 경우에는 음수가 들어가면 조건이 깨지긴 해서 올바른 예는 아니다.

아무튼 이건 assert와 동작은 거의 같지만, release 빌드에서는 코드에서 제거되기 때문에 손실이 별로 없다.
개발 중에만 체크가 된다.


그래서 만약 조건에 위배되는 코드가 있더라도, 개발 모드에서만 뻑나고, release 모드로 실행하면 잘 돈다.
