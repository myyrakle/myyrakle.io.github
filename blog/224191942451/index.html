[k8s] kdash

  • ::

#Kubernetes
[원본 링크]

kdash는 CLI 기반 k8s 리소스 시각화 도구다.
이 방면의 대표자인 k9s의 대체제 중 하나라고 할 수 있다.

다만, k9s와는 다르게 리소스의 UPDATE/DELETE나 셸 접속 같은 가변적인 제어는 불가능하다.
이건 Read-only한 시각화만 제공하며, 시각적 UX의 개선만을 목표로 한다.

Rust로 만들어졌다.




설치

플랫폼마다 설치 방법이 좀 다르다.
근데 패키지 상태가 다 영 좋지가 않아서, 가장 아래의 방법, bash를 통한 설치를 권장한다.

# MacOS 
brew tap kdash-rs/kdash
brew install kdash

# Arch Linux
yay -S kdash 

# Rust Cargo 
cargo install kdash

# bash (Linux, MacOS)
curl https://raw.githubusercontent.com/kdash-rs/kdash/main/deployment/getLatest.sh | bash
sudo mv ./kdash /usr/bin/kdash




기본 사용법

사용법은 간단하다.

프로그램을 실행하면 즉시 시각화 창이 활성화된다.


양쪽 방향키나 숫자 키를 누르면 누르면 리소스타입의 전환이 되고, 위아래 방향키를 누르면 리소스를 상하로 이동한다. 엔터를 치면 해당 리소스의 상세 정보로 진입한다.


Pod의 로그를 보려면 파드 상세페이지에서 엔터를 치면 된다.


n을 치면 네임스페이스 선택 모드로 전환된다.

그리고 위에서 언급했듯이, 이건 현 시점에서 조회 기능만을 제공한다.
Deployment의 rollout, pod delete, shell 접속 같은 실질적인 제어 기능은 없다고 보면 된다.

내가 볼때 괜찮은 부분도 있는데, k9s를 대체하기에는 부족한 점이 많다.
