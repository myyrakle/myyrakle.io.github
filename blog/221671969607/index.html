실습 환경은 MySQL이다.
저장된 데이터의 내역은 아래와 같다.

update는 기존에 존재하던 데이터를 수정하는 쿼리다.
아래의 구조를 가진다.

update 테이블명 set 속성명1=값1, 속성명2=값2, ... where 조건식

delete와 다르게 조건식은 필수다. 없으면 에러가 난다.

저기서 이름 boom을 will로 바꿔보자.

잘 바뀐 걸 볼 수 있다.

만약 저기서 1년이 지나서, 나이와 학년을 전부 증가시킬 필요가 있다면 어떻게 할까?
update문은 기본적으로 delete처럼 조건식을 생략할 순 없다.
그냥 이렇게 하면 된다.

true는 참을 의미하는 논리값이다. 저렇게 쓰면 전부 매칭시켜준다.
