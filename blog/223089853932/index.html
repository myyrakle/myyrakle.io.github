EKS는 Elastic Kube Service의 축약으로, AWS에서 제공하는 관리형 쿠버네티스 클러스터 서비스다.

구성 방법은 EC2, Fargate, 온프레미스 중 하나를 골라서 쓸 수 있다.




비용

https://aws.amazon.com/ko/eks/pricing/
비용은 대강 클러스터 비용을 기본으로 깔고, 추가로 비용을 사용한 만큼 청구하는 형태다.

클러스터 하나에 시간당 0.1 달러를 깐다. 한달에 72달러=10만원 정도를 내는 셈이다. GCP 등과 동일하다.

그 외에는 EC2로 구성하면 EC2 비용이, Fargate로 구성하면 Fargate 비용을 청구하는 형태다.

접속해보기: CloudShell

콘솔을 통해 클러스터를 만들었다면, root 계정으로 쿠버네티스 마스터 계정이 생성된다.
그럴때는 aws 콘솔에서 shell로 들어가서 접속하는게 편하다.
root로 로그인한 상태에서 들어가면 인증이 root로 구성되고, aws-cli나 kubectl이 다 깔려있기 때문이다.

다음과 같이 kubeconfig를 먹여주면, kubectl로 바로 들어갈 수 있다.

aws eks update-kubeconfig --name 클러스터명 --region 리전명

aws eks update-kubeconfig --name test-cluster --region ap-northeast-2




노드

노드 띄우는건 직접 해야한다... GKE는 자동으로 해줘서 좋았는데.

EC2나 Fargate로 노드를 구성해야 하는데, 이건 별도 포스트에 정리해보겠다.
