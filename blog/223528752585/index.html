Proxy Chain

  • 2024.07.29
  • ::
  • 2 min read
#보안
[원본 링크]

프록시체인은 주로 공격자 측에서 사용하는 IP 추적 회피 기법이다.
사실 말이 거창학게 프록시 체인이지, 프록시 몇중으로 돌려쓰면서 추적 안당하게끔 한다는 것이다.


https://medium.com/@turbo4gproxy/proxy-chaining-an-advanced-proxy-technique-explained-fd1c113557b5
기본적인 원리는 별게 없다. 그냥 실제 프록시 서버 아무거나 가져다가 뺑뺑이 돌리는 것이다.
이러면 요청 받는 서버 측에서는 원본 IP가 뭔지를 찾는게 거의 불가능하다. tor 기반의 브라우저들도 거의 이런 식이고.

작정하고 정부 수준의 기관이 프록시 서버 하나씩 털면서 쫓으면 털릴 수야 있긴 하지만, 시간이 걸린다.
공격자가 도주하기에는 충분한 시간이다.




proxychains CLI 설치

Kali 같은 잼민이들 OS에는 기본으로 깔려있는듯하지만, 일반 리눅스에서는 직접 깔아야 한다.
make, gcc가 필요하다.

git clone https://github.com/haad/proxychains
cd proxychains/
./configure
make
sudo make install

설치되면

proxychains4라는 이름으로 사용할 수 있다.




프록시 설정

먼저 설정파일에 들어가서 dynamic_chain으로 체인 사용법을 변경하겠다.

sudo vi /usr/local/etc/proxychains.conf

dynamic_chain

이게 좀 편하다. 프록시 리스트에서 그때그때 사용 가능한 프록시 가져다가 조합해서 쏘는 방식이다.

그리고 프록시 리스트를 가져다 쓰기 위해서 tor 네트워크를 설정하겠다.

sudo apt-get install tor -y
sudo service tor start
sudo systemctl status tor

그냥 깔고 활성화시키면 된다.



그럼 원래 진짜 내 IP가 나오던 것이


proxychains4 curl -m 20 https://.../

프록시의 IP가 찍히기 시작할 것이다.
그리고 여러번 네트워크를 타는거라서 당연히 속도는 굉장히 느리다.
