data 타입 선언은 구조체 같은 새 타입을 선언하는 용도로 쓸 수도 있지만, union 타입을 선언하는데 사용할 수도 있다. Rust를 경험해본 적이 있다면, enum의 기능을 그대로 가져간다고 이해하면 된다.




enum 선언

타입에 |를 쳐서 나열하는 것만으로도 기본적인 형태의 enum은 정의할 수 있다.
아래 예제는 색상을 표현하는 Color enum을 정의한 예다.

그럼 그냥 이렇게 상수에 enum 값을 할당해서 써먹을 수 있다.

패턴매칭에도 필드에 따른 분기를 사용할 수 있다.

근데 저렇게만 하면 뭔가 값 디버깅이 잘 안되니까, deriving을 통해서 문자열 변환 기능을 제공해보자.

저러면 이 enum에 대해서도 Show 타입클래스가 구체화된다.

정수형 Enum

아직까지는 Enum에 이름만 있고 어떠한 값이 부여되지 않았다. Red는 그냥 Red일 뿐이다.
하지만 Enum의 많은 usecase에서는 이걸 실질적인 정수값 등으로 변환해야 할 경우가 많다.

그런걸 원한다면 직접 Enum을 구현해도 되는데, deriving Enum을 쓰면 순서대로 정수값을 자동부여해준다.

이렇게. fromEnum으로 enum->정수형 변환을, toEnum으로 정수형->enum 변환을 수행할 수 있다.




타이핑된 enum 필드

위에서는 enum 필드들을 타입 없이 그냥 이름 나열만 했지만, 타입을 직접 부여해서 다형적인 형태의 데이터 구조를 선언할 수도 있다.
아래는 Int 값이거나 String 값을 가질 수 있는 enum을 정의하는 예시다.

값을 직접 전달하면서 생성할 수 있고, 패턴매칭을 통해 실제 내부 값을 꺼내와서 사용할 수 있다.




제너릭

data 타입 선언에도 타입 인자를 통해 제너릭 기능을 부여할 수 있다.
대표적으로, monad 타입인 Maybe 같은 것이 제너릭 data로 구현된 것이다.

이 경우 Maybe는 아무것도 없는 Nothing 상태이거나, 실제로 값이 있는 Just 상태여야 한다.
그리고 Just일 경우에는 구체화된 타입 파라미터 a의 값을 내부적으로 들고 있는다. 그게 다다.
