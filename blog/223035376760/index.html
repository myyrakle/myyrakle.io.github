[Linux] root 디렉터리

  • 2023.03.05
  • ::
  • 5 min read
#Linux
[원본 링크]

리눅스에서 루트 디렉터리로 이동하면 대략 저런 형태로 폴더들이 보일 것이다.

저것들은 대부분 특수한 의미를 갖고 있는 디렉터리라서, 함부로 건드리면 큰 부작용이 날 수도 있다.
대부분 파일 시스템 계층구조 표준(Filesystem Hierachy Standard:FHS)에 의해 정의된다.

하나씩 대강 살펴보도록 하자.




/bin, /sbin

기본적인 실행 프로그램들이 들어있는 디렉토리다.

sbin은 ipconfig 같은 약간 내부시스템용 프로그램이 들어있는 곳이다.

이거 이외에도 다른 서브디렉터리에도 실행파일을 위한 .../bin, .../sbin 디렉터리는 여러개가 있을 수 있다.
전체 목록은 PATH 환경변수를 까보면 확인할 수 있다.




/boot

리눅스 커널의 부트로더 관련한 물건들이 들어있는 디렉토리다.
직접 건드릴 일은 거의 없겠다.




/dev

처음 보면 develop의 준말인가 싶을 수도 있는데, device를 뜻하는 영역이다.

실제 이 디바이스에 대한 구성 정보들을 갖고있다.
하드디스크 장치파일(/dev/sda)이나 CD-ROM 장치파일(/dev/cdrom) 같은 것 말이다.

아마 여기서 /dev/null과 /dev/tty...들을 자주 보았을 것이다.

/dev/null은 쉽게 말하면 쓰레기통이다. 어떤 프로그램 실행에서 출력되는 로그를 씹고 버리고 싶을때 "... > /dev/null"의 형태로 보내버리곤 한다.

/dev/tty는 터미널에 대한 정보를 관리하는 부분이다.
그래서 텍스트를 보내면

이렇게 echo가 된다.




/etc

시스템의 자잘한 설정파일이 들어있는 곳이다.

슈퍼유저를 관리하는 파일도 여기에 있고

크론탭 설정파일도 여기에 들어있다.




/home

홈디렉토리다. 홈디렉토리 안에는 존재하는 계정들에 대한 개별 home 서브디렉터리들이 있다.

내 현재 계정명이 myyrakle이라서 디렉터리는 myyrakle 하나만 있는 상태다.

그 안에는 뭐, gui를 깔았다면 Windows처럼 사용자가 흔히 쓰는 폴더들이 기본으로 들어있을 것이다.

아니면 텅 비어있을 수도 있다.

보통 시스템의 소스코드같은거나 다운받은 무언가나, 이것저것 자잘한 것들은 여기에 넣어두는 경우가 많다.
막 써도 된다.




/lib, /lib32, /libx32, /lib64

커널 모듈과 동적라이브러리(.so) 같은게 들어있는 곳이다.

libx32는 x32용 라이브러리, lib32 32비트용, lib64는 64비트용이다. 이건 리눅스 배포판마다 조금씩 차이가 있을 수도 있다.

일례로 lib의 gcc 디렉터리를 파고들어가면 이런식으로 .so, .o 같은 라이브러리 구성요소들이 들어있다.




/media

DVD, CD-ROM, USB 등과 같은 탈부착이 가능한 장치들의 마운트포인트로 사용되는 디렉토리다.

지금 내 경우에는 VirtualBOX로 시뮬레이션을 하고 있어서, 그에 대한 구성파일만 들어있다.




/mnt

/media 디렉토리와 비슷한 용도다.
탈부착이 가능한 장치들에 대해서 일시적인 마운트포인트로 사용하는 디렉토리란다.

그래서 보통은 비어있다.




/proc

"가상파일시스템(Virtual File System:VFS)"라고 부르는데, 현재 메모리에 존재하는 모든 작업들이 파일형태로 존재한다.

여기에 들어있는 파일들은 다 디스크상에 실제 존재하는 게 아니고, 메모리상에 존재한다. 그냥 리눅스에서 디스크 파일인 것처럼 표시해줄 뿐이다.

그래서 함부로 건드리면 큰일나고, 굳이 건드릴 일도 잘 없을 것이다.




/sys

커널이 쓰는 하드웨어 정보들이 들어있다.

버스나 펌웨어나.. 그런 것들 말이다.

보통 직접 건드릴 일은 없다.




/root

root 계정의 사용자 전용 디렉터리다. root 권한으로만 들어올 수 있다.




/tmp

공용디렉터리다.

모든 사용자가 공동으로 사용할 수 있다.
그래서 mysql 의 설정파일(mysql.sock)같은게 들어가기도 한다.

중요한 정보들이 많이 들어가는만큼 보안에 유의할 부분이기도 하다...




/usr

일반사용자가 쓰는 범용 디렉터리다.

이를테면 사용자가 직접 설치한 실행프로그램 같은 걸 /usr/bin에 두기도 하고, 사용자가 직접 설치한 동적 라이브러리를 /usr/lib에 두기도 한다.




/var

임시적으로 저장되는 데이터들이 모여있는 곳이다.

예를 들면 시스템 수준의 로그는 /var/log에 저장된다.




/lost+found

root 계정만 들어갈 수 있다.

이건 사실 루트 디렉토리에만 존재하는 게 아니라 파일경로로마다 존재할 수 있는 디렉토리다.

fsck, e2fsck 같은 파일시스템 체크니 복구용 도구 실행후에 주로 생성이 되는데, 복구되지 않은 채로 블록(block)만 존재하는 파일, 그러니까 연결이 끊어진 inode 들이 숫자파일형태로 존재하는 곳이다.
그 숫자형태로 존재하는 파일들은 파일이름만 바꾸면 바로 복구될 수 있다.

그래서 용량 관리할때는 다 지우기도 하는데, 재복구 가능성을 염두에 둔다면 지우지 않는 편이 좋겠다.




/opt

나는 처음에 봤을때 optimize 뭐시기인가 싶었는데, optional의 준말이라고 한다.

이건 뭔가 추가적인 외부 프로그램을 설치할때 사용되는 경로다.
왜 이런게 따로 있는건지는 잘 모르겠다. 역사적인 이유가 있겠지...

내 경우에는 containerd 같은게 들어있다.




/srv

이건 뭔가 다른 시스템에게 제공할 파일을 넣어두는 곳이다.

따로 뭘 하지 않았다면 보통 비어있을 것이다.

ftp로 뿌려줄 파일을 보관할 때나, nginx로 외부에 정적 서빙할 이미지 같은걸 넣어준다거나 하는 용도로 쓰는 것 같다.
근데 뭐 굳이? 지금은 반쯤 버려진 느낌이다.




/run

원래 /var/run이었던게 루트로 독립한 물건이다. 원래부터 있던 건 아니고, 근래에 이렇게 생긴 모양이다.
런타임 임시 데이터 같은걸 보관한다고 한다.

직접 건드릴 일은 별로 없을 것이다.
