[MongoDB] TTL

  • ::

#MongoDB
[원본 링크]

MongoDB는 자체적으로 만료 삭제 처리를 위한 TTL(Time To Live) 기능을 제공한다.
캐시 기능을 구현하기도 좋고, 용량 확보를 위한 방법으로 도입하기에도 적합하다.

사용법은 그리 어렵지 않다.

먼저 도큐먼트에 ttl용 시간을 관리하기 위한 Date 타입 값이 존재해야 한다.
저걸 데이터의 "생성 시간"으로 간주한다.




TTL 인덱스 생성

ttl은 인덱스의 구조를 통해 제공된다,
expireAfterSeconds를 통해 몇 초 후에 삭제되게 할 지를 결정한다.

db.weather.createIndex( { "ttl": 1 }, { expireAfterSeconds: 60 } )

예를 들어 이렇게 입력하면 "ttl" 시간 기준으로 60초가 지나면 만료된 데이터로 판단하고 삭제한다.

그리고 데이터를 확인해보면

데이터가 삭제된다.

다만 MongoDB는 TTL 스케줄러를 1분 주기로만 돌리기 때문에, 삭제가 최대 1분까지 지연될 수 있음에 유의한다.
