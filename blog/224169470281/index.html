VPN: Tailscale

  • 2026.02.03
  • ::
  • 3 min read
#Proxy
[원본 링크]

Tailscale은 VPN 구축에 주로 사용되는 프록시 도구다.
VPN 구축 방면에서는 거의 절대적인 1인자의 위치를 차지하고 있다.

오픈소스는 아니라서 자체 구축은 안되고, 전용 클라우드 서비스를 통해서만 사용할 수도 있다.




사용 구조

다른 VPN 서비스들과 크게 다르진 않다.
TailScale 중계 서버가 있으면, 각 피어는 스스로를 중계 서버에 등록해서 Tailscale 전용 IP를 발급받는다.

그러면 그렇게 발급받은 IP를 통해서 서로서로 통신이 가능해지는 것이다.

저 상태에서 Tailscale 내부 IP를 통한 통신이 시도되면, 각 장비에 세팅된 Tailscale 에이전트가 그걸 가로채서 실제 피어와 연결해준다.

Tailscale은 UDP 홀펀칭을 통해 각 피어를 직통으로 연결해주기 때문에, 불필요한 전송 딜레이는 거의 존재하지 않는다고 봐도 된다. 물론 하드웨어 상태에 따라서 홀펀칭이 불가능할 수도 있는데, 그럴 경우에는 tailscail 중계기가 relay 역할을 해준다.




비용

https://tailscale.com/pricing

이 서비스의 가장 좋은 점은, 개인 용도의 플랜이 완전히 무료로 제공된다는 것이다.

팀 계정이 3개로 제한된다는 것이 단점이긴 한데, 기기는 100개까지 되니까 꽤 넉넉하다고 볼 수 있다.
트래픽과 관련된 비용은 부과되지 않는다.

상위 플랜들도 팀 계정의 한도가 늘어나는 형태로 확장된다.
Personal Plus는 월 고정 5달러라서 상당히 저렴하고, 6인이라서 소규모 팀에는 충분할 것이다.

그 이상의 제대로 된 프로덕션 사용에서는 인당 비용을 받는다.
Starter의 경우에는 1인당 6달러, 프리미엄은 1인당 18달러 같은 식이다.




시작해보기

클라우드 전용 서비스이기 때문에, 먼저 서비스 사이트에서 회원가입을 하고 대시보드에 진입해야 한다.
https://login.tailscale.com
가입 자체는 매우 간편하다. Github 계정만 있어도 바로 가입 가능하고, 카드 등록을 강제하지도 않는다.


잘 가입이 되면, 이런 식으로 대시보드가 뜰 것이다.

이제 이 계정을 통해 장비들을 한 곳에 묶어주면 된다.




에이전트 설치

전용 에이전트를 설치해야만 중계기에 피어로서 등록이 되고 상호 통신이 가능해진다.
Mac, Linux, Windows를 모두 지원하며, iOS와 Android까지도 지원한다.

Windows와 MacOS 인스톨러를 제공하니까, 그거 받아서 설치하면 된다.
https://tailscale.com/download/mac
https://tailscale.com/download/windows

리눅스의 경우에는 다음 스크립트를 통해 간단하게 설치할 수 있다.

curl -fsSL https://tailscale.com/install.sh | sh

설치가 잘 됐다면, up 명령을 통해 에이전트를 실행할 수 있다.

sudo tailscale up --hostname 기기명

그러면 전용 데몬인 tailscaled가 뜨면서 자동으로 네트워크 구성이 초기화된다.




활용

tailscale status 명령을 사용하면, 현재 중계기에 등록된 피어들의 전체 목록을 확인할 수 있다.

그럼 저기에 표시되는 내부 IP를 통해서 서로간의 통신을 수행할 수 있는 것이다.

ping만 날려봐도 직통으로 가고,

해당 IP로의 네트워크 접근은 전부 해당 피어로 전달된다고 보면 된다.


그리고 네트워크 설정을 끄고 싶다면 down 명령을 사용하면 된다.




SSH 사용

VPN을 사용하는 주된 이유 중 하나가 상호간의 자유로운 접근이다.
이를 위해 tailscale은 피어간의 ssh 통신을 직접 지원한다.

ssh 통신을 "받을" 서버의 경우에는, 에이전트를 등록할때 ssh 플래그를 넣어줘야 한다.

sudo tailscale up --ssh

그럼 다른 클라이언트 피어는 해당 서버 피어의 IP를 통해서 직접 SSH를 꽂고 들어갈 수 있다.




이슈: DNS 오류

기존 네트워크 구성에 따라서는, tailscale로 인해서 모든 네트워크 통신이 차단될 수도 있다.

tailscale은 MagicDNS라고 해서 자동으로 DNS 관련 설정을 덮어씌우는 뒷동작을 돌리기 때문이다.

그걸 해소하려면 그냥 dns 관련 플래그를 하나 끄면 된다. --accept-dns다.
