#Kubernetes
[원본 링크]

job은 일회성 작업을 돌리는데 최적화된 컨트롤러 단위다.
보통 인프라 초기 세팅에 필요한 작업만 지정하고, 운영중에는 잘 사용되지 않는다.

yaml은 보통 아래와 같이 구성된다.

apiVersion: batch/v1
kind: Job
metadata:
  name: bg-deployer
  labels: 
    app: bg-deployer
    version: "1"
spec:
  backoffLimit: 3
  template:
    spec: 
      containers:
      - name: bg-deployer
        image: myyrakle/bg-deployer
        env:
        - name: HEALTH_CHECK_TARGET_URL
          value: "https://naver.com"
      restartPolicy: Never

실행할 컨테이너 이미지를 정의하고, 재시도횟수(backoffLimit)를 정한다.

그리고 job을 띄우면

job이 준비중에 들어갈 것이고

pod를 띄워서 실행하게 될 것이다.

pod가 성공적으로 작업을 끝내고 내려가면 job은 complete 상태가 되고, 더이상의 기능을 하지 않는다.
