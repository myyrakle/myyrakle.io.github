nullable
null은 아무것도 가리키지 않음을 의미하는 일종의 상수 포인터다.
그래서 레퍼런스 변수에는 사용에 아무런 문제가 없다.

근데 값 타입을 다룰 때 문제가 발생한다.
int 변수를 보자. 사용자에게서 정수값을 받아서 처리를 해야하는 상황이고, 사용자에게서 잘못된 값을 받았거나, 받았지 않았을 경우를 어떻게 표현해야 할까? 0? 0 자체도 값인 상황이 있을 수 있다.
C는 null을 그냥 0으로 표현해버리지마는.. C#은 이 부분에서는 엄격하게 처리를 한다.
그러면 값 타입에서 아무것도 없다는걸 어떻게 나타내야 할까
이럴 때 값 타입에 null을 표현하기 위해 생긴게 nullable이다.

원래 제네릭방식으로
Nullable num;
이러면 null을 담을 수 있는 int 타입이 완성된다. 근데 썩 보기 좋지는 않다.

여기에서 설탕을 치면
int? num;
이렇게 표현된다.

As
as는 캐스팅 연산자다. 이건 그냥 소괄호 캐스팅과는 다르게 참조타입으로의 변환만 허용을 한다.
그리고 실패하면 에러가 아니라 null을 반환한다.
이 null의 반환가능성 때문에 참조타입으로만 변환이 되는 것이다.
용법은 대강 이렇다.

변환할값 as 변환할타입




Is
is는 원래 그냥 타입변환 가능여부만 체크하는 연산자였다.

변수 is 타입;

해서 변환이 되는 상황이면 true를 반환하고 끝이었는데,

버전 7.0부터 as의 기능을 어느정도 포함하게 되었다.

변수 is 타입 바인딩변수;
해서 is의 결과가 true면 저기다 바인딩을 해준다.
