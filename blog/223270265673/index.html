[AWS] health status 구독하기

  • 2023.11.21
  • ::
  • 1 min read
#AWS
[원본 링크]

AWS는 여타 인프라 서비스들과 마찬가지로 헬스체크용 콘솔을 제공한다.

아래 링크에 들어가면 글로벌 장애나 업데이트, 혹은 내 계정에서 발생한 특수한 이벤트를 확인할 수 있다.
https://health.aws.amazon.com/health/status

가끔 들어가서 뭔일 없나 확인해보면 된다.

근데 이걸 매번 들어가서 보는 것도 일이고, 제대로 활용을 하려면 알림이 오게 하는 편이 좋다.

AWS는 이거에 대한 뭔가 구독체계가 썩 잘 되어있지는 않아서 직접 구현을 해줘야 한다.
GCP가 RSS 피드 형태로 구독데이터를 제공하는 것과 상반된다.

aws 상태 알림은 aws 자체의 이벤트로 수신이 가능하다.
그래서 eventbridge를 통해 제어할 수 있다.




Slack 구독 시스템 구현

나는 슬랙 웹훅을 사용해서 직접 슬랙 통합을 구현했다.
Lambda 하나 파고, 적당히 다음과 같이 구현한다.

const axios = require('axios');

// 슬랙 웹훅 링크
const SLACK_URL = `https://hooks.slack.com/services/...`;

exports.handler = async (event) => {
    console.log(event);
    const detail = event?.detail;

    const status = detail?.statusCode;
    const region = event?.region;
    const service = detail?.service
    const description = detail?.eventDescription?.[0]?.latestDescription;

    const attachments = [
        {
            "color": "#ff0000",
            "pretext": "AWS 상태 알림",
            "title": "Status 콘솔 이동",
            "title_link": "https://health.aws.amazon.com/health/status",
            "text": "AWS 상태 알림",    
            "fields": [
                {
                    "title": "상태",
                    "value": status,
                    "short": false
                },
                {
                    "title": "서비스",
                    "value": service,
                    "short": false
                },
                {
                    "title": "리전",
                    "value": region,
                    "short": false
                },
                {
                    "title": "내용",
                    "value": description,
                    "short": false
                },
            ],
        }
    ];

   await axios.post(SLACK_URL, {
		username: 'Amazon Cloud Service Health Updates',
		attachments,
		channel: '채널ID',
	});

    // TODO implement
    const response = {
        statusCode: 200,
        body: JSON.stringify('Hello from Lambda!'),
    };
    return response;
};

받아서 적당히 파싱하고 슬랙에 쏴주는 로직이다.

그리고 eventbridge 규칙으로 트리거를 걸어준다.

소스는 aws.health면 된다.

그리고 방금 만든 람다를 걸어주면 끝이다.

이게 이벤트 발행자가 내가 아니라 aws라서 실제 테스트는 어렵다.
만약 위의 코드를 가져다 만들었으면 이렇게 뜰 것이다.

나는 잘 쓰고있다.
