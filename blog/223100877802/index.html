[k8s] k9s

  • 2023.05.13
  • ::
  • 2 min read
#Kubernetes
[원본 링크]

k9s는 kubectl의 기나긴 명령어 세트들을 좀 간소화해서 사용할 수 있게 해주는 모니터링 도구다.

kubectl은 명령어 자체가 몹시 긴 편인데다가, description->desc 같은 축약도 없는게 많아서 쌩으로 치다보면 몹시 피곤해진다...
쿠버네티스 팀 자체가 이런것에 대해서 매우 무관심하고 불친절한 편인 것 같았다. desc 추가해달라는 이슈가 있었는데, 그냥 바로 close해버리더라.

아무튼

이런걸 좀더 편하게 사용할 수 있게 해주는 것이다.
다른 Lens 등의 유사 도구들과 비교해서 터미널에서 가볍게 사용하기 좋다는 장점이 있다.




설치

패키지매니저에 k9s나 derailed/k9s/k9s가 들어있다면 그걸로 깔면 되고, 없다면 go로 까는게 편하다.

go install github.com/derailed/k9s@latest

설치가 잘 됐다면, k9s cli 실행이 가능해질 것이다.




사용법

k9s만을 치면, 바로 전용 모니터링 모드로 진입한다.

k9s

그럼 클러스터의 상태와 기본 Pod들의 목록을 쭉 뽑아볼 수 있다.
저기서 방향키로 이동도 가능한데

거기서 'd' 키를 누르면 describe 모드로 파드 정의를 바로 볼 수 있다.

나가려면 "ECS"를 누르면 된다.

혹은, "Enter"를 치면 로그 및 상태 모니터링, Pod 내의 컨테이너 리스트가 뜬다.




필터링

명령어들이 대부분 vi 베이스로 디자인되어있다.
만약 deployment들만 골라보고 싶다면, :deploy를 치고 엔터를 치면 된다.

그외에도 다양한 기능들이 많다.
스페이스바를 누르면 특정 리소스를 선택할 수 있고, 컨트롤+D를 누르면 바로 삭제를 할 수도 있다.

e를 누르면 바로 파드 등의 yaml을 수정할 수도 있고

파드 페이지에서 s를 누르면 컨테이너에 바로 shell 접속도 된다.
