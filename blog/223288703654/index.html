[Linux] io_uring

  • ::

#Linux
[원본 링크]

io_uring은 리눅스 커널 5.1부터 도입된 비교적 최신의 비동기 I/O 모델이다. epoll보다도 한 단계 진화한 형태라고 할 수 있다.

관련 포스트
https://m.blog.naver.com/sssang97/223049662481



기본 구조

I/O는 그 자체로 엄청난 system call을 발생시키고, 보통은 system call 자체가 성능 병목이 되곤 한다.

그래서 io_uring의 기조는 중간 레이어를 추가해서 system call 횟수를 최소한으로 줄이는 것에 있다.

해서 꽤나 복잡한 구조를 가진다.

우선, io_uring은 사용자 영역과 커널 영역 사이에 2개의 ring buffer를 둔다.

하나는 사용자 영역에서 발생한 I/O를 커널로 쏘는 Submission Queue.
하나는 커널에서 처리 완료된 I/O의 결과를 다시 사용자 영역으로 반환해주는 Completion Queue다.

io_uring은 저 2개의 버퍼 큐를 통해 여러개의 I/O 요청/응답을 버퍼 단위로 묶어서 싸그리 처리하는 방식을 취한다.

이 덕분에 system call을 최소화하고 유저<>커널 사이의 context switch를 줄일 수 있다.
게다가 큐를 통한 이벤트 처리 등의 여러가지 응용에도 유리한 부분이 있다.

아래는 epoll과 io_uring을 비교한 벤치마크 사례 중 하나다.

낮을수록 빠르다. 처리량이 많아질수록 io_uring의 성능이 더 돋보이는 것을 볼 수 있다.




현재

아직 완성 단계는 아니고, 프로덕션에서 사용되는 사례가 많지는 않은 것 같다.

특히 보안 취약점이 가끔 나와서 아직 전면적으로 쓰기에는 무리가 있어보이더라.
