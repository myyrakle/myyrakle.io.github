[k8s] 대시보드: kubernetes-dashboard

  • 2024.06.13
  • ::
  • 2 min read
#Kubernetes
[원본 링크]

k8s은 기본적으로 CLI 기반의 제어 도구를 지원하지만, 아무래도 하다보면 불편하거나 시간이 걸릴 때가 많다.
좀 빨리빨리 파악하고 분석하고 싶다면 GUI 대시보드를 사용하는 것이 당연히 빠르고, 이에 대한 도구 지원도 많은 편이다.

대시보드 도구는 종류가 몇개 있는 편인데, 여기서는 k8s 그룹에서 추천하는 kubernetes-dashboard라는 걸로 해보겠다.




설치

설치 방법은 간단하다. 그냥 apply만 먹여주면 된다.

kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.6.1/aio/deploy/recommended.yaml

이거 최신 버전은 helm으로만 설치되는데, 문서에 쓰인 것과 동작이 일치하지도 않고 쓰레기같아서 버렸다.

아무튼

이렇게 설치가 되고 서비스가 뜨면 잘 된 것이다.




사용해보기

이걸 가장 간편하게 접속하는 방법은 프록시를 열고, 로컬 머신에서 들어가는 것이다.
보안상의 이유로 외부 접속은 권장하지 않고, 하기도 어렵게 만들어놨다.

http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/#/login

이렇게 들어가보면

로그인하라고 뜰텐데, 계정 토큰을 넣으면 된다.

토큰을 만드는 방법은 별도 포스트에서 다뤘다.
https://blog.naver.com/sssang97/223476409094

그러면 이런식으로 뜰텐데

권한을 주지 않았다면 뜨는 것도 없고 잔소리만 시끄럽게 해댈 것이다.


apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: test-admin-rolebinding
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: testaccount
  namespace: default

전체적으로 다 보고 싶다면 이렇게 어드민을 먹여주는 것이 편하다.


그러면 이렇게 리소스별로 가시화해서 볼 수 있고, 로그나 리소스 상세정보도 쉽게 조회할 수 있다.
