#nats
[원본 링크]

nats는 Go로 작성된 in-memory 분산 메시징 브로커 시스템이다.
일단은 그냥 큐라고 생각하면 된다.

pub/sub 기반의 메세징 기능을 제공한다.
그리고 JetStream이라는 확장 기능을 사용하면 Kafka와 동일하게 on-disk, FIFO로도 사용이 가능하다. 이건 추후 다뤄보겠다.

기본적으로는 경량 시스템이라 가볍게도 쓸 수 있지만, 클러스터링을 통해 처리량을 높일 수 있다.
라즈베리파이에도 올라갈 수 있는 초경량 서비스부터, 대형 분산 시스템까지 가능한, 다재다능함을 추구한다.




도커로 시작해보기

docker를 통해서 nats 서버를 실행해보도록 하겠다.
이미지를 다운받고, 네트워크 설정하고, 실행시킨다.

docker pull nats
docker network create nats
docker run --name nats --network nats --rm -p 4222:4222 -p 8222:8222 nats --http_port 8222

4222, 8222, 6222 포트를 사용한다.
4222는 클라이언트가 사용할 외부 노출 포트고, 6222는 클러스터에 사용하는 포트, 8222는 내부 관리용 HTTP 포트다.

Pub/Sub 사용해보기 (nats-cli)

nats가 이런저런 잡다한 기능을 많이 갖고 있긴 한데, 근간이 되는 기능은 Pub/Sub이다.
Redis Pub/Sub과 비스무리하다고 보면 된다.

컨슈머 프로그램이 특정 주제를 subcribe하고, 프로듀서 프로그램이 이벤트를 같은 주제에 publish하면, 컨슈머가 그 데이터를 받아서 처리할 수 있는 것이다.

먼저 Cli 도구를 설치한다. go를 이용해서 까는게 편하다.

go install github.com/nats-io/natscli/nats@latest

이걸로 nats 서버에 접속해서 pub/sub을 한번 시도해보자.

구독(sub)은 다음과 같이 할 수 있다.
아래는 "hello"라는 주제를 구독하는 간단한 명령이다.

nats sub --server nats://localhost:4222 hello

그리고 게시(pub)는 다음과 같이 할 수 있다.
다음 명령은 "hello"라는 주제에 "i dont know who you are" 메세지를 날린다.

nats pub --server nats://localhost:4222 hello "i dont know who you are"

그러면 구독자 측에서도 그걸 바로 받아볼 수 있다.

조금 유의할 점은, 구독자가 여럿이면 여러명의 구독자가 동시에 같은 메세지를 받을 수 있다는 것과

subscriber가 죽어있는 상태에서 publish만 하면, 데이터가 증발한다는 것 정도다.
다 Redis Pub/Sub도 동일하게 갖고있는 특징이다.
