[Java] 클래스

  • 2019.11.27
  • ::
  • 2 min read
#Java
[원본 링크]

자바의 (거의) 모든 것은 클래스, 객체이다.
int, boolean 같은 기본타입을 빼면 전부 클래스라고 봐도 무방하다.

클래스로 정의된 타입은 new 키워드를 통해 생성되며, 동적할당되어 가비지 컬렉터에 의해 관리된다.

클래스는 class 키워드로 선언할 수 있다.

class 클래스명
{
**    필드들...**
**    메서드들...**
}

좌표값 x와 y를 표현하는 클래스를 구현해보자.
대강 이렇게 짜볼 수 있겠다.

잘 돌아간다.

저기서 public Point라는 놈이 생성자다.
new ..로 객체 생성을 시도하면 저게 호출되고, 객체의 필드를 초기화하는 것이다.


접근제어자
그리고 public은 접근제어자로, 외부에서 사용을 할 수 있게 할것인가, 말 것인가를 제어한다.
public은 가능. private는 불가능. protected는 상속자만 가능이다.

보통 외부에 공개할 인터페이스 메소드를 public으로 두고,
필드엔 전부 private를 것이 바람직하다.
protected는 특정 패턴을 구현할때 빼고는 잘 안쓴다.

여튼 저걸 다시 바람직하게 재구성한다면, 이렇게 할 수 있겠다.
필드를 숨기고, 별도의 출력 메서드를 정의한다.

사용자 측의 코드가 훨씬 깔끔해진 것을 볼 수 있다.


정적필드, 정적메소드
필드가 메소드 앞에 static을 붙이면 정적요소가 된다.
이렇게 된 녀석들은 따로 객체를 생성하지 않아도 접근이 가능하다. 프로그램 로딩 시에 초기화가 다 되기 때문이다...

그리고 정적멤버는 객체(변수)의 이름이 아니라 클래스명으로 접근한다. 클래스에 하나씩만 존재하기 때문이다.

변수명으로 접근해도 동작하긴 하는데, 절대 좋은 방법은 아니다...


this 키워드
클래스의 동적 메소드 안에서만 사용할 수 있는 키워드다. 정적메소드에선 안된다.

이건 객체 자기자신을 가리킨다.
이걸 사용하면 객체 자신의 멤버를 명확하게 표현할 수 있다.

this.x = x; 처럼 쓰면 변수명이 같더라도 명확하게 구분의 된다.

또, 객체 자신을 반환하거나 어딘가로 전달할 때도 사용할 수 있다.

또, 생성자에서 다른 생성자를 호출할 수도 있다.
