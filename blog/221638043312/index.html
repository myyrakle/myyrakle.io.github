자바스크립트는 기본적으로 함수를 통해 모듈의 분리와 재사용을 달성한다.(ES6 이전까지)

함수의 형태는 아래와 같다.
동적 언어이기 때문에 타입을 표시하지 않는다.

function** 함수명(인자...)**
{ ... }

아래 코드는 인자를 받지도, 반환하지도 않는 간단한 함수를 구현한 것이다.

아래는 두 수를 더해서 반환하는 함수 add를 구현한 것이다. 별거없다.


함수의 객체성
자바스크립트는 거의 모든 것이 객체이다.
이게 무슨 말이냐 하면 함수를 변수에 저장해서 변수처럼 다룰 수가 있다는 것이다.

대충 이런식으로 말이다.


익명함수
근데 자바스크립트는 이름이 없는 함수도 지원을 한다. 기존의 함수 선언에서 함수명만 빼면 된다.

아래는 boom이라는 변수에 익명함수를 할당한 것이다.

이렇게만 보면 기존의 함수선언과 별 차이가 없어보이지만... 익명함수의 진가는 함수를 인자로 넘길 때 발한다.

아래는 함수를 인자로 받아서 단순 호출만 하는 고차함수 caller를 정의하고, 익명함수로 콜백을 넘기는 코드다.

이걸 기존의 선언법으로 해결한다면, 함수를 따로 선언하고 그 함수명을 기입하는 등의 혼잡함이 발생할 수 있다...


람다식(ES6)
화살표 함수(arrow function)라고도 부른다.
익명함수가 더 간결하고 아름다워진 형태다.
(인자...) => { ... }

위의 형태를 가지지만, 소괄호는 인자가 하나일 경우에 뺄 수 있고, 중괄호도 단일표현식인 경우 뺄 수 있다.

위의 코드에 람다식을 적용하면 이렇게 된다.

아주 간결하다.
