[Elasticsearch] 복합 쿼리 (Boolean)

  • 2021.07.16
  • ::
  • 2 min read
#Elasticsearch
[원본 링크]

https://blog.naver.com/sssang97/222431994969
지난번 포스트에서는 단일 조건으로 검색을 하는 방법을 다뤄봤었다.
그리고 당연한거지만, 그것만으로는 충분한 검색을 구현할 수는 없다.




bool 절

여러가지 검색조건을 한번에 묶어주려면 AND나 OR 같은 논리식이 필요한데,
그런걸 묶어주는게 바로 bool 절이다.
bool에는 다음과 같은 구문들이 들어갈 수 있다.
must, should, must_not, filter다.

POST products2/_search
{
  "query": {
    "bool": {
      "must": [], 
      "should": [],
      "must_not": [],
      "filter": []
    }
  }
}

하나씩 한번 살펴보겠다.




must 절

must는 우리가 알던 AND 절과 거의 동등한 기능이다.
must 안에 들어있는 조건들이 전부 일치하는 경우에만 점수를 부여해서 결과에 노출해준다.

이렇게 하면 shop_id가 1이고 brand_id가 1인 항목만 노출된다.




should 절

이건 OR 절이라고 할 수 있다.
안에 있는 조건들 중에 하나만 부합해도 점수가 부여돼서 노출된다.
그리고 여러개의 조건이 일치하면 그만큼 더 많은 점수를 받고 더 위에 노출된다.

이렇게 했을 경우 shop_id과 brand_id 둘중 하나만 일치해도 다 노출되지만, 둘다 일치하는게 제일 앞에 나오는 것을 볼 수 있다.




must_not 절

이건 must를 그대로 반전해놓은 기능이다.
안에 있는 조건들이 전부 거짓일 경우에만 점수를 부여한다.

shop_id와 brand_id 둘다 안맞는 "시니컬한 불닭볶음면"이 노출되는 것을 볼 수 있다.




filter 절

위에서 다룬 조건절들은 전부 해당 조건에 부합할 경우 '점수'를 부여하는 시스템이다.

점수를 부여하고 점수가 높은 순서대로 보여주는 것 뿐인데, 점수와 별개로 그냥 후보에서 아예 걸러버리고 싶을 수도 있다.

필터를 사용하면 이걸 가능하게 할 수 있다. 이건 점수를 따지지 않고 조건에 부합하는 것만 전부 필터링해버린다.

그래서 다음과 같이 must 등을 쓴다면 점수를 저렇게 부여한 다음에 보여주지만

필터를 쓴다면 점수를 주지 않은 채로 걸러버린다.

이건 특히나 검색에 추가조건을 주렁주렁 매달 것이 많을 때 유용하다.

검색 텍스트로 조건을 거는건 당연히 유연하게 처리를 할 필요가 있지만,
막 5000원 이하, 500원 이상. 이런 식으로 추가조건을 거는 것을 점수로만 처리하면 결과가 조잡스러워질 수 있기 때문이다.

그런건 필터를 쓰는게 깔끔하다.
