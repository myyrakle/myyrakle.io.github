[Go] go cli 정리

  • 2023.05.26
  • ::
  • 1 min read
#Go
[원본 링크]

따로 정리한적이 없는 것 같아서 정리해본다.

자주 쓰이는 기본적인 것만 넣어놨다.




go mod init

nodejs을 써봤다면, npm init과 비슷하다고 보면 된다.
프로젝트를 생성하고 가장 먼저 사용해야할 명령이다.

이걸 통해 모듈 config 파일인 go.mod를 생성한다.




go run

소스코드를 바로 실행하는 명령이다.




go build

소스코드를 바이너리로 빌드하는 명령이다.




go get

go에서 서드파티 모듈을 다운받는 첫번째 방법이다.
소스코드를 다운받고, 종속성 관리파일인 go.mod와 go.sum을 자동으로 조정한다.




go mod tidy

이것도 모듈 설치하는 명령인데, go get과 선후관계가 다르다.
코드에서 설치되지 않은 모듈이 있다면, 그걸 다운받고 go.mod와 go.sum에 반영한다.




go install

이건 내 프로젝트에 모듈을 설치하는게 아니라, 이미 완성된 CLI 실행파일을 설치할때 사용한다.

그냥 Go로 작성된 CLI tool들을 설치할때 쓴다.

이걸로 설치된 파일은 GOBIN 환경변수에 지정된 경로에 깔린다.
