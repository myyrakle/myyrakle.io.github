[AWS] S3 - Glacier DEEP ARCHIVE

  • 2026.01.22
  • ::
  • 2 min read
#AWS
[원본 링크]

Glacier DEEP ARCHIVE은 S3의 가장 저렴한 백업용 스토리지 타입이다.
비용이 매우 싸지만, 조회하는데만 며칠이 걸리는 진짜 아카이브용 클래스다.




비용

매우 저렴한 저장 비용이 유일하면서도 최대의 강점이다.

서울 기준으로 하면, 저장 요금이
Standard는 GB당 USD 0.025
Glacier DEEP ARCHIVE는 GB당 USD 0.002다.

5TB를 들고 있다면 기본 125달러 정도가 나가지만, DEEP ARCHIVE를 쓰면 10달러 정도가 나오는 것이다.
게다가 동일 리전 내에서의 S3 OBJECT COPY는 데이터 전송이 무료이므로, 대량 백업이 필요한 경우에는 비용적으로 상당히 좋은 선택이 될 수 있다.

대신 읽기 비용은 다소 비싸다.
비용은 개수와 크기 2가지로 다 뜯어간다.
표준 검색의 경우에는 1000개당 USD 0.1086고, 데이터 크기 기준으로는 GB당 USD 0.022가 발생한다.
대량 검색의 경우에는 1000개당 USD 0.0275, 데이터 크기는 GB당 USD 0.005이다.

https://aws.amazon.com/ko/s3/pricing/




읽기 제한

세상만사가 그렇듯 공짜로 주어지는 것은 없다.
Deep Archive로 옮기거나 올리는 것은 매우 쉽다. 그러면 콘솔에서도 다음과 같이 표시되는데

이 상태에서는 즉시 조회를 하거나 복구-copy를 할 수가 없다.
접근하려면 매우 오래 걸리는 복원을 트리거해야 한다.




파일 복원

파일 복원과 데이터 전송에는 당연히 추가 비용이 발생한다.

이런 식으로 말이다.
표준 검색과 대량 검색은 각각 최대 12시간, 48시간 안에 복원이 완료되며, 상기한대로 서로 비용도 다르다.
급한게 아니라면 대량검색 걸어놓고 느긋하게 받는 편이 낫다.


복원이 완료되면 이렇게 조회 가능한 상태가 되며,


파일을 다운받거나 copy해서 복원할 수 있다.
