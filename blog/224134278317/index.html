[PostgreSQL] Range 타입

  • 2026.01.04
  • ::
  • 3 min read
#PostgreSQL
[원본 링크]

Range는 PostgreSQL에서 꽤 유용한 보조 기능 중 하나다.

말 그대로 (A, B) 한 쌍의 범위 값을 제공하는데, 이를 통해 꽤 편리하게 시간 범위나 기타 숫자 범위 값에 대한 로직 처리를 구현할 수 있다.




Range의 종류

범위 타입은 기본적으로 숫자와 날짜 타입들에 대해서만 지원된다.
총 6가지다.

-- 숫자 범위
select int4range(100, 210);
select int8range(100, 210);
select numrange(100, 210);

-- 날짜 범위
select daterange(date '2025-11-01', date '2025-11-10');
select tsrange('2026-01-01 23:02:50.635806+09', '2026-01-04 23:02:50.635806+09');
select tstzrange('2026-01-01 23:02:50.635806+09', '2026-01-04 23:02:50.635806+09');

숫자 Range 타입들은 각각 순서대로 int4, int8, numeric 타입들에 대응된다. 그걸 2개씩 붙였을 뿐이다.

날짜 타입들은 순서대로 date, timestamp, timestamptz 타입에 대응된다.




공통 특성 & 제약

Range 타입들은 범위 값으로서 갖춰야 하는 최소한의 보장을 제공한다.
무조건 왼쪽 값이 오른쪽보다 작아야 한다는 것이다.

그 제약을 위반하면 즉시 오류가 발생한다.

Range는 (시작, 끝) 값으로 구성되는데, 시작 값은 Range에 포함되지만, 끝 값은 Range에 포함되지 않는다.
그래서 (10, 100)의 진짜 표현 값은 10~99다.

만약 양쪽 값이 같다면, empty 값으로서 특별하게 표현된다.

이건 사실상 null에 대응되는 상태라고 보면 된다.
이 상태의 Range 값에 대해 연산을 하면 유효하지 않은 결과를 방출할 확률이 높다.




Range 연산

Range 타입은 범위 처리를 위한 기본적인 연산자들을 제공한다.
그 기본적은 목록은 다음과 같다.

@> 연산자는 Range에 단일 스칼라 값이 포함되는지를 확인한다. 포함된다면 true를 반환한다.

&& 연산자는 2개의 Range가 서로 겹치는지를 확인한다. 겹친다면 true를 반환한다.


  • 연산자는 2개 Range에 대해서 교집합을 만든다.

+은 2개 Range에 대해 합집합을 만든다.

이건 겹치는게 없으면 실패한다.

-|- 연산자는 2개의 Range가 서로 인접해있는지를 확인한다.
아래의 경우에는 한쪽의 시작값이 10이고, 나머지 한쪽의 끝값도 10이므로 인접한다. 따라서 true가 반환된다.

isempty 함수는 Range 값이 비어있는지(양쪽이 같은지)를 확인한다.

upper와 lower는 각각 오른쪽 값과 왼쪽 값을 반환하는 함수다.

여기서 유의할 점은, range가 empty 상태일 경우 upper/lower가 null을 반환하는 것이다.


, <, >=, <= 같은 비교 연산도 제공한다. 의미론은 특별할건 없다.

기본적인 연산은 이 정도고, 전체 목록은 아래 문서에서 확인할 수 있다.
https://www.postgresql.org/docs/9.3/functions-range.html




Gist 인덱스

Range 값에 대해서 동등 비교만 한다면 기본 B-Tree 인덱스로 충분하지만, 사실 그럴거면 Range를 쓸 이유도 없었을 것이다. 범위 값에 동등 비교를 할 일이 얼마나 되겠나?

아무튼 Range 타입에 대한 Range 연산은 단순한 스칼라 비교 연산이 아니기 때문에, B-Tree 인덱스로는 인덱싱이 불가능하다. Gist라는 특수한 인덱스 타입을 사용해야만 한다.

한번 테스트를 해보자.

create table test_range
(
    id int8 PRIMARY KEY, 
    range_value int8range NOT NULL
);

INSERT INTO test_range(id, range_value)
VALUES
(1, int8range(10, 100)),
(2, int8range(20, 120)),
(3, int8range(10, 60)),
(4, int8range(20, 30)),
(5, int8range(40, 50)),
(6, int8range(50, 70)),
(7, int8range(50, 60)),
(8, int8range(50, 55)),
(9, int8range(10, 40)),
(10, int8range(10, 60));

INSERT INTO test_range(id, range_value)
SELECT t.n, int8range(t.n, t.n + 100)
FROM
(
    SELECT generate_series(11, 100000) as n
) as t;

적당히 값을 집어넣고

기본 인덱스를 추가해봤다.

CREATE INDEX idx_normal_range on test_range(range_value);

그럼, 기본적인 동등 비교(=)에 대해서는 인덱스를 탈 것이다.

하지만 @>, && 같은 범위 연산은 인덱스를 타지 못한다.

무시하는 것을 볼 수 있다.

인덱스 타입만 Gist로 설정해서 만들면, 그때부터는 인덱스를 탄다.

CREATE INDEX idx_gist_range ON test_range USING GIST (range_value);

포함 연산도 인덱스를 잘 타고

겹침 연산이나 인접 연산 등도 인덱스를 잘 탄다.

그렇다.
