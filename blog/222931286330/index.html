fuzzing은 난수 기반의 무작위 테스트 기법이다.
그냥 랜덤값을 자동으로 쑤셔넣어서 잘못된 엣지 케이스가 있나 검증해보는 것이다.

Go 1.18부터 기본기능으로 사용 가능하다.

가령, 이런 단순한 덧셈 함수가 있다고 해보겠다.

의도적으로 lhs가 10일때만 잘못된 값을 계산하는 로직이 존재한다.
실제로 이런 말도안되는 코드를 짤 리는 없지만, 어떤 형태로든지 나타날 수 있는 상황이다.

그래서 사람들은 이런걸 최소화하기 위해서 단위테스트를 작성하곤 한다.

효과는 있지만, 단점 또한 극명하다.
결국 노가다로 케이스 하나하나를 다 치거나, 예상되는 엣지케이스를 찝어서 다 만들어줘야 하는데, 사람이 그걸 다 체크하는건 불가능에 가깝거나 힘들기 때문이다.





Fuzzing 테스트하기

이제 다시 해보자. 기존 테스트용 소스파일에 다음과 같은 형태로 코드를 추가한다.
함수의 이름은 Fuzz로 시작해야 하고, 파라미터 타입도 F여야 한다.
그리고 Fuzz 함수를 호출해서, 거기 안에서 진짜 테스트케이스를 작성하면 된다. 2번째 매개변수부터는 대충 추가하면 그 타입에 맞는 값을 알아서 넣어준다.

func FuzzTest2(f *testing.F) {
	f.Fuzz(func(t *testing.T, lhs int, rhs int) {
		a := Add(lhs, rhs)
		b := Add(rhs, lhs)

		if a != b {
			t.Errorf("lhs: %d, rhs: %d", lhs, rhs)
		}
	})

}

덧셈에 사용할 양변 값을 다 정수로 받아와서 돌리게 했다.
그리고 교환법칙이 성립하는지를 검증하게 했다. 이건 입력값을 예측할 수 없기 때문에, 이런식으로 좀 우회적 검증을 해야 한다.

그리고 테스트케이스를 돌릴때 fuzz 옵션을 주기만 하면 된다.

go test -fuzz=Fuzz

그럼 알아서 난수 돌려보다가 오류 케이스를 찾아줄 것이다.
딱 맞춰서 던져줬다.
