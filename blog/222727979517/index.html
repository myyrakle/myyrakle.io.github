#AWS
[원본 링크]

DocumentDB는 MongoDB 시스템을 제공하는 데이터베이스 서비스다.




비용

https://aws.amazon.com/ko/documentdb/pricing/
그렇게 싸지는 않다.
다음 4가지 항목으로 비용을 지불한다.

인스턴스 스펙, IO, 저장공간, 추가 저장공간(S3)이다.

인스턴스 요금은, 서울의 경우에는 다음과 같다. 표시된 금액은 시간당이다.

가장 작은 t3.medium만 한달동안 올려놓으면 85달러가 나오는 것이다.




클러스터 생성

생성 버튼을 누르면 다음과 같은 창이 뜰 것이다.

사이즈를 신중하게 고르고, 인스턴스 개수를 정한다.
인스턴스 개수만큼 비용이 곱해지니 잘 생각하길 바란다.

이런식으로 생성되면 잘 된 것이다.

그럼 이렇게 접속 가능한 엔드포인트 등이 뜰 것이다.





접속해보기 (EC2)

DocumentDB는 Elasticache 등의 다른 시스템들과 마찬가지로, 성능을 위해 public access를 제공하지 않는다.
다시 말해, 같은 VPC 안에서만 접속이 가능하다.
이걸 회피하려면 SSH 터널링을 하거나, 뭐 이래저래 둘러가는 방법을 사용해야 한다.
나는 여기서는 EC2를 통해 접속해보겠다.

EC2를 만들때, 저 보안그룹을 같게 맞춰줘야 한다.

저 보안그룹으로 들어가서, 인바운드 규칙에 mongodb의 포트가 없다면 추가해준다.


그리고 인스턴스에 들어가면, 다음과 같이 mongo DB용 클라이언트를 설치한다.
먼저 레포지토리 깔아주고

echo -e "[mongodb-org-4.0] \nname=MongoDB Repository\nbaseurl=https://repo.mongodb.org/yum/amazon/2013.03/mongodb-org/4.0/x86_64/\ngpgcheck=1 \nenabled=1 \ngpgkey=https://www.mongodb.org/static/pgp/server-4.0.asc" | sudo tee /etc/yum.repos.d/mongodb-org-4.0.repo

그걸로 설치한다.

sudo yum install -y mongodb-org-shell

잘 설치됐다면, 클러스터 상세에 들어가면 접속하는법을 잘 알려준다.
따라치면 된다.

그럼 이렇게 접속이 될 것이다.

그럼 이제 MongoDB와 동일하게 사용할 수 있을 것이다.

문법 등과 관련된 자세한건 MongoDB 관련 포스트를 참고하길 바란다.

https://blog.naver.com/sssang97/221746854470




메리트?

나는 이걸 쓸만한 상황을 잘 못찾겠다.
보통 NoSQL을 사용하려는 이유는 관리가 쉬우면서 단순 읽기/쓰기 작업이 RDB보다 훨씬 저렴하고 빠르다는 것이다.

근데 MongoDB는 그렇게 저렴한 편은 아닌 것 같다. 이런저런 관리포인트도 꽤 많고, 인스턴스가 늘면 그만큼 비용도 훅훅 늘어난다.

나는 많은 쓰기가 필요한 상황에서는 거의 DynamoDB를 더 쓸 것 같다.
더 확실하게 완전관리형이고, 더 저렴하고 가볍기 때문이다.
