podman이 Docker와 가장 다른 부분이 바로 pod 단위를 지원한다는 것이다.

pod의 서브커맨드로 사용 명령어를 제공한다.




Pod 생성하기

create 명령을 사용하면 바로 파드를 생성할 수 있다.

sudo podman pod create -p 80:80 --name testpod

알다시피 pod는 그 자체로는 기능을 하지 않고, 내부에서 구동된 컨테이너들을 단일 머신에 격리하는 역할만을 한다.

그래서 이 상태에서는 inspect를 찍어봐도

"포트포워딩"을 위해서 만들어진 내부 컨테이너만 하나 있을뿐. 동작을 하지는 않는다.





Pod에 컨테이너 집어넣기

실질적인 컨테이너를 띄우려면 기존의 run 명령을 사용해야 한다.
--pod 플래그에 기존 pod 이름만 넣어주면 그 pod에다가 컨테이너를 끼워서 실행시켜준다.

sudo podman run -idt -p 80:80 --pod testpod docker.io/nginx

이렇게 컨테이너도 추가된다.
마지막 것이 nginx로 띄운 저거다.
