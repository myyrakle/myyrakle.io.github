[Git] stash

  • 2022.01.19
  • ::
  • 3 min read
#Git
[원본 링크]

여럿이서 Git을 사용해 협업을 하다보면, 자신이 진행하던 작업을 킵해놓고 다른 작업을 해야할 경우가 빈번하다.

stash는 그럴 때 사용하는 "임시 저장"용 기능이라고 할 수 있다.

그냥 전부 다 commit으로 저장해도 되지만, 그러면 커밋 히스토리가 지저분해져서 rebase를 해줘야 한다.

stash는 commit과는 좀 다른 기전으로 동작한다.




stash에 넣고 빼기

stash는 스택구조로 커밋되지 않은 소스 변경사항을 전부 저장한다.
넣는건 git stash, 다시 빼는건 git stash pop 명령으로 처리할 수 있다.

만약 이렇게 커밋이 되어있는 상태에서

코드를 추가로 작성하고

git stash를 한다면,

추가로 작성한 항목이 임시저장되고, 이전의 상태로 돌아갈 것이다.

그리고 다시 저장된 항목을 불러오려면 stash pop을 사용하면 된다.

그럼 다시 잘 돌아올 것이다.

만약 원격 레포지토리(github 등)를 통해서 외부 커밋을 받아 작업하게 되더라도,

stash로 덜된 작업을 꿍쳐놓고, 받아서 작업하면 될 뿐이다.

당연히 stash도 겹치면 conflict가 나는데, 이런건 알아서 잘 merge해주면 된다.




메세지 남기기

stash는 그냥 써도 되긴 하지만, 커밋처럼 -m 플래그로 메세지를 남길 수도 있다.




stash 상태확인

stash의 저장상태를 확인하고 싶다면 stash list 명령을 사용하면 된다.
그럼 현재 저장된 stash의 브랜치와 메세지 등을 보여줄 것이다.

하나 이상의 stash가 쌓여있다면, 역순으로 stash가 노출될 것이다. 스택이니 말이다.




중간에서 빼오기

stash는 기본적으로 스택 형태로 동작하지만, 원한다면 무시할 수도 있다.

stash list에서 확인되는 왼쪽의 식별자를 문자열로 감싸서 인자로 주기만 하면 된다.




stash 날리기

쌓여있는 stash 저장내역을 날리고싶다면, stash clear를 사용하면 된다.




apply & drop

정말이지 간단했던 stash pop은 사실, apply와 drop 2가지 명령이 결합된 동작이다.

apply는 stash에서 변경내역을 가져오는 명령이고, drop은 stash 내역에서 그걸 삭제하는 명령이다.
stash pop은 0 인덱스에 대해 이 2가지를 자동으로 실행해주는 것이라 보면 된다.

apply는 다음과 같이 사용할 수 있다.
아무 인자를 주지 않으면 pop처럼 최근내역을 가져오되, 삭제는 하지 않는다.

drop도 사용조건은 동일하다. 인자를 주지 않는다면 가장 최근내역을 삭제한다.




브랜치로 분리하기

작업해놨던 stash 내역 자체를 브랜치로 분리해서 관리하고 싶을 수도 있다.
stash branch는 그럴 때 사용하는 명령이다.

stash branch <브랜치명>의 형태로 명령어를 사용하면,

가장 최근의 stash를 pop 한다음에, 브랜치를 만들고 그 브랜치로 자동 전환까지 해준다.


그렇다.
