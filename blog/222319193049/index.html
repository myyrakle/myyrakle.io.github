[AWS] Dynamo DB: CRUD 기본구조

  • ::
  • 2 min read

[원본 링크]

이전 포스트
https://blog.naver.com/sssang97/222319050576

이전 포스트에서는 DynamoDB 객체로 전체 테이블 목록을 조회했지만,
실제 테이블에 접근해서 수정이나 조회를 행하려면 DocumentClient 객체의 메서드를 이용해야 한다.
이를 통해 간단한 삽입, 수정, 삭제, 조회를 처리하는 방법을 다뤄보겠다.

테스트 런타임은 Node.js다.




삽입하기(put)

단일 삽입은 put 메서드를 통해 처리할 수 있다.
TableName은 삽입할 테이블명, Item은 삽입할 데이터 오브젝트다.
다른건 막 넣어도 괜찮은데, 기본 키로 지정한 값은 꼭 정의한 것에 맞춰서 필수적으로 넣어줘야 한다.

이렇게 작성하고 실행하면

잘 들어가있을 것이다.




수정하기(put)

수정은 put을 통해서도 처리할 수 있다.
기본키값이 같다면, 삽입이 아니라 덮어써버린다.




수정하기(update)

put으로도 대충 수정처리가 가능하긴 하지만, 편리하진 않다.
수정이라기보단 덮어씌우는 것에 가까운지라...

수정을 하려면 update를 사용하는 것이 좋겠다.
Key로 조회할 조건값을 넣고, UpdateExpress에 set 컬럼=값, 컬럼2=값2, ...의 형태로 수정할 값을 넣어준다.

그리고 위에서는 name이 키워드라서, ExpressionAttributeNames로 이스케이프 처리를 했고, 값은 ExpressAttributeValues로 주입되게 했다.

그럼

잘 바뀌어있을 것이다.




삭제하기 (delete)

삭제는 delete 메서드를 사용한다.
update와 동일하게 Key값을 주기만 하면 된다.

그럼 잘 지워질 것이다.




대량 삽입 or 삭제 (batchwrite)

그런데 위의 put, delete 기능들은 전부 개별 요소만을 처리하는 기능이다.
한번에 여러 연산을 돌리고 싶다면 이걸로 여러개의 연산을 묶어버리면 된다.

테이블마다 연산목록을 배열로 받아서 한번에 처리해주는 기능이다.

여기서는 PutRequest만 넣었는데, DeleteRequest도 사용할 수 있다.

두둥



단일 개체 조회 (get)

딱 뭐 하나만 골라서 조회하고 싶다면 get 메서드를 사용한다.

테이블명과 조회할 키값만 잘 넣어주면 된다.




복수 개체 조회 (scan)

여러개의 항목들을 한번에 긁어오고자 한다면 scan 메서드를 사용한다.
별다른 조건을 걸지 않는다면 모든 데이터를 읽어올 것이다.

필터 조건을 걸고자 한다면, FilterExpression 속성으로 정의하면 된다.

이외에도 다양한 기능 옵션들이 존재하지만, 일단은 여기까지만 하겠다.
