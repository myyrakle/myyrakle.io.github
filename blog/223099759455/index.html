[k8s] minikube

#Kubernetes
[원본 링크]

minikube는 간단하게 쿠버네티스 클러스터를 띄워서 사용할 수 있게 해주는 학습자용 도구다.
프로덕션용은 아니다.



웹 브라우저에서

따로 설치를 하지 않고도 브라우저 단에서 사용을 해볼 수도 있다.
대신 엔드포인트가 제대로 안열려서 접속 테스트같은건 어려울 수 있다.

그냥 아래 사이트에 들어가서
https://kubernetes.io/ko/docs/tutorials/hello-minikube/

버튼만 누르면 바로 클러스터가 생성된다.

이렇게

그럼 kubectl을 통해서 바로 사용이 가능해질 것이다.




직접 설치하기 (windows)

windows, linux, mac 전부 지원된다.
https://minikube.sigs.k8s.io/docs/start/

windows의 경우에는 파워쉘로 열어서 실행하고

New-Item -Path 'c:\' -Name 'minikube' -ItemType Directory -Force
Invoke-WebRequest -OutFile 'c:\minikube\minikube.exe' -Uri 'https://github.com/kubernetes/minikube/releases/latest/download/minikube-windows-amd64.exe' -UseBasicParsing

설치가 다 되면 이런 경로에 생성이 될 것이다.

환경변수에 추가하고 start 명령으로 클러스터를 시작하면 된다.





파드 생성해보기

아래와 같이 yaml 파일을 정의한다.
아래는 nginx를 띄워주는 간단한 구성의 파일이다.

apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
  - name: container-name
    image: nginx

그리고 apply 명령으로 띄우면 올라간다.

파드에 대한 더 자세한 사용법과 정보는 별도 포스트를 참고한다.
https://blog.naver.com/sssang97/223013640738





deployment 만들어보기

이번에는 deployment를 구성해서 앞에 로드밸런서를 달아보겠다.
우선 다음과 같이 deployment를 하나 띄운다.

kubectl create deployment hello-node --image=registry.k8s.io/echoserver:1.4

deployment에 대한 더 자세한 정보를 별도 포스트를 참고한다.
https://blog.naver.com/sssang97/223027848144




서비스 생성

근데 저렇게 그냥 리소스를 만들기만 해서는 우리가 접속을 할 수가 없다. 기본적으로는 전부 내부 IP로만 개방되기 때문인데, 이걸 열어주려면 서비스를 만들어야 한다.
자세한건 아래 포스트를 참고한다.
https://blog.naver.com/sssang97/223084158042

방금 만든 deployment에 대해 로드밸런서 서비스를 만들어주면 개방시킬 수 있다.

kubectl expose deployment hello-node --type=LoadBalancer --port=8080

로컬 환경이라면 External-ip에 접속 가능한 내부아이피가 뜰 것이다.
