객체의 타입

타입스크립트에서도 자바스크립트 형식의 오브젝트를 그대로 사용할 수는 있다.
타입정의가 덧대어질 뿐이지.

그리고 위의 객체값은 사실 아래의 타입정의를 가진다.
{
** 속성명: 속성타입;**
** ....**
}
의 형태다.




동적인 값 주입

그리고 TS에서는 이제 객체에도 명확한 타입이 생기기 때문에, JS처럼 오브젝트에 필드를 동적으로 주입하는 것은 거의 불가능하다.

만약 동적으로 주입을 하고 싶다면,
오브젝트에 필드를 유동적으로 넣을 필요가 있다면 해당 필드를 선택값으로 표시해두거나

아예 그냥 any로 써야할 것이다.




클래스와 인터페이스

타입스크립트에서 오브젝트의 타입은 클래스와 큰 차이가 없다.

일례로, 같은 컬럼 타입들을 가지는 오브젝트 타입과 클래스라면, 별다른 문제 없이 호환이 된다.

그래서 생성자를 사용하지 않고 클래스 타입에 일치하는 오브젝트값을 할당하는 것도 허용이 된다.

인터페이스도 마찬가지로 오브젝트 타입과 호환된다.

굳이 값 할당을 필수적으로 하지 않아도 되기 때문에 오히려 더 오브젝트 타입과 가까우며, 그냥 오브젝트 타입 자체를 정의하기 위해서 인터페이스를 사용하는 경우도 잦다.
