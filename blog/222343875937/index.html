[AWS] SQS

  • 2021.05.09
  • ::
  • 3 min read
#AWS
[원본 링크]

SQS는 Single Queue Service의 약자로, 말 그대로 큐를 제공하는 서비스다.

큐는 여러가지 방식으로 사용될 수 있지만, 가장 많이 사용되는 형태 중 하나는 작업에 대한 대기열이다.
작업이 막 미친듯이 들어올 경우에, 일단 대기열에 쌓아두고 천천히 처리하게 하는 것이다. 물론 꼭 당장 실시간으로 처리할 필요가 없을 경우에만 말이다.

SQS에는 다음의 2가지 대분류가 존재한다.
표준은 그냥 막 던지고 마는 것이고, FIFO는 보낸 순서대로 받을 수 있는 것이다.
표준은 빠르고 저렴한 대신 한 메세지를 여러번 받을 수 있다거나 하는 문제가 있을 수 있고,
FIFO는 느리고 비싼 대신 편리하고 안정적이란 특징이 있다.

필요에 따라 잘 선택해서 사용하면 된다.
우선 기본옵션대로 표준으로 만들어보겠다.

이름만 지정하고 생성을 했다.

그럼 아래와 같이 큐 페이지가 뜨기 시작할 것이다.

보통 이렇게 사용하지는 않지만, 콘솔에서도 직접 메세지를 쏘고 받아볼 수 있다.
한번 쏴보자.

저기로 들어가면

송수신 창이 뜨는데, 여기서 메세지를 직접 쏴볼 수가 있다.

그냥 쏴도 되지만

속성값을 추가로 넣을 수도 있다.

그럼 수신 창에서

메시지 폴링 버튼을 누르면

현재까지 저장된 메세지를 전부 읽어올 것이다.

메세지는 각각 고유한 UUID 형식의 ID를 가지고 있다.




With Lambda (전송)

메세지를 쌓아두기만 하면 뭐하겠는가?
쌓아둔걸 처리도 해야지.

꼭 그래야하는건 아니지만 이런 처리는 람다로 스케줄링를 걸어서 처리하는 편이다.
10분마다 큐를 읽고 작업할 게 있으면 처리해서 지우고... 그런 식으로 말이다.

일단 큐 메세지를 쌓을 함수를 하나 만들어보겠다.
테스트 런타임은 일단 간단하게 노드로 하겠다.

그리고 만들어진 람다 구성에 들어가서 실행역할->

SQS 접근권한을 연결해준다.

그러면 이제 이 람다에서는 큐에 읽기나 쓰기를 할 수가 있다.
아 그리고, 접속할 큐의 URL을 긁어오자.
이게 있어야 접속을 할 수 있다.

메세지 발송 코드는 대략 아래와 같이 작성하면 된다.
메세지 내용과 URL을 객체로 전달한다.

const AWS = require("aws-sdk");
const sqs = new AWS.SQS();

exports.handler = async (event) => {
    // 보낼 메세지
    const message = {
        productId: 1, 
        command: 'DELETE',
    };

    await sqs.sendMessage({
        //문자열만 들어갈 수 있음
        MessageBody: JSON.stringify(message),
        // 접속할 큐 주소
        QueueUrl: '주소', 
    }).promise();

    const response = {
        statusCode: 200,
        body: JSON.stringify('Hello from Lambda!'),
    };
    return response;
};

이외에도 속성도 보낼 수 있고 다양한 옵션이 있는데, 그건 문서를 참조하길 바란다.
https://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/SQS.html#sendMessage-property

이렇게 해서 실행한다면

실제로도 잘 쌓여있을 것이다.

이제 이걸 읽어보자.




With Lambda (수신)

이번엔 큐를 읽어서 처리할 함수를 하나 만들어보겠다.

자잘한 구성 설정은 이전과 같다.

메세지를 읽는건 receiveMessage 함수를 사용하면 된다.
큐 주소만 필수항목이고 다른건 옵션이다.

const AWS = require("aws-sdk");
const sqs = new AWS.SQS();

exports.handler = async (event) => {
    const result = await sqs.receiveMessage({
        // 접속할 큐 주소
        QueueUrl: '주소', 
    }).promise();

    console.log('메세지');
    console.log(result);

    const response = {
        statusCode: 200,
        body: JSON.stringify('Hello from Lambda!'),
    };
    return response;
};

이외에 다양한 옵션이 존재하는데, 그건 문서를 참조하길 바란다.
https://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/SQS.html#receiveMessage-property

저렇게 해서 실행하면 대기열에 있는 메세지를 하나씩 읽어올 것이다.
한번에 가져올 수 있는 메세지의 최대 개수는 10개인데, 기본 옵션값은 1이다.

읽은 메세지는 약간의 시간이 지나면 자동으로 사라질 것이다.
근데 여기서 표준 SQS는 한 메세지를 2번 이상 읽을 수 있다는 난점이 존재한다.

그래서 이 상황에서 테스트를 여러번 돌리면 한 메세지가 2번 이상 보이는 현상이 나타날 것이다.

이런게 싫다면 비용을 감수하고 FIFO 큐를 쓰거나 수동 처리를 해야한다.
