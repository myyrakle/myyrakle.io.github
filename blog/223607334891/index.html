rupring v0.10.0 배포

  • 2024.10.05
  • ::
  • 2 min read
#라이브러리
[원본 링크]

https://github.com/myyrakle/rupring/releases/tag/v0.10.0

몇가지 개선과 기능 추가가 포함된다.




application.properties 탐색경로 확장

원래 rupring 서버가 application.properties를 찾을때 실행 경로에서만 찾았었다.

하지만 이제는 rupring 서버 실행파일이 존재하는 위치에서도 찾는다.
docker로 말거나 할때도 그냥 실행파일과 application.properties를 같은 위치에 두고 말아서 쓰면 되는 것이다.




응답 압축 기능 추가

이제 application.properties에 관련 옵션을 활성화하면, 응답을 전송할 때 압축해서 전송한다.

현재 지원되는 알고리즘은 gzip과 deflate다.

min-response-size는 기본값이 2k다. 애매하게 작은 데이터는 압축하면 오히려 커질 수도 있고, 효율적이지가 않아서 적당히 크게 주는게 좋다.

옵션 문서화는 여기다가 해놨다.
https://docs.rs/rupring/latest/rupring/application_properties/index.html#special-options




Graceful shutdown 기능 추가

shutdown 관련 필드가 2개 추가되었다.

shutdown 옵션을 graceful로 해두면 Ctrl+C 등으로 강제종료를 시도하더라도, 서버를 바로 종료시키지 않는다.
새로 온 요청은 거부하는 상태로 기존 요청들을 기다리다가, 기존 요청들이 전부 종료되면 그제서야 서버를 종료한다.

timeout-per-shutdown-phase는 그런 상황에서도 너무 오래 지나면 강제로 종료를 시킬 수 있는 시간값이다.
저렇게 넣어두면 기다릴 만큼은 기다리다가 3초 뒤에는 강제로 종료시키는 것이다.

유의할 점이 하나 있는데, keep-alive가 켜진 상태에서는 connection이 바로 종료되지 않는다는 것이다.
그래서 요청이 다 처리되더라도 바로 종료되지 않고 딜레이가 좀 생길 수 있다. 이건 차차 개선해보려한다.

옵션 문서화는 여기다가 해놨다.
https://docs.rs/rupring/latest/rupring/application_properties/index.html#special-options
