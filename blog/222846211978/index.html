git blame은 특정 소스코드에서 코드를 누가, 언제 마지막으로 수정했는지 알려주는 기능이다.

기본 사용형태는 다음과 같다.

git blame 파일명

만약 다음과 같은 파일이 있고, 저기서 최종 수정정보를 알고싶다면

이런식으로 사용이 가능하겠다.

그럼 소스코드와 함께, 매 라인을 누가 언제 수정했는지 보여준다.

그런데 소스코드가 너무 크다면, 저걸 제대로 보기 힘들 수가 있다.
그럴때는 L 플래그로 보고 싶은 라인 범위만 지정해주면, 딱 거기만 뿌려준다.
4-5 라인만 봐도 된다면 이렇게 사용하면 된다.




다른 방법

사실, 꼭 커맨드만 사용해야 하는 환경이 아니라면, 더 편리한 대안이 있는 편이다.
에디터, 개발도구의 보조기능을 사용하는 것이다.

예를 들어 Visual Studio Code에서는 확장도구 GitLens를 깔고 커서를 대기만 하면 해당 정보를 보여준다.
