  • ::

#Java
[원본 링크]

자바에는 json 관련 기능이 따로 내장되어있지 않다. 그래서 모듈을 따로 땡겨써야하는데, 가장 쓰기 적당한게 구글에서 구현한 Gson이다.

라이센스도 아파치라서 자유롭게 사용할 수 있다.

maven으로 받아도 되지만 나는 그냥 jar을 다운받아서 썼다.
여기서 .jar 파일을 받으면 된다. 다른 버전 받고싶으면 디렉터리 이동해서 찾으면 되고.
https://repo1.maven.org/maven2/com/google/code/gson/gson/2.8.6/

그리고 IDE에서 이걸 등록해준다. 이클립스는 영 불편해서 난 인텔리제이를 쓴다.
Project Structure를 들어간다.

Java를 눌러서 들어가고, 아까 받은 .jar을 잘 찾아넣는다.

OK

다 Ok하면 이제 등록이 된 것이다.

문서는 이걸 보면 된다.
https://www.javadoc.io/doc/com.google.code.gson/gson/2.8.6/index.html
객체를 생성하려면, JsonObject라는 클래스를 사용하면 된다.

아래는 객체를 생성하고 요소값을 몇개 추가하는 간단한 예제다.

objectd의 add 메서드는 키와 값을 인자로 받는데, 값은 추상클래스 JsonElement의 서브타입이야만 한다.
서브타입으로는 프리미티브값을 나타내는 JsonPrimitive,
널값을 나타내는 JsonNull, 배열을 나타내는 JsonArray, 또 객체를 나타내는 JsonObject가 있다.

근데 프리미티브 값은 넣어야겠고, JsonPrimitive를 귀찮게 쓰고싶지 않을때 적절한 것이 addProperty다.
더 간결하다.

이미 구성되어있는 json 텍스트는 JsonParser의 정적메서드들로 파싱해서 JsonObject 등으로 만들 수 있다.

parse 메서드들은 공통 상위타입인 JsonElement를 반환하므로, getAs...등으로 캐스팅을 해줘야한다.

이외에도 입력 스트림을 전부 받아서 객체로 만들어주는 파싱 메서드도 있다.

근데 버전에 따라서 전부 parse로 합쳐서 오버로딩하기도하고, 저렇게 parseReader, parseString 등으로 나누기도 하는 것 같다.
