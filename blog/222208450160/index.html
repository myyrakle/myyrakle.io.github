전부 읽어오기

값 목록을 전부 읽어오는 데는 모델의 findAll 메서드를 사용한다.

대충 이렇게 그냥 가져오면


위와 같이 객체의 배열로 반환을 해준다.

뭔가 그냥 보면 지저분한데

그냥 배열처럼 쓸 수 있고, 각 컬럼값 접근은 그냥 오브젝트처럼 할 수 있다.



limit와 offset

페이지네이션 등을 처리하다 보면, 오프셋과 리미트 등을 활용해서 페이지를 쪼갤 일이 아주 잦다.
시퀄라이즈로 이를 처리하려면, 옵션값에 리미트와 오프셋을 넣어주면 된다.

이러면 0인덱스부터 2개만 가져오는 구문이 된다.




orderby로 정렬하기

정렬을 하려면 order 속성에 정렬조건들을 나열하면 된다.
여러개의 조건을 받을 수 있고, 각 조건은 [컬럼명, asc/desc]의 형태를 가진다.



하나만 가져오기: findOne, findPk

이 함수들은 findAll과 다르게 딱 하나만 가져올 수 있다.
대신, 가져올게 없다면 null을 반환한다.

일단 findPk는 자동으로 생성되는 id값을 기반으로 값을 골라온다.

findOne은 좀더 일반적인 기능의 메서드다.
보통 where 옵션으로 가져올 값의 조건을 지정해서 가져온다.

위 조건은 id가 2인 값을 가져온다는 것이다.

근데 저렇게 쓸수 있는건 딱 값만 넣어도 될 때 충분한 것이고,
이런저런 다양한 조건을 부여하려면 Op 연산자 값들을 사용해야 한다.

위 코드는 in (...)에 대응되는 쿼리로 값을 가져온다.


이 where 옵션은 모든 find 함수에 공통된 기능이다.



컬럼 골라오기

따로 설정을 하지 않는다면 find 메서드들은 모델에 정의된 컬럼들만을 전부 가져온다.
그러고 싶지 않다면 attributes 속성으로 가져올 컬럼들의 이름을 써주면 된다.
