[원본 링크]

Amazon Q Developer는 개발자용 LLM 종합 패키지다.

코파일럿, cursor, cline 등이 제공하는 코드 생성/수정 보조기능들을 제공한다고 보면 된다.




비용

기본적으로는 월 구독료 기반으로 제공된다.
원래 아마존이었으면 API 호출당으로 제공했을텐데, 경쟁의 힘이 이렇게 강하다.


추가비용이 발생하는 부분이 있긴 하지만 예외적인 기능이다. 일반적으로 저런걸 쓸 일은 잘 없을 것이다.
무료로 프리뷰 버전을 사용할 수도 있다.




문제: with 파트너사

파트너사를 통해서 AWS 계정을 관리할 경우에는 매우 심각한 문제가 있다.
이건 사람마다 계정을 발급해서 쓰고, 계정별로 요금이 부과되는 동작 방식을 가지고 있는데, 파트너사를 끼고 있으면 그게 안된다.

파트너사를 사용하게 되면 파트너사 측에서 전용 Organization을 만들고 root의 핵심 권한을 다 빼앗아가기 때문이다.


이런 식으로 말이다.

그래서 문의도 해봤는데

결국 안된다는 소리였다.


아무튼 시작해보자.
권한 다 있는 개인계정이라 가정하고 진행한다.


계정 생성

시작하기를 누르고


계정을 적당히 만들어준다.

만약 IAM Identity Center 세팅이 되어있지 않다면 거기로 이동해서 만들어야 한다.

만들고 나면 저 이메일로 확인 링크가 전송된다. 받고 비밀번호 설정하면 계정 설정은 끝이다.

뭔가 초기화가 되면 구독 페이지가 활성화가 된다.

이동해서 세팅해주면


이렇게 뭐가 잔뜩 뜨고, 시작 링크도 뜬다. 저걸 써서 IDE에서 연동을 하면 된다.




IDE 설정 (VSCode)

먼저 확장을 설치한다.

그러면 체험판으로 쓸 수도 있고, 방금 만든 구독 계정으로 시작할 수도 있다.

방금 받았던 Start URL을 입력하고


브라우저 기반으로 인증을 하면 된다.
이거 기능구현 이상하게 되어있어서 처음 로그인할때는 로그인 후 종료하고 다시 진입해야 저 화면이 뜨더라.

아무튼 로그인이 잘 되면

이렇게 채팅이 가능해진다.
이거 아직도 다국어 지원이 전혀 안되고 영어만 되더라. 무슨 자신감인지 모르겠다. 장사할 생각이 있긴 한건가?

그렇게 연동이 되면

구독 창에도 카운트가 늘어났을 것이다.




기본 사용법

가장 기본적인 기능은 코파일럿처럼 사용자 타이핑에 반응해서 약간의 코드를 생성해주는 기능이다.

이건 코파일럿과 사용성에 큰 차이는 없었다.

그리고 채팅창에 /dev를 치면 바로 개발모드가 되는데

여기서 요청을 하면 cline처럼 자체적인 피드백 루프를 시작해서 코드 생성을 시도한다.
코드 생성 시간은 좀 긴 편이다. 다른 것들에 비해 느리다.


이것도 코드를 제시하고 approve하는 식으로 코드 생성이 된다.
대신 cline과 다른 부분은, 중간 과정을 보여주지 않고, 컴파일 체크 같은 것도 하지 않는다는 것이었다.
사용성은 떨어진다.


그래서 돌려봤더니 컴파일도 안되는 코드더라.

그리고 좀 더 깨작거려봤는데 뭔가 내가 원한 수준의 퍼포먼스는 나오지 않았다.
