#Azure
[원본 링크]

.png?type=w800) azure functions는 aws lambda, gcp function에 대응되는 서버리스 함수 실행 서비스다.




비용

비용은 다음 문서를 참조하면 좋겠다.
https://azure.microsoft.com/ko-kr/pricing/details/functions/
한달에 100만건까지는 무료고, 이후부터는 추가 요금이 부과된다.





함수 생성

함수 콘솔로 이동해 "만들기"를 클릭한다.

구독과 리소스 그룹은 취향껏 넣어주고, 이름과 런타임을 결정한다.

이름이 도메인으로 박혀서 unique 검사가 까다로우니 감안을 좀 해야하고, OS도 선택할 수 있다.


런타임은 그렇게 많지는 않다.
이외에 필요한게 있다면 커스텀을 사용해야할 것이다.

완료를 하면, 리소스가 생성될 것이다.
세팅에는 시간이 좀 걸린다.




실행

Azure 함수가 꽤 편리한 점이, URL을 기본으로 뽑아준다는 것이다.

접속하면 이렇게 템플릿 프로그램이 실행된다.




코드 편집

Azure Function은 좀 불편하게도 콘솔에서 바로 코드를 보거나 고칠 수는 없다.

빠른 시작에 보면 편집기를 세팅하고 배포하는 법에 대해 알려준다.

나는 VSCode를 자주 사용하므로 VSCode 위주로만 다루겠다.

Azure Function용 확장을 깔고

npm으로 다음 모듈을 설치해준다.

npm install -g azure-functions-core-tools@4 --unsafe-perm true

functions용 커맨드툴도 깔아두는게 좋다.
아래 링크에서 다운받거나, 다운받는법을 볼 수 있다.
https://docs.microsoft.com/en-us/azure/azure-functions/functions-run-local?tabs=v4%2Cwindows%2Ccsharp%2Cportal%2Cbash#v2

배포까지 커맨드로 하려면 azure cli도 있어야 한다.
다음 링크에서 플랫폼대로 설치하자.
https://docs.microsoft.com/ko-kr/cli/azure/install-azure-cli

확장을 깔았다면 아래와 같은 관리탭이 추가된다.

그리고 폴더를 만들고 관리도구로 설정파일을 초기화해준다.

func init

그 다음에는 함수 코드를 추가한다.

func new --name HttpExample --template "HTTP trigger" --authlevel "anonymous"

그러면 다음과 같이 프로젝트 폴더가 구성될 것이다.

그럼 생성된 템플릿을 실행해서 엔드포인트를 알려줄 것이다
