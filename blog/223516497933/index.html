[k8s] Failed to inspect image "..." did not resolve to an alias and no unqualified-search...

  • 2024.07.18
  • ::
  • 1 min read
#Kubernetes
[원본 링크]

crio를 컨테이너 관리자로 쓰면 이런 에러가 발생할 경우가 있다.

Failed to inspect image "myyrakle/node-server-for-test": rpc error: code = Unknown desc = short-name "myyrakle/node-server-for-test:latest" did not resolve to an alias and no unqualified-search registries are defined in "/etc/containers/registries.conf"

내 경우에는 문제없이 사용하다가 어느날부터 갑자기 이런 문제가 생겼는데, 계기가 뭐였는지는 잘 모르겠다...
생각없이 우분투 업데이트 때린게 문제였나?

아무튼 원래 레지스트리를 명시하지 않으면 도커허브 기본 레지스트리에서 이미지를 땡겨오는데, 특정 버전부터는 모호성을 제거하기 위해서 명시해야만 하도록 동작이 바뀌었다는 모양이다.

근데 난 그런건 관심없고 그냥 이전처럼 동작하기를 원했다.
애초에 기존 스택들도 끄냥 명시하지 않고 띄우게끔 쌓여있는데 어쩌란 말인가?

registries.conf라는 파일을 수정해서

sudo vi /etc/containers/registries.conf

unqualified-search-registries = ["docker.io"]

도커를 저 기본 레지스트리로 설정하고


crio를 재부팅하면 된다.


그럼 이제부터는 잘 될 것이다.
