[Git] worktree

#Git
[원본 링크]

worktree는 git의 유용한 부가기능 중 하나다.
여러개의 작업 상태를 동시에 관리할 수 있게 해준다.

마법처럼 엮어서 처리해주는 그런건 아니고, 기실 동작 원리 자체는 프로젝트를 여러개 clone해주는 것이 다다.
매번 clone하거나 복붙해서 관리하는 것보다는 편하다.

git worktree의 서브커맨드들을 통해서 사용할 수 있다.




worktree 추가

worktree add 명령을 사용하면 tree를 추가할 수 있다.
tree가 생성될 경로를 지정해야 한다. 보통은 부모 경로를 지정해서 (../tree명)의 형태로 보낸다.

git worktree add ../treename # 브랜치를 만들면서 생성
git worktree add ../treename 브랜치 # 특정 브랜치로 생성

그리고 tree가 만들어지면

실제로 부모 디렉터리에, tree의 이름으로 프로젝트의 clone이 생성된다. 거의 동일한 복제다.




worktree 다루기

그리고 worktree list 명령을 사용하면, 전체 tree 목록을 조회할 수 있다.

worktree는 branch처럼 코드 버전을 자동을 전환해주고 그런건 없다.
새로 만든 tree에 대해서 작업을 하려면, 방금 만든 tree 디렉토리에 들어가서 해야 한다.
커밋이나 브랜치도 완전히 별도로 관리된다.

tree라는 글로벌 컨텍스트를 통해 최소한의 정보만 공유할 뿐이다.




Worktree 제거

작업을 완료했다면 tree를 정리하기도 해야한다.
remove 명령을 사용하면 worktree를 실제 디렉토리와 함께 지워준다.

worktree의 상태가 깨끗하지 않다면 지워지지 않기도 한다. 그럴때는 --force 옵션을 주거나 직접 지워줘도 된다.
