당연히 클래스도 지원이 된다.

선언 키워드는 class.
생성자는 클래스명과 동일한 함수로 선언하면 된다.

다트엔 접근제어자가 따로 존재하지 않고, 모든 멤버가 public 수준을 갖는다.

아래 코드는 좌표상의 위치정보를 저장하는 타입 Point를 구현한 것이다.


new 생략
위에선 객체를 생성할때 new를 썼지만 dart2부턴 new를 생략할 수 있다.


생성자 간략화
그리고 생성자의 인자는 다음과 같이 지정할 수도 있다. 위 코드들과 같게 동작한다.


이름있는 생성자
그리고 생성자도 다른 메서드처럼 오버로딩이 불가능하다.
또다른 생성자들을 정의하고 싶다면 이름있는 생성자를 사용할 수 있다.
타입명.생성자명(인자...) {...}
의 형태를 가진다.

아래 코드는 인자를 받지않는 생성자를 추가한 것이다.

그리고 toString 메서드를 오버로딩했는데, 저러면 print에 들어가면서 자동으로 문자열로 변환된다.


정적멤버변수와 정적메서드
기존 변수와 메서드에 static만 붙이면 객체와는 별개로 존재하는 정적멤버가 된다.
아래 코드는 기존 Point에 객체가 생성된 횟수를 기억하는 변수 _count와 그걸 가져오는 메서드 count를 추가한것이다.


게터와 세터
타입 get 게터명 => 표현식;
set 세터명 (단일인자) { ... }
와 같이 쓰면 게터와 세터가 선언된다. 

보통 메서드들은 같은 이름을 가질 수 없지만 게터와 세터는 예외적으로 이름을 겹칠 수 있다.

게터로는 읽기작업만이 가능하고, 쓸수는 없다.

읽는건 잘 되지만


쓰는건 안된다.

타입 생략도 가능하다.
어차피 그냥 변수랑만 대응시키니까 굳이 명시할 필요는 없을거같다.

세터는 단일 인자를 갖는 함수처럼 구현된다.

이제 값이 잘 들어간다.


var 멤버변수
멤버변수에 타입을 생략하고 var로 때울수도 있지만, 그러면 해당 멤버변수는 동적타입이 돼서 예상치 못한 오류를 초래할 수 있다.
