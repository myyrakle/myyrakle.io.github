[GCP] Google Compute Engine

  • 2021.12.01
  • ::
  • 3 min read
#GCP
[원본 링크]

Google Compute Engine은 GCP의 가상머신 제공 서비스다.
AWS의 EC2 등에 대응된다고 할 수 있다.

막 그렇게 복잡한건 아니고, 그냥 컴퓨터 하나 띄워서 돌릴 수 있게 하는 거라고 보면 된다.




비용

Compute Engine은 기본적으로 사용 시간을 기반으로 요금을 부과한다.
네트워킹이나 이런저런 잡다한 부분으로도 나가긴 하지만, 기본은 그렇다.

자세한 것은 아래 페이지에서 확인할 수 있을 것이다.
https://cloud.google.com/compute/all-pricing?hl=ko-KRGoogle

그리고 GCP는 가격 관련해서는 안내가 친절한 편이라, 생성창에서도 비용을 대충 볼 수 있다.





인스턴스 생성

그럼 이제 인스턴스를 생성해보자.
Compute Engine 페이지로 이동해서, 인스턴스 만들기를 클릭한다.

그럼 여기서 이름과 리전, 가용영역, 스펙을 선택하면 된다.

리전은 실제로 저 인스턴스가 어느 지역에 있는 데이터센터에 올라갈지를 결정한다. 가급적이면 사용할 지역에서 가까운 곳에 두는게 성능상 유리하다.

그리고 머신 스펙은 바로 오른쪽에 예상 가격도 표시해주니까, 필요에 따라 선택해주면 된다.

OS 종류도 선택할 수 있다. 기본값은 데비안이다.

데비안 외에도 센토스, 페도라, 레드햇, 우분투, 윈도우즈 등이 제공된다.


그리고 HTTP 웹서버를 열거라면 저기서 방화벽을 열어주면 된다.

잘 생성했다면 이런식으로 인스턴스 목록에 나타날 것이다.

그럼 이제 만들어진 인스턴스에 접속해보도록 하자.

인스턴스 페이지로 이동하면 다음과 같은 창이 뜰 것이다.

인스턴스 접속 (SSH)

하지만 저게 싫다면 그냥 ssh 클라이언트로 접속할 수도 있다.

아래와 같이 ssh 키를 먼저 생성한다.
아이디는 계정 이메일에서 @ 오른쪽을 다 뗸걸 넣으면 된다.

 ssh-keygen -t rsa -f ./파일명 -C 아이디 -b 2048

그러면 passphrase를 입력하라고 하는데, 여기에 접속할때 사용할 패스워드를 넣어주면 된다.

그럼 이렇게 비공개 키와, 공개 키가 나올 것이다.

공개키는 방금 만든 .pub 파일을 말한다.
이런식으로 쭉 뽑아서 복붙해주면 된다.

그러면 이제 외부 IP를 통해 다음과 같은 형식으로 접근이 가능해질 것이다.

ssh 아이디@외부IP -i 비밀키파일

그렇다.




인스턴스 일시정지

인스턴스를 잠시간 사용하지 않는다면, 정지를 해놓는게 내 지갑을 위한 선택이 될 것이다.
하지만 안타깝게도, GCP는 정지된 인스턴스에 대해서도 어느정도의 요금을 청구한다.

그냥 쌩돈을 뜯어가는게 아니라, 말 그대로 일시정지라서 상태값을 완전히 보존해주기 때문이다.




인스턴스 중지

상태의 유지가 굳이 필요하지 않고, 다시 쓸때 재부팅되기만 하면 된다면, 그냥 중지를 하면 된다.

중지된 인스턴스에 대해서는 요금이 따로 나가지 않는 것 같다.
