Namespace
Java에서는 패키지라는 디렉토리개념으로 이름의 충돌을 관리한다.
그리고 C++에서는 네임스페이스라는 논리적인 개념으로 이름 충돌을 관리한다.

이 부분에서 C#은 패키지는 걷어차버리고 네임스페이스를 가져왔다.

namaspace 스페이스이름
{
...
}

접근을 스코프연산자' :: '를 따로 쓰지 않고 . 으로 접근한다는것만 빼고 전부 C++과 같다.


Global
전역 네임스페이스를 가리키는 키워드다.
이건 ,아 아니라 ::를 써야한다.
이렇게



Using
그리고 이름공간을 포함하는 것도 using을 가져와서 쓴다.
이건 C++보다 제약이 좀더 있는데 무조건 파일의 맨 위에다 모아놔야 한다. C++처럼 막 로컬범위에서도 쓰고 그런거 안된다.

alias 기능도 고대로 가져왔다.
using 네임스페이스별명 = 네임스페이스.네임스페이스2. ...;

그리고 버전 6.0부터는 정적 클래스 자체를 포함시키는 using static도 추가됐다. 자바의 import static과 동일한데,
using static System.Console;
이러면 Console이라는 정적클래스를 명시하지 않고 WriteLine 등을 쓸 수 있는 것이다.

마지막 용법으로, using은 특수한 형태의 예외처리에 대한 축약 구문으로 쓰이기도 한다. 이건 예외파트에서 쓰겠다.
