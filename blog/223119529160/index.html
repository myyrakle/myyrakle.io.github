API Gateway와 Lambda를 활용한 서버리스 Websocket 서버 구성을 말아보았다.

websocket 서버를 구성하는건 생각보다 번거롭고 운영상에서 신경쓰이는 부분이 많은데, AWS에서 제공해주는 Websocket Gateway를 사용하면 서버리스하게 운영이 가능하다.
보통은 Redis 같은 인메모리로 관리하는 커넥션에 대한 부분을 Websocket Gateway가 알아서 처리해준다.




기본구성

먼저 git 레포지토리를 준비한다.
다음 템플릿을 복사해서 쓰는 것을 추천한다.
https://github.com/myyrakle/websocket-server-node

그리고 다음과 같이 terraform 변수를 정의한다.

region            = "ap-northeast-2"
server_name       = "foobar"
environment       = "prod"
codestar_arn      = "arn:aws:codestar-connections:ap-northeast-2:...:connection/50e...a29735d"
buildspec_path    = "./setup/master/buildspec.yml"
github_user       = "myyrakle"
github_repository = "websocket-server-node"
github_branch     = "master"

codestar는 없으면 github 연동해서 생성해야하고, 이미 있다면 aws cli를 통해서 조회해볼 수 있다.

그리고 생성해주면 된다.

이걸 통해서 다른 client들에게 메세지를 쏘는 것이다.




CI/CD

CodePipeline으로 구성되어있고, 소스코드를 수정하면 3개의 Lambda를 동시에 다 업데이트쳐주도록 했다.




코드 구조

Lambda는 connect, disconnectm, default가 생성되는데, connect와 disconnect는 수정할 일이 많지는 않을 것이다. connect는 Gateway에 의해 소켓이 연결되었을때 다이나모 테이블에 커넥션 정보를 저장하고, disconnect는 삭제하기만 한다.

그래서 보통은 default를 수정하게 될 것이다.

여기서 필요한 action들을 정의하고, 그때그때 필요한 유저들의 커넥션 목록을 긁어와서 postToConnection으로 메세지를 쏘아주면 되는 것이다.
