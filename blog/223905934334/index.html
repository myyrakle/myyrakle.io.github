[Python] ruff: lint & formatter

  • 2025.06.20
  • ::
  • 2 min read
#Python
[원본 링크]

ruff는 uv 등의 핵심 보조도구를 만든 astral 그룹의 또다른 역작이다.
원래 부실하거나 어정쩡하던 lint+formatter 환경을 통합된 형태로 제공하는 린트 도구다.

빠른 성능과 다양한 규칙 지원, lint+formatter가 통합된 기능, 사용 편의성을 기존 도구들과 차별화되는 장점으로 삼는다.

https://docs.astral.sh/ruff/
이건 astral에서 내놓은 벤치마크라 좀 불공정한 부분이 있을 수도 있는데, 빠르긴 빠른 것 같다.
확실히 알 수 있는 건 pylint가 가장 느리다는 것이다.




설치 (CLI)

uv를 쓴다고 가정한다.
설치는 별거 없다.

uv add --dev ruff




기본 사용법 (CLI)

이런 코드가 하나 있다고 가정하겠다.

import os

def Fibonacci(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    return Fibonacci(n-1) + Fibonacci(n-2)

if __name__ == "__main__":
    num = 10
    print(f"Fibonacci number at position {num} is {Fibonacci(num)}")
uv run ruff check 파일명

파일 경로를 지정하지 않을 경우에는, 현재 경로를 기반으로 알아서 다 찾아서 체크한다.




auto fix 기능

모든 lint 항목에 대해서 지원되는 것은 아니지만, 일반적으로 처리할 수 있는 경고 항목은 자동으로 수정되게 처리할 수도 있다.
위에서 발생한 unused import 같은 경우에는 check를 돌릴때 --fix 플래그를 넣으면 잘 고쳐준다.




formatting 기능

ruff는 다른 lint 도구들과는 다르게, 포맷터 기능도 함께 제공한다. check 대신 format 명령어를 사용하면 된다.

이런 코드가 있다면


<br>

돌렸을때

uv run ruff format

이렇게 예쁘게 잘 맞춰줄 것이다.
이 경우 강제개행, 화이트스페이스 제거 등이 처리됐다.




규칙 관리

lint 규칙은 uv를 쓸 경우 pyproject.toml로 관리할 수 있고, 그게 아니라면 ruff.toml로 관리할 수 있다.
여기서는 전자를 쓴다고 가정한다.

일단, ruff는 규칙을 따로 설정하지 않는다면 formatting은 black 규칙을 따르고, lint는 Flake8의 F 규칙을 따른다. Flake8는 PEP8 기준의 표준 컨벤션을 지향하는 lint다.

규칙 추가는 이런 식으로 할 수 있다.

이러면 한 줄에 79자 미만으로 유지가 되도록 포맷팅을 하겠다는 뜻이다.


그리고 E501라는 린트 룰을 추가하면 저 line-length 값을 파라미터로 받아서 실제로 린트를 돌리게 된다.

저 전체 규칙 목록은 다음 페이지에서 확인할 수 있다.
https://docs.astral.sh/ruff/rules/

포매팅 규칙은 tool.ruff.format 절로 관리할 수 있다.

포매팅 규칙은 다음 문서에서 참조하면 된다.
https://docs.astral.sh/ruff/settings/#format
