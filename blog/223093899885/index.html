#Proxy
[원본 링크]

HAProxy는 로드밸런서에 주로 사용되는 프록시 시스템이다.

nginx가 로드밸런서 외에도 여러가지 역할을 할 수 있는 만능 재주꾼이라면, HAProxy는 로드밸런서 기능에만 충실하게 집중한 녀석이라고 보면 된다.

그래서 로드밸런서로서는 이게 더 우수하지만, 로드밸런싱 외에도 뭔가 동적인 기능이나 로직을 추가해야한다면 이것보단 nginx 같은 녀석들이 더 나을 것이다.

그리고 생각보다 연식이 꽤 오래된 놈이다. 2006년에 나왔다.
근데 뭔가 세월의 흐름에는 잘 따라가지 못하는 것 같다. SSL 지원도 매우 느렸고, MSA 기능 지원도 부실하다.




설치 (Linux)

devian 계열의 경우에는 그냥 패키지 매니저로 바로 설치가 가능하다.
설치하고

sudo apt-get install haproxy

바로 cli로 확인을 해볼 수 있다.




기본 설정 구성

haproxy는 nginx처럼 깔았다고 바로 뭐가 뜨진 않는다.
한번 다른 서버를 라우팅해보자.

1111 포트로 서버를 띄워둔게 있다.

이걸 가리키게 하려고 한다.

설정파일을 수정한다.

sudo vi /etc/haproxy/haproxy.cfg

초기에는 대강 이런식으로 되어있을 것이다.

밑에다가 아래 코드를 추가한다.

frontend test-front
   mode http
   bind *:80
   default_backend test-back

backend test-back
   server master 54.180.123.74:1111

프론트엔드와 백엔드 설정을 둘다 해줬는데, 백엔드만을 노출하고 싶다고 해도 프론트엔드를 통해서만 최종 export가 가능한 구조기 때문이다.

frontend에서는 외부로 공개한 ip와 port를 지정하고, 라우팅할 백엔드를 지정한다.
backend는 서버 원본을 가리킨다. 여기에는 여러개가 들어갈 수 있다.

그리고 systemctl로 재시작하면 반영이 될 것이다.

sudo systemctl daemon-reload
sudo systemctl restart haproxy
sudo systemctl status haproxy

재배포되는 사이의 downtime을 없애고 싶다면 restart 대신 reload 명령을 사용하면 된다.


잘 된다.
