[PostgreSQL] 열거형

  • 2021.04.25
  • ::
  • 1 min read
#PostgreSQL
[원본 링크]

PostgreSQL에서도 열거형 타입에 대한 기능을 제공한다.
다른 언어들의 열거와 마찬가지로, 미리 지정된 값만을 받을 수 있도록 제약이 걸린 타입을 만든다.

생성은 다음과 같이 할 수 있다.

CREATE TYPE 타입명 AS ENUM(열거 값들...);

나는 내가 원하는 특정 색깔만을 넣을 수 있는 색상 타입을 정의했다.
내가 넣은 열거 속성은 정열적인 적색, 분홍색, 검정색이다.

열거 값 목록에는 같은 타입의 값만을 넣을 수 있다.

열거를 잘 정의했다면, 다음과 같이 그냥 타입으로 사용할 수 있다.
색상을 포함하는 간단한 테스트용 테이블을 정의했다.




값 제한

열거를 사용하는 이유는, 해당 타입이 특정한 값만을 가질 수 있다는 강력한 보장을 가지기 때문이다.
그래서 열거에 정의한 속성은 얼마든지 넣을 수 있지만

정의하지 않은 속성은 들어갈 수 없다.




타입 고유성

열거는 다른 타입을 통해 정의되지만, 그 자체로도 하나의 고유한 타입이다.
그래서 문자열로 열거를 구성했다 하더라도, 그건 열거타입이지, 문자열이 아니다.
그래서 다음과 같은 쿼리를 타입 오류를 발생시킨다.

내부 타입의 값을 꺼내서 유효하게 처리하고 싶다면 명시적 캐스팅을 사용하면 된다.
