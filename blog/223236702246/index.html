[Go] type alias와 named type

  • 2023.10.14
  • ::
  • 1 min read
#Go
[원본 링크]

Go에서는 type에 새 이름을 부여하는 것에 2가지 방법이 제공된다.

이 2가지 신택스는 일견 비슷해보이지만, 완전히 다른 기능이다.




type alias

type alias는 = 연산자가 가운데 들어간다.
이거는 정직하게 진짜 별칭만 만드는 것이고, 별칭과 원본 타입은 완전히 동일한 타입이다.

그래서 그냥 스펠링만 다르고 완전히 동일하게 사용할 수 있다.

package main

import "fmt"

type SomeType struct {
	Counter int
}

type A = SomeType // 이건 alias
type B SomeType   // 이건 새로운 타입

func main() {
	someValue := SomeType{Counter: 1}

	var aValue A = someValue // 완전히 같은 타입이므로 할당 가능
	fmt.Println(aValue.Counter)
}




named type

=를 빼면 named type 신택스가 된다.
이건 단순히 별칭을 만드는데만 그치는게 아니라 새로운 타입을 만든다.

이게 무슨 소리나면

package main

import "fmt"

type SomeType struct {
	Counter int
}

type A = SomeType // 이건 alias
type B SomeType   // 이건 새로운 타입

func main() {
	someValue := SomeType{Counter: 1}

	var bValue B = someValue // 다른 타입이므로 type check
	fmt.Println(bValue.Counter)
}

타입이 1:1로 매칭되지 않는다는 것이다.

이건 기존 타입을 재활용해서 사용할 때 유용하다. 타입체크도 빡세게 되니 사용처를 엄격하게 제한하기도 좋다.

상호간에 변환이 필요하다면 아래와 같이 캐스팅을 하면 된다.

named type의 가장 큰 장점은 사실, 새로 중복정의된 타입에 대해서 메서드를 추가 정의할 수 있다는 것이다.

외부 패키지에 대해서도 메서드 추가가 가능할 뿐더러

원래는 되지 않는 슬라이스 같은 타입에 대해서도 named type 정의가 된 상태에서는 메서드를 구현할 수 있다.

내 경험상, type alias보다는 named type이 훨씬 적합한 경우가 많았다.
