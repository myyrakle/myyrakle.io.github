[AWS] Cloudfront: Function

[원본 링크]

Cloudfront Function은 Cloudfront에서 요청이나 응답을 가로채서 조작할 수 있는 엣지 수준 부가기능이다.

Lambda Edge와도 비슷한 느낌으로 동작하나, 조금 다르다.

**Lambda Edge **

  1. Lambda Edge는 이미지 최적화 같은 상당히 무거운 연산을 포함한 기능을 구현할때 적합하다.

  2. 응답이나 요청 등 다양한 시점에 트리거가 가능하고, 확장성이 뛰어나다.

  3. 그대신 무겁고 비싼 편이다.



Cloudfront Function

  1. Function은 간단한 HTTP 응답 헤더 조작이나 리디렉션 같은 매우 가벼운 조작에 적합하다.

  2. 매우 작은 코드만 올릴 수 있고, 제한적인 대신에 빠르고, 훨씬 많은 엣지에 배포되어 물리적으로도 빠르다. 1밀리초 미만의 실행시간을 보장한다.

  3. 비용이 저렴하다.

2. 함수 코드 작성하기

Function 콘솔에 들어가면 AWS Lambda 처럼 자체적인 에디터를 바로 제공한다. 그리 편하진 않지만

개발, 라이브 2개의 스테이지를 기본으로 제공한다.

먼저 개발 코드에다가 아래 코드를 집어넣자.

function handler(event) {
  const request = event.request;

  const supported_countries = ["de", "it", "fr", "kr"];

  if (request.uri.substr(3, 1) != "/") {
    const headers = request.headers;

    let newUri;

    const country = headers["cloudfront-viewer-country"];

    if (country) {
      const countryCode = country.value.toLowerCase();
      if (supported_countries.includes(countryCode)) {
        newUri = "/" + countryCode + request.uri;
      }
    }
    if (newUri === undefined) {
      const defaultCountryCode = "en";
      newUri = "/" + defaultCountryCode + request.uri;
    }
    const response = {
      statusCode: 302,
      statusDescription: "Found",
      headers: {
        location: { value: newUri },
      },
    };
    return response;
  }
  return request;
}

로직은 그리 복잡하지 않다.

  1. 이미 다국어 라우팅이 된 /de... 같은 경로라면 request.uri.substr(3, 1) != "/"에 걸려서 리디렉션 없이 그대로 반환한다. 아니라면 다국어 라우팅 시도한다.
  2. Cloudfront에서 자체적으로 국가를 구분하기 위해 받는 cloudfront-viewer-country라는 헤더를 가져다가, 국가를 구분한다. 국가코드는 ISO 표준에 기반한다.
  3. 국가코드가 없거나 모르는거면 영어 /en로 라우팅한다.
  4. 그렇게 해서 독일, 이태리, 불란서, 한국이 들어오면 /kr/, /de/, ... 같은 경로로 리디렉션하도록 구성한 것이다. (location 헤더 활용)

여기서는 location으로 떠넘겨서 uri까지 변경되게 하는 방법을 취했지만, request.uri를 변경해서 반환하는 방법도 있다. 그러면 리디렉션 없이 S3 오리진만 바뀐 채 동일한 uri로 가져올 수도 있을 것이다.

아무튼 저렇게 넣어서 변경사항을 저장하고, 테스트 기능을 활용해서 잘 동작하는지 확인해보자.
