#Terraform
[원본 링크]

인프라를 구성하다보면, 환경별로 동일한 리소스를 구성해야할 경우가 매우 많다.

예를 들어, 실무에서는 프로덕션 버전과 개발 버전을 동일한 형태로 복제해서 동시에 띄워놓는 경우가 대부분인데, 이런것도 하다보면 관리에 손이 꽤 많이 가게 된다.
그럼 terraform에서는 어떻게 해야할까? 그냥 prod, dev, stage별로 github repository 하나씩 다 파서 띄워야 할까? 그렇지는 않다.

terraform은 workspace 단위를 통해서 이러한 문제를 해결한다.
하나의 terraform 구성은 여러개의 workspace를 가질 수 있으며, 이를 통해 하나의 코드로 여러가지 환경의 분기를 처리할 수 있다.




기본 명령어

workspace는 terraform cli의 workspace 서브커맨드를 통해 제어할 수 있다.

list 명령을 사용하면 현재 워크스페이스 목록을 조회할 수 있는데
