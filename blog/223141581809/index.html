[Rust] trait: object safety

  • 2023.06.28
  • ::

#Rust
[원본 링크]

Trait은 정적 디스패치를 통한 컴파일타임 Generic 기능 또한 가지고 있지만, 본질적으로는 여러가지 하위 구현 타입에 대한 공통적인 인터페이스(interface)다.
동적으로 할당된 힙 데이터에 다형성을 제공해주는 용도라는 것이다.

그래서 trait의 메서드들은 공통적으로 vtable에 적재되어 런타임에 명확하게 식별이 될 수 있는 형태여야 하고, 그런 형태를 object safety라고 부른다.
object safety를 준수하지 않아도 트레잇을 정의하고 사용할 수는 있지만, 동적할당에 사용할 수는 없다.

하지만 Box 등을 통해 동적 디스패치 객체를 만들려고 하면 문제가 생긴다.

trait이 object safety하지 않아졌기 때문이다.

다시 보자.

여기서 Self는 하위 구현타입을 가리키게 된다.
그러면 Foo 구조체는 &Foo를, Bar 구조체는 &Bar를 반환 타입으로 가지게 될텐데, 항상 동일한 타입으로 동작해야 하는 trait 메서드가 타입따라서 다른 타입을 반환하는게 말이 되겠는가?

그래서 이러한 메서드는 vtable에 적재될 수 없고, 그 때문에 동적 객체에 할당될 수 없는 것이다.

이건 리턴타입뿐만 아니라 파라미터 타입에 쓸 때도 마찬가지인 부분이다.

그래서 이러한 기능들은 별도 트레잇으로 분리해내거나, 아니면 단순 구조체 구현부에 각각 두는게 맞다.
애초에 잘못된 설계라고 봐야한다.
