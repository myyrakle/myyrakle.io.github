cursor도 여타 AI 보조도구들처럼 프롬프트 규칙을 정의하는 기능을 제공한다.

원래는 프로젝트 루트 경로에 .cursorrules라는 단일 파일을 두는 형태였는데, 한번 엎어서 그건 레거시고 rules라는 기능이 따로 추가됐다.

.cursor/rules라는 디렉터리에 ".mdc"라는 파일형태로 여러개 둘 수 있는 구조다.

이렇게 말이다.

이것도 단순 텍스트 형식이긴 한데, cursor를 쓴다면 기본적인 메타데이터 설정은 가시화를 해준다.

규칙이 항상 적용되게 하려면 always를 선택하면 되고


실제 raw 데이터에는 이렇게 적힌다.

저장해서 찔러보면, 알아서 읽고 반응할 것이다.

Auto Attached는 특정 파일을 스캔할때만 반응해서 참조하게 되는 규칙이다.

Agent Requested는 설명을 써놓으면, Agent가 설명을 기반으로 필요하다고 판단할때만 읽는 규칙이다.

Manual은 사용자가 프롬프트에 명시적으로 언급할 때만 읽는 규칙이다.
