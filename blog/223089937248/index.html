[AWS] EKS: EC2 노드 구성

  • 2023.05.01
  • ::
  • 1 min read
#AWS
[원본 링크]

ec2로 노드를 구성하는 방법을 정리한다.




점검

EC2로 노드그룹을 만들려면, 프라이빗 엔드포인트 액세스가 활성화되어있어야 한다.

안돼있으면 켜준다.



노드그룹 만들기

우선 현재 클러스터의 퍼블릭 서브넷에 퍼블릭 주소 자동할당을 활성화한다.

그리고 노드그룹에 쓸 IAM 역할을 만든다.

EC2 템플릿으로 해서

AmazonEKSWorkerNodePolicy 찾아서 넣고

AmazonEC2ContainerRegistryReadOnly 찾아서 넣는다.

그리고 클러스터로 이동해서 노드 그룹 추가

아까 만든 역할 달고

이렇게 해서

적당히 만들어준다.


그럼 생성모드 들어가고

노드 EC2 인스턴스도 뜰 것이다.

저 노드들을 기반으로 aws-node 데몬셋같은 기본 pod들이 다 뜨면, 잘 된 것이다.




서버 띄워보기

kubectl을 사용해서 한번 띄워보자.

아래는 내가 말아놓은 express 서버 이미지다.

kubectl create deployment test-server --image=myyrakle/node-server-for-test

그리고 확인을 하려면 로드밸런서를 달아준다.

kubectl expose deployment test-server --type=LoadBalancer --name=test-server --port=80

이러면 Elastic Loadbalancer를 자동으로 만들어서 서비스로 매핑해준다.

이렇게 뜨고

들어가서 쓸 수 있을 것이다.
