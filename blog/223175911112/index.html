[Go] circular import 문제

  • 2023.08.06
  • ::
  • 1 min read
#Go
[원본 링크] 
import cycle not allowed

Go 모듈시스템의 불편한 한계점 중 하나다.

계층형으로 모듈을 쌓아올리는 Rust 같은 모듈시스템에서는 이런 문제가 별로 없고 회피하기도 쉬운데, Go의 모듈시스템은 각 패키지 경로가 다 독립적이고 수평적인 관계라서 이런 귀찮은 문제가 있다.

예를 들어, A가 B 패키지를 import하고, B가 A 패키지를 import한다면 이런 문제가 발생하게 된다.

코드로 한번 예시를 들어보겠다.
각 프로세서의 동작을 정의하는 인터페이스 하나와, 그 프로세서들의 매핑 목록이 있다.

특정 문자열이 전달될때 해당 프로세서를 실행시키기 위한 구조 매핑이다.

그리고 저 인터페이스에 대한 간단한 구현이 있다.

여기까지는 괜찮다. 어차피 고랭의 다형성은 덕타이핑이라 인터페이스를 명시적으로 가져오지는 않으니까

하지만 가져올 일이 생길 수도 있다.
내 경우에는 지금 Processor 객체를 하나 받아서 연달아 실행하는 기능을 추가하려 한다.

그러면 이제

에러가 터진다.




일반적인 해결법

보통은 파일, 패키지단위를 잘 쪼개는 것만으로 해결을 볼 수 있다.
패키지-폴더 단위로 import를 하다보니 무조건 디렉터리 단위로 쪼개야 하는게 불편하긴 하지만..

일단 이 경우에는 interface를 독립적으로 분리하는 것이 좋다. 여기저기 불려다니는 녀석이니까.

경험상 인터페이스는 꼭 이런 경우가 아니라도 단독파일로 분리해두는걸 권장한다.
여기저기서 공통으로 불려다니기 쉬운 녀석이니, 당장은 괜찮더라도 쓰다보면 import가 꼬이기 딱 좋기 때문이다.

그리고 전역변수 매핑 Processors는 main 패키지로 옮겼다.

그럼 이제 잘 실행이 될 것이다.
