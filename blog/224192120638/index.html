OpenClaw는 최근에 부상한 에이전트 프레임워크다.

Claude Code 같은 CLI 도구가 파일시스템 수준의 제한적인 상호작용으로 에이전트 액션을 취했다면, 이건 좀 더 다양하고 넓은 상호작용과 피드백 루프로 공격적이고 풍부한 액션을 가능하게 해준다.

현 시점에서는 Linux, MacOS를 위주로만 지원하며, GUI 앱은 MacOS에서만 지원된다.

여기서는 기본적인 설치 방법, 기본 사용법만 다룬다.




사전 조건

Node.js 22 버전 이상이 필요하다.

사실 저렇게 한줄 설치 스크립트가 있으면 저런것도 알아서 깔거나 처리해줘야하는데, 세팅부터 날림으로 만든게 눈에 띈다.

버전 조정은 fnm을 추천한다.
https://blog.naver.com/sssang97/224101702777




설치

한줄 설치 스크립트와 npm 설치가 있는데, 개인적으로는 npm 설치를 권장한다.
한줄 스크립트는 어떻게 만든건지 이상하게 환경변수 꼬이거나 오동작하는 경우가 있다.

curl -fsSL https://openclaw.ai/install.sh | bash # 스크립트 

npm install -g openclaw # npm으로 설치

설치가 잘 된다면, CLI 명령을 통해 실행할 수 있을 것이다.




기본 구성

온보드 명령을 사용하면 tui로 세팅을 구성할 수 있다.

openclaw onboard

처음부터 보안 경고를 때리고 약관 동의를 받는다.

UI 방식


그리고 도중에 갑자기 이렇게 뻗고 오류만 뜨는 버그가 있다.
이럴때는 디바이스 목록을 지우고 처음부터 다시 세팅해야 한다.

rm -rf ~/.openclaw/devices/

아무튼 세팅이 완료되면, 즉시 채팅 기반의 tui가 실행된다.

온보딩 이후에 재진입하려면 openclaw tui를 치면 된다.

여튼 여기서 이제 바로 원하는 것을 요구하면 된다.
내 경우에는 간단한 테트리스의 구현을 요청했다.

만드는 것 자체는 금방 한다.

하지만 실질적인 동작 확인까지 알아서 하지는 않는다. 그럴 때면 요청을 해야한다.

그러면 실제로 실행하고, 가상 디스플레이를 통해서 의도한대로 동작하는 것까지도 확인을 해준다.

봇이 만든 코드는 .openclaw 내부 워크스페이스에서 관리된다.

만들어준 것을 가져다가 실행해보면

사실 많이 모자라긴 하지만, 기본적인 모양 자체는 구현이 되었다.
근데 양옆으로 블럭을 움직이는 것만 되고 블럭 회전을 구현하지 않았더라. 그럼 그렇지...

그래도 활용성은 꽤 괜찮은 것 같긴 하다. 보안 위험이나 토큰 소모 위험만 빼면 말이다.
