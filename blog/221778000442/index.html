[Spring] MyBatis 시작하기

  • 2020.01.19

#개발노트
[원본 링크]

마이바티스는 iBatis란 녀석의 후신으로,
데이터베이스 관리를 편리하게 처리할 수 있도록 도와주는 보조 라이브러리다.

대세는 JPA인듯 하지만, 여전히 이것도 많이 쓰이는 중이다.

먼저 세팅부터 하자.
메이븐에선 pom.xml을 열어서, 대충 이렇게 추가해주면 된다.

그리고 사용할 데이터베이스의 커넥터를 라이브러리로 등록해준다. 난 jar을 받아서 집어넣었다.

마이바티스가 설정된 상태에서 드라이버가 하나도 없으면 기본적으로 실행 자체가 안된다.

그리고 스프링부트 설정파일을 열어서, 아래와 같이 연동정보를 입력한다.

mysql의 test 데이터베이스에 연동한다는 뜻이고, 사용자명과 패스워드, 그리고 드라이버 클래스명이다.

그러고나선 설정파일을 하나 만든다.
마이바티스는 쿼리를 실행하기 위해서 SqlSession이란 객체를 사용하는데, 그걸 설정해주는 거다.

package com.example.mybatis.configuration;

import org.apache.ibatis.session.SqlSessionFactory;
import org.mybatis.spring.SqlSessionFactoryBean;
import org.mybatis.spring.SqlSessionTemplate;
import org.mybatis.spring.annotation.MapperScan;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.core.io.support.PathMatchingResourcePatternResolver;

import javax.sql.DataSource;

@Configuration
@MapperScan(basePackages = "com.example.mybatis.repository")
public class MyBatisConfiguration
{
    @Bean
    public SqlSessionTemplate sqlSessionTemplate(SqlSessionFactory sqlSessionFactory)
    {
        return new SqlSessionTemplate(sqlSessionFactory);
    }

    @Bean
    public SqlSessionFactory sqlSessionFactory(DataSource dataSource) throws Exception
    {
        var sqlSessionFactoryBean = new SqlSessionFactoryBean();
        sqlSessionFactoryBean.setDataSource(dataSource);

        //쿼리가 정의된 xml 파일들의 위치 지정
        sqlSessionFactoryBean.setMapperLocations(
                new PathMatchingResourcePatternResolver().getResources("classpath:mapper/*.xml")
        );

        return sqlSessionFactoryBean.getObject();
    }
}


그리고 테스트할 테이블은 이렇게 정의가 되어있다.

내용물은 이렇고.

이제 저걸 자바 객체로 매칭해보자.
각 행(Row) 값을 표현할 타입이다.

각 필드의 이름은 실제 테이블의 컬럼명과 맞춰줘야 한다.
맞추지 않아도 에러는 안 나지만, 값을 못 가져온다.

이번엔 레포지토리를 정의한다.
@Mapper만 붙이면 마이바티스 xml과 매핑되는 빈즈 클래스가 된다.

Name 값들을 전부 검색해서 가져올 selectAllName을 선언했다.
반환타입은 당연히 리스트에 담아놓고.

구현은 하지 않아도 된다. 마이바티스가 알아서 해준다.

자 그럼 xml을 통해 내부 쿼리를 정의해보자.
아까 Configuration에서 지정한 위치에다가, xml 파일을 대충 추가한다.

그리고 이렇게 써넣는다.

<?xml version="1.0" encoding="UTF-8" ?>

<!DOCTYPE mapper
        PUBLIC
        "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd"
>

<mapper namespace="com.example.mybatis.repository.NameMapper">
    <select id="selectAllName" resultType="com.example.mybatis.model.Name">
        SELECT * FROM NameTable
    </select>
</mapper>

mapper의 네임스페이스엔 방금 만든, 매퍼 인터페이스를 집어넣었다.
그리고 select 태그의 id는 매퍼 내에 정의했던 메서드와 이름을 맞춰주고, 반환될 타입도 지정해준다.
내부에는 실행될 쿼리를 써준다.

그럼 이제 다 짰다. 돌려보자.
아까 만든 매퍼 빈즈를 가져와서 검색을 시도했다.

잘 돌아간다.
