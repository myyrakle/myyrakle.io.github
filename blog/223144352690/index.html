[PostgreSQL] 상속 (Inheritance)

  • 2023.07.02
  • ::
  • 1 min read
#PostgreSQL
[원본 링크]

postgres는 특이하게도 테이블 간 상속에 대한 기능을 제공한다.
일반적으로 프로그래밍 언어들에서 제공하는 클래스 상속과 맥락을 같이한다.

만약 "사람"에 대한 테이블이 있고

create table Person
(
    id serial primary key, 
    name text, 
    age int
);

그 구성요소들을 그대로 포함하는 "학생"에 대한 테이블이 있다고 가정하겠다.

create table Student
(
    id serial primary key, 
    name text, 
    age int, 
    school text
);

사실상 school을 제외한 컬럼 정의들을 불필요한 중복이라고 할 수도 있다.
이럴때 컬럼 정의 재사용을 위해 사용하는 것이 상속이다.

상속을 활용해서 Student를 다시 정의하면, 이렇게 된다.

create table Student
(
    school text
) INHERITS(Person);

그러면 아래와 같이 생성이 된다.

그대로 다 가져왔고, 상속관계임이 표시된다.

한번 상속을 한다면, 하위 테이블은 지속적으로 그 상위 테이블에 대한 동기화를 유지한다.
그래서 나중에 상위테이블의 설정을 뭔가 바꾸더라도

alter table Person alter column name type varchar(100);

하위테이블에도 바로바로 반영될 것이다.
