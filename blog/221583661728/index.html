[Go] 패키지

  • 2019.07.12
  • ::
  • 1 min read
#Go
[원본 링크]

go는 패키지 단위로 모듈을 구성하고 재사용할 수 있게 해준다.

패키지의 선언은 package 키워드로 하는데, 패키지의 이름은 디렉터리명과 같게 한다. 소스파일 이름은 패키지명과 달라도 되고, 여러 소스파일이 같은 패키지명을 공유할 수 있다.


main
유일하게 특별한 의미를 갖는 패키지명은 main이다. go 컴파일러는 main 패키지를 진입점으로 인식하고 거기 들어있는 main함수를 찾아다가 실행시킨다.


import
import "패키지명"을 하면 만들었던(혹은 받았던) 패키지명으로 해당 패키지의 멤버들을 사용할 수 있다. fmt.Println을 쓰듯이...

여기서 또 까다로운 스타일 강요가 나타나는데, 소문자로 시작하는 이름은 숨겨져서 사용할 수 없고, 대문자로 시작하는 이름만 패키지에서 꺼내쓸 수 있다는 것이다.

그리고 import문은 자신의 디렉터리에 있는것뿐만 아니라 깃헙같은 코드 저장소에서도 직통으로 코드를 끌어다 컴파일을 할수도 있다.


init
init이라는 특이한 함수가 있다. 이건 해당 패키지가 로드되면 무조건 처음 호출된다.
이는 main 패키지에도 해당된다.
