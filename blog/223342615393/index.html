자잘한 기능들이 좀 추가됐다.



1. PathVariable 바인딩 구현

이전에도 request 파라미터를 통해 path param 값을 문자열로 가져올 수는 있었지만, 이제는 바로 파라미터에 바인딩해서 받아올 수 있다.

유효하게 타입 변환이 되지 않을 경우에는 400을 던진다.
PathVariable이 너무 길면 Path, path로도 줄여쓸 수 있다.

optional parameter라면 Option으로 감싸줘야 한다.

그래야 비어있거나 변환이 되지 않는 값이 들어와도 400을 던지지 않는다.

저기다 커스텀 타입을 박아넣고 싶으면 전용 역직렬화 trait을 구현해야 한다.
기본 타입들에 대해서는 이런 식으로 되어있다.

저 Path Param에 대해서 문서화를 해주고 싶다면 Description 어노테이션을 추가해줘야 한다.

이것도 Desc, desc로 줄여쓸 수 있다.




2. Swagger 구현

완전하지는 않지만, Swagger 문서 서빙은 좀 구현이 되었다.
아직 request body나 query param, response 정의 명시는 안된다.

이렇게 route를 만들어놓고 서버를 실행하면

/docs 경로로 문서가 서빙된다.

그래도 이정도는 된다.

태그 붙이고, 설명 붙이고, path param 설명 적는 정도다.


이렇게.
문서화 관련해서는 앞으로 할게 많다.




3. route 파라미터 동적바인딩

기존에는 request, response 같은 기본 파라미터의 순서를 맞췄어야 했는데, 이제는 마구잡이로 넣어도 돌아가게 동적바인딩 비스무리하게 해놨다.

저게 다 동작한다.
동적바인딩처럼 보이지만 매크로로 뇌절하는거다.


이외에는 몇가지 자잘한 기능들, 상수 정의, 버그 수정들이 포함된다.
