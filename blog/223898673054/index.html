[DuckDB] 시작

  • 2025.06.14
  • ::
  • 2 min read
#DuckDB
[원본 링크]

기본적인 사용법 정도만 다뤄본다.

duckdb는 sqlite와 많은 부분에서 닮았다.
RDB들처럼 중앙 DB 서버를 띄워서 쓰는 구조가 아니라, 필요할 때만 실행해서 쓰는 구조를 가진다.

그래서 그냥 CLI로 실행해서 가볍게 쓸 수도 있고, 프로그래밍 언어 수준에서 import해 사용할 수도 있다.
여기서는 언어에 독립적이고 간단한 CLI 사용법 정도만 다룬다.




CLI 설치

https://duckdb.org/docs/installation/?version=stable&environment=cli&platform=macos&download_method=direct

설치는 가이드 문서가 있어서 꽤 간편한 편이다.

리눅스의 경우에는 단일 명령으로 바로 설치할 수 있다.

curl https://install.duckdb.org | sh

그래서 이렇게 실행이 되면, 설치는 잘 된 것이다.




CLI로 바로 실행 (인메모리 모드)

따로 디스크에 데이터를 저장할 필요 없고, 간단한 휘발성 테스트만 하고 싶다면 그냥 duckdb만 쳐서 실행하면 된다.

이 상태에서 write한 데이터들은 현재 duckdb 실행 컨텍스트에서만 유효하고, 종료하면 다 날라간다.




CLI로 바로 실행 (디스크 모드)

duckdb를 실행할때 저장할 스토리지 파일명을 지정하면, DB 파일을 생성하면서 디스크 모드로 실행된다.

이 상태에서 write를 하면 그건 이제 스토리지에 저장이 된다.
종료했다 재실행해도 보존이 된다는 것이다.


이런 식으로 파일이 생성된다. wal도 함께다.




Query Syntax

DuckDB는 기본적으로 PostgreSQL 스타일의 문법을 지향하나, Sqlite, MySQL에 영향을 받은 부분도 있다.




데이터 output 기능

duckDB의 편리한 부분 중 하나는, 다른 데이터 포맷들과의 연계가 쉬운 편이라는 것이다.
쿼리 실행시에 .으로 시작하는 명령어를 적용할 수 있는데, 이러면 출력결과를 마크다운 파일로 만들겠다는 뜻이 된다.

.mode markdown
.output series.md
SELECT *
FROM generate_series(5);

csv도 되고

.mode csv
.output series.csv
SELECT *
FROM generate_series(6);

일반적인 포맷들은 다 된다고 보면 된다.

지원되는 전체 출력포맷 목록은 다음과 같다.

https://duckdb.org/docs/stable/clients/cli/output_formats.html





데이터 import

duckDB는 외부 데이터 포맷을 바로 데이터처럼 사용할 수 있는 기능을 제공한다.
예를 들어 이런 csv가 있다면

바로 쿼리를 통해서 로드해서 쓸 수 있는 것이다.

SELECT *
FROM read_csv('series.csv',
    delim = ',',
    header = true,
    columns = {
        'generate_series': 'VARCHAR'
    });
