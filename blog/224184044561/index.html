Asahi Linux KDE 세팅

  • 2026.02.14
  • ::
  • 1 min read
#Linux
[원본 링크]

내 취향에 맞춘 자잘한 세팅값들이다.
기록 겸 공유를 위해서 적어둔다. 수시로 업데이트될 수 있음




Command 키를 컨트롤 키로 매핑

맥북 자판 배열에 익숙하다면, Command 버튼을 컨트롤 키로 활용하는 것을 선호할 것이다.
하지만 아사히 기본 키보드 매핑에서는 진짜 컨트롤 키로 사용하지 않는다.

맥북과 동등한 경험을 원한다면, 키보드 키 바인딩에서 저걸 켜주면 된다.




한영 변환을 Option 키로 설정

기본 한영 전환 키는 컨트롤+스페이스다. 근데 이건 좀 불편하다.

이러면 오른쪽 Option 키로 한영 전환이 된다.




VSCode 키 바인딩

다른 플랫폼에서는 자동으로 바인딩되는 것들이 어째서인지 페도라+KDE에서는 안되더라

파일 익스플로러에서 엔터를 쳐도 파일명 변경 사애로 전환되지 않고, 컨트롤 백스페이스를 눌러도 파일이 삭제되지 않는다.
그걸 원한다면 키 바인딩에 다음 설정을 추가한다.

// Place your key bindings in this file to override the defaults
[
    {
        "key": "enter",
        "command": "renameFile",
        "when": "filesExplorerFocus && foldersViewVisible && !explorerResourceIsRoot && !explorerResourceReadonly && !inputFocus"
    },
    {
        "key": "ctrl+backspace",
        "command": "moveFileToTrash",
        "when": "explorerResourceMoveableToTrash && filesExplorerFocus && foldersViewVisible && !inputFocus"
    }
]
