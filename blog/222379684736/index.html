[AWS] SES (Simple Email Service)

  • 2021.06.01
  • ::
  • 2 min read
#AWS
[원본 링크]

걸그룹이 아니다.

SES는 아마존에서 제공하는 이메일 송수신 서비스다.
AWS 치곤 아주 비싼 편은 아니다.

들어가서 한번 써보자.




이메일 인증하기

이메일을 보내려면 뭘 해야할까?
당연히 이메일을 전송할 전송자의 이메일이 필요하다.
이메일이나 도메인을 하나 넣어주면 된다.

인증 메일이 오면 클릭해주면 된다.

그래서 이런 상태로 되면 성공이다.

그리고 잠깐 여기서 주의할 점이, 기본 설정에서 뭘 더 건드리지 않는다면 SES는 샌드박스 안에서만 동작한다.
샌드박스는 말그대로 테스트 모드인데, 이 상태에서는 수신할 이메일도 반드시 인증을 받은 이메일만 사용할 수가 있다. 아무 이메일로나 문자를 보내게 하려면 프로덕션 모드로 허가를 받아야 한다.

그래서 테스트모드라면 여기서 발신자도 미리미리 추가해주는 것이 좋겠다.




이메일 보내보기

우선 콘솔을 이용해서 메일을 전송해보자.
프로덕션에서는 이렇게 쓸 일은 없겠지만, 테스트용으로는 가끔 쓸 것이다.

전송자로 사용할 이메일을 체크한 후, 위의 버튼을 누르고

전송할 이메일 내역을 입력해서 전송하면 된다.
subject가 이메일의 제목이고, body가 내용이다.

저렇게 해놓고 보내면, 잘 올 것이다.




Lambda에서 전송해보기 (Node.js)

우선 람다를 적절히 생성한 다음 실행 역할로 들어가서

람다에서 SES에 접근할 수 있도록 권한을 연결해준다.

그리고 다음과 같이 소스를 작성해주면 된다.

const aws = require("aws-sdk");
const ses = new aws.SES({ region: "ap-northeast-2" });

exports.handler = async (event) => {
    const params = {
        Destination: {
            ToAddresses: ['rusty.wiki.dev@gmail.com'], //수신자 목록
        },
        Message: {
            Body: {
                Text: { Data: "으악" },
            },
            Subject: { Data: "Test Email" },
        },
        Source: "sssang97@naver.com", //전송자
    };

    return await ses.sendEmail(params).promise();
};

이렇게 해서 돌리면

잘 올 것이다.

그런데 저건 그냥 텍스트만 줘서 메일 형식이 많이 밋밋하다.
그래도 뭐 제대로 된 서비스다 하면 로고도 넣고 알록달록하게 꾸며야 하지 않겠는가?

그래서 일반적으로는 이메일의 html 소스를 직접 작성해서 그걸 보내도록 한다.
Text 대신에 Html 속성을 사용하기만 하면 된다.

그럼 지정한대로 잘 올 것이다.





프로덕션 모드로 전환

아무에게나 이메일을 전송할 수 있게 하고 싶다면 샌드박스 모드에서 벗어나 프로덕션 모드로 전환하게끔 해야 한다.
저기서 Edit을 누르고

실제 프로덕션 관련 정보를 입력한 뒤 아마존 측에 허가를 받으면 된다.

웬만하면 1시간 안에는 된다는 것 같다.

그렇다.
