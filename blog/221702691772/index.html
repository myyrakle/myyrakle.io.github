[C++] IIFE 트릭

  • 2019.11.09
  • ::
  • 1 min read
#C++
[원본 링크]

IIFE는 바로 호출되는 함수 표현식(Immediately-invoked function expression)의 약자로, 복잡한 초기화를 정돈할 때 사용한다.

모든 초기화 작업이 딱 생성자나 단순 대입만으로 끝나면 참 좋겠지만.
아쉽게도 그렇지 않은 경우가 많다. 특정 값을 대입 후에 추가적인 조건식을 수행하거나... 루프를 돌거나... 로직상으로 그렇게 될 수밖에 없는 경우들.

ㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡ
int foo = 1;

if(?)
**    foo+=?**
else
**    foo*=?**
//이후로 foo는 변경되지 않음
ㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡ
위와 같은 경우는 어떨까. 의미상으로는 상수지만, 불가피하게 작업할게 있어 const를 붙이지 못했다.

그럼 뭘 어떻게 해야할까? 함수로 분리할까?
그것도 나쁘지 않은 방법이긴 할 것이다.

하지만 더 좋은 방법이 있다. C++11의 람다를 쓰는 것이다.
람다를 응용하면 이렇게 바꿀 수 있겠다.
ㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡ
const int foo = []{
**    int sum = 1;**

**    if(?)
****        foo+=?**
****    else****
****        foo*=?****

****    return;****
}();
ㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡㅡ

람다를 생성해서 바로 호출, 변수에 바인딩한다.
이러면 변수의 상수성도 보존할 수 있고,
초기화 작업들을 저 람다 안에 가둬서 분리할 수도 있다.
