[AWS] localstack

  • 2026.02.11
  • ::
  • 2 min read
#AWS
[원본 링크]

localstack은 온프레미스 환경에서 AWS API를 쓸 수 있게 해주는 mocking용 시스템이다.

당연하지만 AWS와 완전히 동작이 같진 않고, 모든 기능을 제공하지도 않는 카피캣이다. 가벼운 테스트를 위해 존재한다고 보면 된다.
AWS에서 직접 관리하는건 아니지만, AWS 공식 문서에도 소개되어있을 정도로 나름 인지도 있다.




기능 목록

모든 것이 있지는 않지만, 일반적으로 쓰이는 기능들은 얼추 있다.

https://docs.localstack.cloud/aws/services/




Docker로 시작하기

네이티브 설치도 지원은 하는데, 가장 관리하기 편한건 아무래도 컨테이너다.

sudo docker run \
  --rm -it \
  -p 4566:4566 \
  -p 4510-4559:4510-4559 \
  localstack/localstack




응용

AWS API와 호환되기 때문에, aws cli를 사용해서 제어하는 것이 가능하다.
근데 aws cli를 쓰려면 먼저 로그인을 아무거로라도 해줘야 한다. 로그인 정보가 없으면 localstack과 상관없이 aws cli 자체에서 제한을 걸기 때문이다.

그러면 이제 Docker로 띄운 엔드포인트를 지정해서 API를 날려볼 수 있다.
다음 명령은 s3에 버킷을 만들고 버킷 목록을 조회한다.

aws --endpoint-url=http://localhost:4566 s3 mb s3://my-bucket
aws --endpoint-url=http://localhost:4566 s3 ls

비슷한 원리로 SQS나 다른 서비스들도 다 제어가 가능하다.

aws --endpoint-url=http://localhost:4566 sqs create-queue --queue-name my-queue
aws --endpoint-url=http://localhost:4566 sqs list-queues

특기할 점은, 계정 ID가 비어있다는 것이다. 계정 관리 시스템이 따로 존재하지 않는다.

그리고 API 수준에서 호환되기 때문에 다른 상황에서도, 프로그램 코드에서도 당연히 호환된다.
다음은 파이썬의 aws sdk를 사용해서 SQS에 메세지를 넣고, 읽고 지우는 간단한 코드다.

엔드포인트만 localstack으로 교체해서 날리면 일반 AWS와 거의 동등하게 동작한다.
