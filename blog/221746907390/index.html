[MongoDB] 도큐먼트: CRUD

  • 2019.12.23
  • ::
  • 1 min read
#MongoDB
[원본 링크]

도큐먼트는 MongoDB에서 컬렉션 내부에 존재하는 실제 데이터 묶음이다.
각 도큐먼트는 json 구조로 구성된다.


도큐먼트 삽입

도큐먼트, 그러니까 데이터의 삽입은 insertOne 메서드로 수행할 수 있다.
원하는 key:value의 묶음을 오브젝트로 묶어서 보내주면 된다.

그리고 여러개의 도큐먼트를 한번에 넣고싶다면, insert 메서드에 저것들을 배열로 묶어서 던져주면 된다.



도큐먼트 조회

도큐먼트들을 조회할땐 보통 find 메서드를 사용한다.
모든 도큐먼트를 조회하고 싶다면 빈 객체를 던져주면 된다.

_id는 알아서 해시값을 넣고 관리해주는 녀석이다. 신경쓰지 않아도 된다.

그냥 인자를 던져주지 않아도 다 찾아준다.

이외에 찾고싶은 특정 속성값의 도큐먼트가 있다면, 그 속성과 속성값을 객체에 넣어서 인자로 주면 된다.

잘 찾아준다.



도큐먼트 삭제

remove 메서드를 사용한다.
find처럼 해당하는 도큐먼트의 속성값들을 지정해서 객체로 넘겨주면 된다.

마찬가지로, 다 지우려면 빈 객체를 넘겨주면 된다.

다 잘 날라간걸 볼 수 있다.
아예 인자를 넘기지 않는건 안된다.
