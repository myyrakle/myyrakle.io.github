[Docker] 도커파일

  • 2020.12.04
  • ::
  • 2 min read
#Container
[원본 링크]

도커로 서버를 배포한다면 기본적으로 컨테이너에 도구나 컴파일러를 설치하고, 이런저런 설정을 해놓고, 소스도 받고, 구동도 시키는 등의 이런저런 작업들이 필요하다.

이런 작업을 자동화할 수 있게 해주는 것이 바로 도커파일이라는 것이다.
도커파일은 전용 스크립트를 사용하는데, 이 스크립트로 이런저런 명령어들을 작성해놓고 그걸로 컨테이너를 올리면 위에 언급한 작업들을 전부 자동으로 하게 해줄 수 있다!

도커파일을 작성하려면 파일을 가둬둘 디렉터리를 하나 만들어야 한다.

그냥 아무 디렉터리에서나 빌드를 시도하면 에러를 던지더라.

그리고 편집기로 스크립트 파일을 작성한다. 이름은 확장자 없이 Dockerfile이어야 한다.

간단하게 아파치 엔진만 설치하고 실행시키도록 구성해봤다.

일단 도커 스크립트는 전부 한줄 단위로 명령을 수행한다.
FROM centos는 centos 이미지를 바탕으로 이미지를 생성한다는 뜻이고
RUN은 그냥 컨테이너 내에서 명령을 수행하는 명령어다. yum을 업데이트하고 httpd를 설치하도록 했다. 여기서 유의할 점은 -y 옵션으로 전부 오케이를 누르게 해야한다는 것이다. 저거 없으면 선택이 안돼서 오류가 난다.

EXPOSE 80은 80 포트를 사용할 것임을 알린다.

그리고 CMD 명령은 컨테이너가 시작될때마다 실행될 명령을 지정하는 명령어다.
이 명령은 도커파일에서 딱 한번만 사용될 수 있다.
저기선 apachectl -DFOURGROUND로 아파치 서버가 자동으로 실행되도록 했다.

잘 만들었다면 해당 위치에서 docker build 명령으로 도커파일을 이미지로 빌드할 수 있다.
-t는 생성할 이미지 이름을 지정한다.
