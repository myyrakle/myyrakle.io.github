#Go

서비스 위버는 구글에서 직접 개발한 분산 애플리케이션용 서버 프레임워크다.
그 분야 최강자던 Erlang OTP와 포지션이 비슷하다고 볼 수 있다.

아직 초창기라 버전도 0.3로 낮고, 불안정할 수 있다.




기본 요소 설치

우선 프로젝트 만들고, mod init으로 초기화해준 다음에

전용 CLI를 설치한다.

go install github.com/ServiceWeaver/weaver/cmd/weaver@latest




프로젝트 구성

소스코드와 함께 Hello World 프로그램을 구성해보겠다.
HTTP 서버가 아닌 일반 CLI 프로그램이다.

아래는 문자열을 반전시키는 Reverse 함수를 weaver에서 요구하는 위버 컴포넌트의 형태에 맞춰 구현한 것이다.

package main

import (
	"context"

	"github.com/ServiceWeaver/weaver"
)

// Reverser component.
type Reverser interface {
	Reverse(context.Context, string) (string, error)
}

// Implementation of the Reverser component.
type reverser struct {
	weaver.Implements[Reverser]
}

func (r *reverser) Reverse(_ context.Context, s string) (string, error) {
	runes := []rune(s)
	n := len(runes)
	for i := 0; i < n/2; i++ {
		runes[i], runes[n-i-1] = runes[n-i-1], runes[i]
	}
	return string(runes), nil
}

일단은 스프링이나 Nest.js 등에서 다루는 DI, 서비스 단위와 대충 비슷하다고 보면 된다. 실제로는 아주 많이 다르지만, 일단 지금은 그렇게만 이해하자.

저렇게 정의해놓고 위버 시스템을 통해 꺼내서 쓰게 된다.

아래는 메인 소스코드다.

package main

import (
	"context"
	"fmt"
	"log"

	"github.com/ServiceWeaver/weaver"
)

func main() {
	// Initialize the Service Weaver application.
	ctx := context.Background()
	root := weaver.Init(ctx)

	// Get a client to the Reverser component.
	reverser, err := weaver.Get[Reverser](root)
	if err != nil {
		log.Fatal(err)
	}

	// Call the Reverse method.
	reversed, err := reverser.Reverse(ctx, "!dlroW ,olleH")
	if err != nil {
		log.Fatal(err)
	}
	fmt.Println(reversed)
}

정의했던 reverse 함수를 weaver 런타임을 통해 가져오고, 사용해서 올바르게 뒤집힌 텍스트를 출력하는 단순한 프로그램이다.

한번 실행해보자.

먼저 tidy로 종속성을 초기화해주고

go mod tidy

weaver CLI로 정의했던 위버 컴포넌트에 대한 코드를 생성해준다.

weaver generate .

그럼 이런식으로 "weaver_gen" 소스코드가 추가될 것이다.
저 내용이 중요한건 아니고, 이게 생성되어야 실행이 된다는 것만 알면 된다.

그럼 다음과 같이 실행할 수 있다.

간단하다.

여기서 만들고 사용해본 Reverser가 바로 위버의 근간을 이루는 "컴포넌트" 시스템이다.

그냥 저 코드만 보면 별거 아닌것처럼 보일 수 있지만, 컴포넌트는 위버를 설정하기에 따라서 그냥 함수로 동작할 수도, 아니면 별도 프로세스로 동작할 수도 있고, 별도의 머신에서 동작하도록 할 수도 있다!

하나의 프로젝트 단위를 별도의 머신에서 동작하게 하는 편리한 "분산 시스템"이 바로 위버가 지향하는 목표라고 할 수 있다.




HTTP 서버 만들기

이번에는 그냥 콘솔 찍고 죽는거 말고 제대로된 서버 애플리케이션을 만들어보겠다.

위버에서 딱히 대단한 구조를 제공하는건 아니고, API 정의는 기본 http 모듈로 하면 된다.

전달받은 name으로 인사를 해주는 "/hello"과 위에서 정의했던 reverse 함수로 뒤집힌 문자열을 반환해주는 "/reverse" API를 정의해봤다.

package main

import (
	"context"
	"fmt"
	"log"
	"net/http"

	"github.com/ServiceWeaver/weaver"
)

func main() {
	// Initialize the Service Weaver application.
	ctx := context.Background()
	root := weaver.Init(ctx)

	// Get a client to the Reverser component.
	reverser, err := weaver.Get[Reverser](root)
	if err != nil {
		log.Fatal(err)
	}

	// Get a network listener on address "localhost:12345".
	opts := weaver.ListenerOptions{LocalAddress: "localhost:12345"}
	listener, err := root.Listener("hello", opts)
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("hello listener available on %v\n", listener)

	// Serve the /hello endpoint.
	http.HandleFunc("/hello", func(writer http.ResponseWriter, request *http.Request) {
		name := request.URL.Query().Get("name")
		fmt.Fprintf(writer, "Hello, %s!\n", name)
	})

	// Serve the /reverse endpoint.
	http.HandleFunc("/reverse", func(writer http.ResponseWriter, request *http.Request) {
		reversed, err := reverser.Reverse(request.Context(), request.URL.Query().Get("text"))
		if err != nil {
			http.Error(writer, err.Error(), http.StatusInternalServerError)
			return
		}
		fmt.Fprintf(writer, "%s", reversed)
	})

	http.Serve(listener, nil)
}
