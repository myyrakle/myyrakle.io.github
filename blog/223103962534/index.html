소스코드를 모듈단위로 분리해서 관리하는 방법을 정리한다.



Root 디렉터리

terraform은 기본적으로 root 경로에 존재하는 .tf 소스코드들을 전부 읽어서 load하는 단순한 구조를 가진다.
그래서 따로 import를 하지 않더라도 동일한 디렉터리 내의 파일들끼리는 제한없이 접근하고 공유를 할 수 있다.




module 분리

그런데, 한 디렉터리에 온갖 코드를 다 몰아두면 장기적으로 관리도 어렵도, 보기도 불편해진다.
이럴때는 적당히 디렉터리를 분산해서 관리해야 하는데, 이걸 위해서 module이 존재한다.

기본적인 구성방법은 어려울게 없다.
디렉터리를 아무렇게나 잘 파고, 거기에다가 tf 파일들을 적절히 배치한다.

여기서 중요한 것은, 이러한 sub module들은 다른 모듈들과 독립적으로 존재한다는 것이다.

그래서 이걸 메인 소스에 포함시키려면 메인 모듈에 module 구문으로 명시적인 import를 해줘야 한다.

source가 서브모듈의 경로다.

이러고 메인 디렉터리 path에서 terraform init으로 초기화를 시키면, 서브모듈들을 메인모듈에 연동해줄 것이다.




모듈에 변수 전달하기

각 모듈들은 독립적으로 존재한다고 했었다.
그러면 뭔가 변수값을 전달하려면 어떻게 해야할까?
