코드래빗 (CodeRabbit)

  • 2025.10.09
  • ::
  • 2 min read
#LLM
[원본 링크]

코드래빗은 LLM 기반의 완전관리형 코드리뷰용 서비스다. Github 등에 통합된다.
현재 나와있는 도구들 중에서는 전반적인 퀄리티가 제일 괜찮은 편이다.

몇개월동안 써봤는데, 헛소리할 때도 많지만 사소한 실수들을 꽤 잘 잡아줘서 버그 방지에 효과적이다.




비용

무료 티어도 있고, 유료로 쓴다면 팀에 속한 인당 비용으로 나간다.

https://www.coderabbit.ai/pricing




기본 세팅

사용법은 그렇게 어렵진 않다. 적당히 가입하고 연동한 뒤에

연동할 repo만 적절히 지정해주면 된다.


그러면 바로 쌓여있는 PR에 리뷰를 실행할 수도 있고


대시보드로 진입할 수도 있다.
여기서 Repo 연동 관리나 권한관리 등을 수행할 수 있다.




리뷰 트리거

리뷰를 가장 간단하게 트리거하는 방법은, PR 코멘트로 멘션을 거는 것이다.

이러면 리뷰를 시작한다.

그리고 조금 기다리면 이런 식으로 리뷰를 생성해준다.

PR을 날릴때 자동으로 트리거되게 할 수도 있다.
기본적으로 base branch로의 PR은 자동 트리거가 되고, 추가 설정도 된다.

요약하고 설명한 뒤에 문제가 있다면 코멘트도 달아준다.
위처럼 다이어그램까지 만들어주는데, 가끔 유용할 때도 있다.

코멘트는 이런 식으로 특정 코드 영역을 집어서 잡아준다. 개선 방법까지 잘 알려주는 편이다.


기본적인 상호작용도 된다.


뭐 이상한 것도 추가했더라




리뷰 규칙 정의

LLM 관련된 모든 시스템들이 그렇듯이 프롬프트 커스텀을 포함한 세부 조정은 필수다.

기본적으로는 Web UI에서 직접 설정할 수 있다.

리뷰에 쓰는 언어 같은 기본적인 것부터 해서 통합 관리 같은 것까지를 전반적으로 제어할 수 있다.

여기에서 설정한 옵션은 전역적으로 적용되며, 프로젝트에 config 파일이 있다면 그게 높은 우선순위로 적용된다.
프로젝트별 config 파일 설정은, 프로젝트 루트 경로에 .coderabbit.yaml 파일을 두면 된다.
