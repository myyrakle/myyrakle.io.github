[Meilisearch] Typo tolerance

  • 2023.10.15
  • ::
  • 1 min read
#Meilisearch
[원본 링크]

Typo tolerance는 사용자 수준에서 충분히 발생할 수 있는 오타에 대응하기 위한 기능이다.
잘못 쓰면 오히려 검색의 정확도를 떨어뜨릴 수도 있기 때문에 신중하게 사용해야 할 것이다.

elasticsearch 등의 다른 검색엔진에서는 fuzz라는 용어를 쓰는 것 같았다.

설정은 이렇게 할 수 있다.

따로 옵션을 주지 않으면 Meilisearch는 5자 이상을 포함하는 검색어에 대해 1개의 오타를 허용하고, 용어 길이가 9자 이상인 경우 최대 2개의 오타를 허용한다.

직접 옵션을 조정하려면 이렇게 주면 된다.

아무튼 저렇게 활성화를 해두면 해당 인덱스에서 발생하는 검색에 대해서는 오타 보정 검색이 들어간다.

이건 elasticsearch fuzz보단 정확도가 좀더 나은 것 같다.
elasticsearch는 한글이 들어있더라도 무조건 "한글자" 단위로 fuzz를 돌려서 정확도가 구렸었는데, 이거는 초성-중성-종성 쪼개서 어떻게 처리하는 로직이 있긴 한 것 같다.
