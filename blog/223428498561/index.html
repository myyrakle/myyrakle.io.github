[Rust] Embedded Rust: 기초

  • 2024.04.26
  • ::
  • 2 min read
#Rust
[원본 링크]

임베디드 환경에서 Rust를 사용하는 기본적인 원리와 방법을 정리해본다.

임베디드라고 하면 주구장창 C만 쓸 것 같지만, 의외로 Rust를 실제로 활용하는 경우도 드물게나마 있다.
환경도 그럭저럭 나쁘지 않은 편이다.

C/C++은 지저분해서 싫고, 모던한 언어로 생산성과 안정성을 잡고 싶다면 현재로서 Rust 외에는 마땅한 대안이 없는 것 같다.




no_std

보통 Rust에서 일반 PC 환경으로 작업할 때는 std 표준 라이브러리들을 적극적으로 활용한다.

하지만 표준 라이브러리들을 활용할 수 없는 수준의 환경도 존재한다. MCU 수준의 임베디드 환경이 대표적이다.
그럼에도 불구하고 Rust는 이러한 임베디드 환경을 부분적으로 지원하며, 이에 대한 사용 수요도 존재한다.

그러면 표준 라이브러리를 사용할 수 없는 상황에서 코드를 어떻게 작성해야 할까?
다행히도 Rust는 std가 사용 불가능한 환경에서도 사용할 수 있는 core 라이브러리가 별도로 존재한다.
core를 사용하면 제한적이나마 Rust의 장점들을 활용할 수 있다.

no_std 환경에서는 꽤 많은 기능을 사용할 수 없다.

  1. std 라이브러리를 그대로 사용할 수 없고,
  2. main 함수 호출 전의 초기화가 지원되지 않으며
  3. 스택오버플로 방지가 비활성화되며
  4. HashMap, HashSet 같은 난수 기반 컬렉션도 사용 불가능하다.

힙 할당은 사용할 수 있으나 설정하기에 따라 다르다. allocator 설정이 필요할 수 있다.

no_std 모드로만 컴파일하려면 최상단에 #![no_std]를 지정하면 된다.

그러면 바로 손발이 묶일 것이다.




core 라이브러리

no_std 환경에서는 core 라이브러리밖에 사용할 수 없다.
core는 libc 종속성이 없는 Rust만의 고유한 핵심 라이브러리 구성요소다.

사용 자체는 어렵지 않다. 기본 구성요소이기 때문에 별도의 설정 없이도 바로 사용할 수 있다.

아래 문서에서 사용 가능한 전체 목록을 확인할 수 있다.
https://doc.rust-lang.org/core/
정말 기본만을 갖췄기 때문에, I/O 지원이 전무하고, 힙 할당과 연관된 기능도 전혀 없다.

I/O나 힙 할당 같은게 필요하다면 C FFI를 통해서 처리를 해야할 것이다.




타겟

일반적인 임베디드 환경이 그렇듯이, 보통 개발은 일반 PC에서 하고 결과물을 임베디드 장비에 올려서 실행을 하게 될 것이다. 그러려면 타겟을 설치해서 그걸로 컴파일을 시켜야 한다.
러스트에서 지원되는 타겟은 꽤 많다.
