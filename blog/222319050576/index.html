[AWS] Dynamo DB: 시작

  • 2021.04.22
  • ::
  • 2 min read
#AWS
[원본 링크]

다이나모DB는 AWS 인프라에서만 전용으로 제공하는 키-값 기반의 NoSQL 데이터베이스 시스템 및 서비스다.
NoSQL답게, 단순 쓰기와 읽기는 그 어떤 RDB보다 훨씬 빠르다. 대신 복잡한 관계 Join이나 정렬 등을 사용할 경우에는 훨씬 비효율적일 수 있다.

서버리스 시스템이라 요청이 많든 적든 신경쓸 필요가 없고, 성능도 아주 좋은 편이다.
게다가 기본요금이 없이 딱 쓴만큼만 요금이 나간다.

그 요금이 싼 편이 아니긴 하지만, 아무튼 편리하고 비용관리가 쉽다는 점이 참 메리트가 크다고 하겠다.

돈이 나가는게 싫다면 아마존에서 로컬로 사용할 수 있는 실행형 프로그램을 제공한다.
설치는 여기서 할 수 있다.
https://docs.aws.amazon.com/ko_kr/amazondynamodb/latest/developerguide/workbench.settingup.html



AWS 콘솔에서 테스트

하지만 나는 제대로 사용해보고 싶기 때문에, 그냥 바로 서비스를 사용해서 테스트해보겠다.
테이블 만들기를 클릭한다.

나는 아래와 같이 만들었다.
테이블 이름 잘 정해주고, 기본 키값을 설정한다.
그리고 요금이 정말 쓴만큼만 나가도록 온디맨드 모드로 설정한다.

이외에는 별로 지정할게 없다.
그냥 키-값 기반의 NoSQL이다보니 기본 키 말고는 값을 자유롭게 넣고 뺄 수 있는 시스템이기 때문이다.

그럼 테이블이 생성될 것이고. 자유롭게 데이터를 넣거나 읽을 수 있다.




람다에 연결하기

람다를 대충 만들고, 권한에서 실행 역할을 들어간다.

다이나모를 람다에서 제어할 수 있도록 해보겠다.

정책 연결 누르고

여기서 다이나모에 대한 접근권한을 지정해주면 된다.

일단은 이것저것 다 테스트해볼 거니까 FullAccess로 해보겠다.
경우에 따라서 ReadOnly로 줘도 되고...




돌려보기

그럼 밑작업은 대충 됐으니 이제 돌려보자.
대강 아래와 같이 짜면 된다.
그냥 테이블 목록을 전체조회하는 코드다.

const AWS = require('aws-sdk');

AWS.config.update({
    region: 'ap-northeast-1', // 리전 정보
    endpoint: "http://dynamodb.ap-northeast-1.amazonaws.com"
});

const dynamodb = new AWS.DynamoDB();
const docClient = new AWS.DynamoDB.DocumentClient();

exports.handler = async (event) => {
