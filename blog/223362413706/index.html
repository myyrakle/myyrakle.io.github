iced는 Rust 환경의 Desktop GUI 라이브러리 중 하나다. 특별히 잘나간다고 하기는 어렵고, 그냥 그 중 하나다.
크로스플랫폼이다.

현재는 말라죽은 프론트엔드 프레임워크인 Elm을 베낀 구조가 가장 큰 특징이다.
그림으로 그리면 이렇다고 한다.

근데 이거까지 다 알 필요는 없는 것 같다.




기본 프로그램 작성

나는 뭘 하든 일단 돌아가는 것부터 보는게 중요하다고 생각한다.

먼저 디펜던시를 추가하고

다음 코드를 넣어서 실행한다.

use iced::widget::{button, column, text};
use iced::{Alignment, Element, Sandbox, Settings};

pub fn main() -> iced::Result {
    Counter::run(Settings::default())
}

struct Counter {
    value: i32,
}

#[derive(Debug, Clone, Copy)]
enum Message {
    IncrementPressed,
    DecrementPressed,
}

impl Sandbox for Counter {
    type Message = Message;

    fn new() -> Self {
        Self { value: 0 }
    }

    fn title(&self) -> String {
        String::from("Counter - Iced")
    }

    fn update(&mut self, message: Message) {
        match message {
            Message::IncrementPressed => {
                self.value += 1;
            }
            Message::DecrementPressed => {
                self.value -= 1;
            }
        }
    }

    fn view(&self) -> Element<Message> {
        column![
            button("Increment").on_press(Message::IncrementPressed),
            text(self.value).size(50),
            button("Decrement").on_press(Message::DecrementPressed)
        ]
        .padding(20)
        .align_items(Alignment::Center)
        .into()
    }
}

하나씩 뜯어보자.

Counter 구조체는 저 앱 자체의 상태를 보관하는 상태 저장 구조체다.
여기서는 실질적으로 저 창 전체를 의미하기도 한다.


그 다음은 메세지다. Elm은 모든 동작을 "이벤트"를 통해서 연결하고 상호작용하도록 하는데, Elm을 베낀 iced도 그런 방법을 취한다.
보일러플레이트가 늘어나서 코드라인이 길어지기도 하지만, 오히려 코드가 정돈되고 깔끔해지는 부분도 있다.

자, 그럼 이제부터 본격적인 기능 구현이다. 샌드박스는 최상위 "창" 자체의 동작을 정의하는 트레잇이다.
이걸 구현해야 창을 띄울 수 있다.

먼저 기본적인 생성 함수부터 만들어준다. 이건 특별할게 없다.


이건 창 제목이다.


이건 이벤트가 날라왔을때 어떻게 동작할지를 정의한다.
"Increment" 이벤트가 날라오면 카운터 값을 늘리고, "Decrement" 이벤트가 날라오면 줄인다.
그럼 이건 어떻게 호출되느냐?


view에서 UI를 정의함과 동시에 액션 또한 정의된다.
버튼을 2개 생성하고 각 버튼이 눌릴때마다 이벤트가 전송되도록 했다. 이벤트가 발생하면 update 함수가 그걸 받아서 처리하고, 다시 렌더링을 하게 되는 것이다.

그 외에는 레이아웃 수준 제어다. 패딩 주고, 정렬하고...
다른 프레임워크들과 크게 다르지 않다.

그리고 저 Counter 구조체는 Sandbox를 구현함으로 창으로 띄울 자격을 얻게 되었다.

run 함수로 실행하면 창이 실행되는 것이다.

이런 느낌으로 쓰는 것이다.
