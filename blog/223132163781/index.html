consul을 사용하는 주 목적 중 하나는 Service Discovery다.
특정 서버들의 IP를 consul의 Service Registry에 등록하고, 그걸 클라이언트가 접속할때마다 꺼내서 호출할 수 있도록 하는 것이다.




서버 에이전트 실행 (with consul CLI)

서버의 IP 주소를 consul에 등록하려면, 우선 consul cli를 설치해야 한다.

다음과 같이 서버용 Pod를 먼저 띄운 다음에

apiVersion: v1
kind: Pod
metadata:
  name: server
  labels: 
    app: server 
  annotations:
    consul.hashicorp.com/connect-inject: 'true'
spec:
  containers:
  - name: server 
    image: myyrakle/node-server-for-test

접속해서

consul CLI를 설치한다.

그럼 아래와 같은 명령어를 사용해서 서비스를 등록할 수 있다.

consul services register -name=서비스명 -address=IP주소 -port=포트 --http-addr=웹UI주소
consul services register -name=test-server -address=$(hostname -I) -port=80 --http-addr=34.123.54.86

그러면 콘솔 UI에도 서비스가 연동이 될 것이고

IP를 확인할 수 있다.

여기서는 직관적인 확인을 위해 컨테이너에 직접 들어가서 작업을 했는데, 실제로 프로덕션에 사용할 때는 Docker Entrypoint에서 자동으로 register 실행을 하게 해야 한다.
아니면 sidecar 컨테이너를 별도로 구성해서 거기에만 consul agent를 설치하는 것도 좋은 방법일 것이다.




Service IP 접속정보 꺼내오기 (Service Discovery)

등록된 서비스의 정보를 꺼내오려면 Restful API를 사용할 수 있다.

.../v1/catalog/service/서비스명

여기서 ServiceAddress와 ServicePort를 꺼내와서 요청할때 사용하면 되는 것이다.




Service Registry 등록 해제하기

CLI에서 deregister 명령을 사용하면 서비스 정보를 다시 내릴 수 있다.
