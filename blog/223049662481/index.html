[Linux] select와 poll. 그리고 epoll

  • 2023.03.20
  • ::
  • 3 min read
#Linux
[원본 링크]

select와 poll, epoll은 Linux에서 제공되는 다중 I/O 처리모델이다.

사실 이 linux api를 바로 쓸일은 거의 없을것이다. 보통 고급언어에섷통신 모듈을 사용하면 이런것들은 알아서 구현을 해놓기 때문이다.

하지만 사용하고 있는 통신 모듈이 어떤 방식을 사용해서 구현되었는지 분간하고, 어떤 한계점이 있는지는 알아볼 수 있어야한다.




select

select는 싱글스레드로 다중 I/O를 처리하는 가장 단순하고 오래된 기법이다.

작은 규모에서는 잘 동작하나, 한계가 많아서 대규모 처리는 어렵다.

기본적인 원리는 이렇다.

리눅스에서 파일처리나 소켓통신은 전부 파일 디스크립터(FD: File Descripter)를 통해 관리된다.

그래서 그냥 무한루프돌면서 관심있는 파일 디스크립터(이하 FD)들의 상태를 계속 검사하는 것이다.
그렇게 검사하다가 FD의 상태가 변경되면 완료나 실패가 된 것이니, 그걸 전달해줘서 처리하도록 한다.

https://applefarm.tistory.com/144

단점은, 파일 디스크립터의 최대 개수가 정해져있어서 그 이상의 동시처리가 불가능하단 것이다.
내부 파일 디스크립터 관리에 fd_set이라는 정적배열을 사용하는데, 보통은 길이가 1024라서 1024개까지만 처리가 가능하다. 왜 이렇게 만들어놨지...?

그리고 매번 파일 디스크립터를 계속 조회하는 것에서 성능 낭비가 발생한다는 것도 단점이다.

C 헤더 sys/select.h에 select 등의 함수를 사용하면 select 기능을 사용할 수 있다.

관련 함수 (sys/select.h)
select
FD_ZERO
FD_SET
FD_CLEAR
FD_ISSET




poll

poll은 select에서 파일디스크립터의 개수가 제한되어있다는 단점을 해소한 기법이다.

정적배열을 사용하지 않고 pollfd라는 구조체를 통해 동적할당을 해서 파일디스크립터 등의 정보를 관리하기 때문에, select와 같은 문제는 발생하지 않는다.
그 외에도 자잘한 부분에서 select보다는 나은 성능을 보인다.

대규모 처리가 가능하긴 하나, 마찬가지로 관심있는 파일디스크립터를 계속 무한루프돌며 검사한다는 점은 다르지 않아서 성능의 저하가 있을 수 있다.




epoll

epoll은 고성능 I/O 처리를 위한 매커니즘이다. windows에 iocp가 있다면, 리눅스에는 이게 있다.

epoll은 이벤트 기반으로 동작해서 관심이 있는 파일 디스크립터만 검사를 한다. 커널이 FD를 대신 관리해주는 것이다.

관심있는 I/O의 FD를 epoll 인스턴스로 만들어주고, wait을 하면. 값이 들어왔을때 wait이 종료되고 그 결과를 받아올 수 있는 구조다.

https://medium.com/@avocadi/what-is-epoll-9bbc74272f7c
epoll_create, epoll_ctl, epoll_wait 함수를 사용해 구현할 수 있다.

단점이랄건 딱히 없는것같다.




epoll: 사용법

  1. epoll 인스턴스 생성
    epoll_create 또는 epoll_create1 함수를 사용하여 epoll 인스턴스를 생성한다. 이 함수는 성공 시 epoll FD를 반환한다.
    이 자체적인 FD를 통해 처리를 제어하게 된다.
﻿int epoll_fd = epoll_create1(0);
  1. 관심 있는 FD 등록
    epoll_ctl 함수를 사용해서 관심 있는 FD를 등록한다.
    이 함수는 epoll FD, 상태값, 관심 FD, 이벤트를 인수로 받는다.
    관심 FD에 뭔가 들어오면 이벤트가 들어오게 하는 것이다.
﻿struct epoll_event event; // 이벤트 저장소
event.events = EPOLLIN; // 관심 이벤트를 읽기 준비 완료로 설정
event.data.fd = fd; // 대상 파일 디스크립터
epoll_ctl(epoll_fd, EPOLL_CTL_ADD, fd, &event); // epoll 인스턴스에 파일 디스크립터 추가
  1. 이벤트 대기
    epoll_wait 함수를 사용하여 이벤트가 발생할 때까지 대기한다.
    이 함수는 뭔가 들어와서 대기가 끝나면 발생한 이벤트 개수를 반환한다.
struct epoll_event events[MAX_EVENTS];
int num_events = epoll_wait(epoll_fd, events, MAX_EVENTS, -1); // 들어올때까지 대기
  1. 이벤트 처리
    epoll_wait 함수로부터 반환된 이벤트를 처리한다. 예를 들어, 읽기 이벤트가 발생한 경우 해당 파일 디스크립터에서 데이터를 읽을 수 있다.
﻿for (int i = 0; i < num_events; i++) {
  int event_fd = events[i].data.fd;

  if (events[i].events & EPOLLIN) {
    // 읽기 이벤트가 발생한 경우 처리
  }
}
  1. 정리
    사용이 끝난 후, epoll 인스턴스의 파일 디스크립터를 닫는다.
﻿close(epoll_fd);
