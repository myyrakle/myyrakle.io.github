#라이브러리
[원본 링크]

이런저런 기능들이 추가되었다.




HTTP1 keep alive 기능 플래그 추가

rupring의 기본 부팅 모드는 http1이다.
그리고 keep-alive는 기본적으로 비활성화되고, true를 아래와 같이 전달한다면 keep-alive가 활성화된다.




HTTP2 및 TLS 기능 추가

HTTP2 모드로 부팅하는 기능과 TLS 설정 기능이 추가되었다.
HTTP2를 쓰려면 어차피 tls도 같이 되어야 하는거라 쓴다면 보통 세트로 쓰게 될 것이다.

TLS를 활성화하려면 features에 tls를 추가하고

플래그에 인증서와 키를 넣어주면 된다.

그리고 http2를 활성화하려면 enabled 플래그만 넣어주면 된다.
http2 플래그가 활성화되면 http1 기반의 플래그들은 전부 비활성화된다.

그렇게 실행하고 찔러보면, 잘 동작하는 것을 확인할 수 있을 것이다.




Request Timeout 구현

글로벌로 적용되는 API 수준 타임아웃 옵션을 추가했다.

이러면 아무리 오래걸려도 3초 지나면 강제로 끊어버리는 것이다.

전달하지 않는다면 타임아웃이 설정되지 않는다. 무한하게 실행된다.




쿠키 관련 제어기능 추가

별건 아니고, 쿠키 핸들링을 좀 편하게 할 수 있게 하기 위한 기능이 좀 추가된다.

Request로 들어온 Cookie 정보는 라이브러리 수준에서 자동으로 파싱해서 전달해준다.

그리고 Response의 add_cookie 메서드로 쿠키 세팅을 좀 더 간편하게 할 수 있다.
