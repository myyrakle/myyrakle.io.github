[TypeORM] Data Mapper 패턴

  • 2021.03.30
  • ::
  • 3 min read
#개발노트
[원본 링크]

Data Mapper 패턴은 typeORM에서 주로 사용하는 패턴 중 하나다.
Active Record 패턴에 비하면, 좀더 규모가 있는 경우에 사용하기 적합하다고 한다.

typeORM에서 자체 제공하는 Repository라는 기능을 사용한다. 이 Repository에는 일반적으로 필요한 다양한 기능들이 기본구현되어 제공된다.
잘 갖다 쓰기만 하면 된다.

테스트에 사용한 테이블 엔티티는 다음과 같다.

그리고 엔티티에 대응되는 레포지토리 클래스를 정의한다.
EntityRepository 데코레이터를 달고, Respository<엔티티>를 상속받으면 된다.

여기서는 메서드를 따로 정의하지 않았지만, 필요하다면 Active Record처럼 메서드를 추가정의해서 사용해도 좋다.
이렇게만 해둬도 Repository에서 기본으로 만들어서 주는 쿼리 메서드들을 사용할 수가 있다.

이 중 대표적인 것들만 몇가지 골라서 정리해보겠다.
자세한 것은 공식문서를 참조하길 바란다. 그렇게 친절하진 않다만.
https://typeorm.delightful.studio/classes/repository_repository.repository.html




save

삽입은 save 메서드를 사용하면 간단하게 처리할 수 있다.
엔티티 데이터를 인자로 넘겨주면 된다.

이 함수는 그냥 insert만 하는 게 아니라, PK가 충돌한다면 update를 수행한다는게 특징이다.

위처럼 그냥 새 값일 경우엔 트랜잭션만 걸고 insert를 수행한다.

PK가 걸리면 select로 중복을 체크한 후 update를 수행한다.

여러 값을 삽입한다면 엔티티의 배열을 넣으면 된다.

삽입할 것과 수정할 것이 뒤섞여도 잘 처리해준다.




find 함수들

탐색 함수로는 find, findOne 등의 함수가 제공된다. find는 다중 행, findOne은 단일 행을 가져온다는 것을 빼면 거의 동일하다.

그리고 다음과 같이 오브젝트로 다양한 옵션을 줄 수가 있다.
가장 자주 쓸만한 옵션은 역시 where다.

위처럼 조건으로 비교할 컬럼과, 값을 쌍으로 주면 동등 비교를 해준다.

연산자도 사용할 수 있다. LessThan 등의 다양한 연산자 함수가 제공된다.

지원되는 연산자 목록은 아래 문서를 참조하길 바란다.
https://github.com/typeorm/typeorm/blob/master/docs/find-options.md

이외에 정렬은 order로

offset&limit는 skip&take로 처리할 수 있다.

특정 컬럼만 골라오고 싶다면 select 옵션을 사용해서 컬럼명을 문자열로 넣어준다.




update

update는 말 그대로 수정만 해주는 함수다.
첫번째 인자로는 조건절에 대한 오브젝트, 두번째 인자로는 수정할 값의 오브젝트를 넘겨준다.

첫번째 인자의 조건절은 select에서 where에 사용했던 것과 거의 동일하게 주면 된다.




insert

insert는 단순히 삽입만을 수행하는 함수다.
그냥 삽입할 값을 넣어주면 된다.
