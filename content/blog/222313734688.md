외래키는 테이블에서 다른 테이블의 값을 참조할 때, 이를 검증하기 위해 사용하는 기능이다.  

쓰지 않아도 참조를 하는 것 자체에는 문제가 없지만, 정합성을 강조할 수 있고, 참조대상값이 바뀌는 것에 대해서 무언가 확실한 보장을 할 수 있다는 장점이 있다.  

성능 때문에 이걸 써야 하는지에 대해 사람들마다 좀 의견이 갈리는 부분이 있는데, 일단 그런건 차치하고 기능적인 부분부터 다뤄보겠다.  

게시판을 구현하기 위해 사용자와 포스트에 대한 테이블을 설계한다고 가정하겠다.  
외래키를 고려하지 않고 짠다면 대충 아래와 같은 구조가 될 것이다.  

```javascript
-- 사용자 테이블
create table tb_user
(
    id serial, 
    name text,
    primary key(id)
);

-- 사용자가 작성한 포스트 테이블
create table tb_post
(
    id serial,
    title text,
    content text,
    writer_id int,
    primary key(id)
);
```
여기서 tb_post의 writer_id 컬럼이 바로 tb_user의 id를 가리키는 사실상의 외래키 컬럼이다.  
따로 외래키 설정을 하지 않고 그냥 그때그때 조인해서 써도 사용에는 문제가 없다.  

하지만 중간에 가리키고 있는 tb_user의 값이 갑자기 삭제되거나 변경돼도 직관적으로 파악하고 대응할 수 없다는 단점이 있다.  
tb_user의 값을 삭제할때마다 직접 이걸 참조하는 tb_post 등의 테이블에서 writer_id 값을 어떻게 할지 수동으로 처리해줘야 하는 것이다.  
게다가 참조키값으로 존재하지도 않는 이상한 키값을 넣는 것도 막을 수가 없다.  

외래키 기능은 이에 대한 편의를 제공해준다.  
외래키의 선언은 다음과 같이 할 수 있다.  
**foreign key(현재 테이블 컬럼) references tb_user(참조할 테이블 컬럼)**이다.  

```javascript
-- 사용자 테이블
create table tb_user
(
    id serial, 
    name text,
    primary key(id)
);

-- 사용자가 작성한 포스트 테이블
create table tb_post
(
    id serial,
    title text,
    content text,
    writer_id int,
    primary key(id),
    foreign key(writer_id) references tb_user(id)
);
```

## ON DELETE 속성
이건 외래키로 삭제에 대한 제약이나 트리거를 걸어주는 기능이다.  

예시 테이블의 경우를 들면, tb_user 값을 delete할때, 해당 값을 참조하는 tb_post가 있을 때 어떻게 반응할지를 정의하는 것이다.  

Cascade, Set null, Set default, Restrict, No Action의 다섯가지 옵션을 제공하고, 기본값은 No Action이다.  

cascade는 tb_user가 삭제되면 참조하는 tb_post도 전부 삭제하는 것이고,  
set null은 writer_id를 null로 설정하는 것,  
set detault는 writer_id를 디폴트값으로 설정하는 것,  
restrict는 그낭 tb_user의 삭제를 막는 것이다.  

restrict의 경우에만 한번 테스트를 해보겠다.  
우선 on delete를 적용해준다.

## ON UPDATE 속성
이것도 ON DELETE와 맥락이 같고, 제공되는 속성도 Cascade, Set null, Set default, Restrict, No Action 다섯개로 동일하다. 기본값은 No Action이다.  

이건 참조하고 있는 컬럼값이 바뀔 경우에. 그러니까 이 경우엔 tb_user의 id값이 바뀔 경우에 tb_post의 writer_id값을 어떻게 할지를 지정해주는 기능이다.  

cascade는 그냥 바꾼 값으로 tb_post에서도 다시 똑같이 바꿔주고,  
set null은 writer_id를 null로 설정하는 것,  
set detault는 writer_id를 디폴트값으로 설정하는 것,  
restrict는 그낭 변경을 막는 것이다.  

이번엔 cascade로 한번 돌려보겠다.
