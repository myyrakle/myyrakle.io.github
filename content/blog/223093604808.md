Bigtable은 초대형 스토리지 데이터베이스 서비스다.  
Hbase 기반으로 만들어졌고, 페타바이트 단위의 무식한 데이터를 쌓아놓는데 최적화되어있다.  

Apache HBase와 Cassandra의 대안 중 하나고, 마이그레이션도 잘 지원하는 편이다.  

보통 bigtable에 무식하게 쌓다가 분석이 필요하면 데이터를 꺼내서 mapreduce를 돌리거나, bigtable->bigquery로 데이터를 옮겨넣어서 쓰는 방법을 많이들 취하는 것 같다.  

다만 자체적으로 MapReduce 기능을 제공하지는 않는다. 자체적으로 클라이언트 수준에서 구현해서 써야한다.

## 테이블 만들기

bigtable도 여타 DB들과 마찬가지로 테이블 단위로 데이터 구조를 관리한다.  

![](/images/img_08839229657c5194.png)
테이블 만들기 누르고  

<br>

![](/images/img_0f90797b2f8232c1.png)
이름을 짓고, 컬럼 패밀리들을 적절히 정의한다.  
제대로 써먹으려면 컬럼 패밀리를 최소한 하나 이상은 둬야한다.  

컬럼 패밀리는 쉽게 말해 컬럼들의 그룹이다.  
Bigtable에서 테이블은 하나의 기본키와 여러개의 컬럼패밀리로 이루어진다.  

![](/images/img_f38f70ac7fce1da7.png)
각 컬럼 패밀리에는 컬럼이 여러개 들어갈 수 있다.

## CLI로 다뤄보기

bigtable은 cbt라는 전용 CLI 도구를 지원한다.  
없다면 gcloud로 설치하면 된다.  

```javascript
gcloud components install cbt
```
cloud shell을 통해 쓴다면 기본으로 깔려있다.  

인스턴스 목록을 조회하는 명령은 다음과 같다.  

```javascript
cbt listinstances
```

![](/images/img_874640b9dd91534c.png)

그리고 특정 인스턴스의 테이블 목록을 조회하려면, ls 명령을 쓰면 된다.  

```javascript
cbt -instance=인스턴스명 ls
```

![](/images/img_eef67adb984ebb4f.png)

테이블 내의 컬럼 정보를 조회하려면 이렇게 한다.  

```javascript
cbt -instance=인스턴스명 ls 테이블명
```

![](/images/img_6c4af235067fa1a4.png)

데이터를 삽입하려면 set 명령을 사용하면 된다.  

```javascript
cbt -instance=인스턴스명 set 테이블명 키값 컬럼패밀리:컬럼=값

예) 
cbt -instance=bigtable-test set orders 1 test:price=5000 test:product_id=12 test:buyer_id=1 
cbt -instance=bigtable-test set orders 2 test:price=7000 test:product_id=15 test:buyer_id=1 
cbt -instance=bigtable-test set orders 3 test:price=3000 test:product_id=11 test:buyer_id=2 
cbt -instance=bigtable-test set orders 4 test:price=1000 test:product_id=13 test:buyer_id=3
```

![](/images/img_b4c1a8ca428f7cbc.png)

읽을때는 read 명령을 쓴다.  

```javascript
cbt -instance=인스턴스명 read 테이블명

cbt -instance=bigtable-test read orders
```

![](/images/img_a4f1a6baac2a7118.png)

지우려면 deleterow 명령을 쓰면 된다.  

```javascript
cbt -instance=인스턴스명 deleterow 테이블명 키
```

![](/images/img_af6b8a83f3134e7b.png)
