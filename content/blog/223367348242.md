DB를 가져다가 실제 작업을 하다보면, 마이그레이션용 스크립트를 작성해서 한번에 여러 DDL을 날릴 경우가 매우 많다.  

근데 이게 이래저래 치다보면 부분적으로는 이미 적용되어있어서 이미 있는걸 또 만드려고 시도하다 에러가 나서 상당히 귀찮은 경우가 발생할 때가 잦다.  
IF NOT EXISTS 절은 이런 상황에 활용하기에 적합한 DDL 전용 조건절이다.  

예를 들어, 이미 있는 테이블을 또 만들려고 시도하면  

```javascript
CREATE TABLE FooBar (
  id serial8 PRIMARY KEY, 
  name TEXT
);
```

![](/images/img_a884415138936678.png)
오류가 발생해서 이후의 쿼리들이 다 끊긴다.  

하지만 IF NOT EXISTS 절을 사용하면 충돌이 발생해도 무시하고 지나간다.  

```javascript
CREATE TABLE IF NOT EXISTS FooBar (
  id serial8 PRIMARY KEY, 
  name TEXT
);
```

![](/images/img_581dc26babec570d.png)

이건 컬럼 추가나  

```javascript
ALTER TABLE FooBar ADD COLUMN name TEXT;
```

![](/images/img_b0da102e5c1021ec.png)

```javascript
ALTER TABLE FooBar ADD COLUMN IF NOT EXISTS name TEXT;
```

![](/images/img_0da5dade0878ea61.png)

인덱스 생성 등에도 동일하게 활용할 수 있다.  

![](/images/img_8c9f1aee19d53869.png)

## 추가: IF EXISTS 절

IF NOT EXISTS가 존재하지 않을 경우에만 실행하도록 하는 조건절이라면, IF EXISTS는 존재할 때만 실행하도록 하는 조건절이다.  
DROP 같은 제거 명령에 사용하기에 용이하다.
