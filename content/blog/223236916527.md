동의어는 검색 품질을 결정하는 또 하나의 중요한 요소 중 하나다.  

예를 들어, 아래와 같은 데이터가 있다고 가정하겠다.  

![](/images/img_c25eb39e3d4d9990.png)
우수한 검색엔진은 구찌로 검색하든, GUCCI로 검색하든, 저 2가지 데이터가 다 검색에 걸릴 수 있어야 할 것이다.  
당연하지 않겠는가? 텍스트 자체는 다르지만 사실상 같은 이름이 동의어이니 말이다.  

이걸 달성하기 위해서는 인덱스에 동의어 설정을 달아줘야 한다. 동의어는, 쉽게 말하면 단순한 치환 기능이다.  
구찌라는 단어가 들어왔을때 그걸 GUCCI로 바꿔주고, GUCCI라는 단어가 들어왔을 때 그걸 구찌로 바꿔주자는 것이다.  

elasticsearch는 동의어 바꾸는게 되게 짜증나게 되어있는데, 이건 좀 편해서 좋더라  

아래 코드는 구찌=>GUCCI 동의어 매핑을 설정하는 코드다.  

![](/images/img_cf31a3020b9fdb3e.png)

```javascript
const { MeiliSearch } = require("meilisearch");
const movies = require("./movies.json");

async function main() {
  const client = new MeiliSearch({
    host: "http://localhost:7700",
    apiKey: "q1w2e3r4asdf",
  });

  await client.index("products").updateSynonyms({
    구찌: ["GUCCI"],
  });

  const result = await client.index("products").getSynonyms();
  console.log(result);
}

main();
```

그럼 이제 구찌로 검색했을 때  

![](/images/img_007523c331376cd1.png)
GUCCI도 함께 뜰 것이다.  

근데 문제는  

![](/images/img_1ce895606e6b3af7.png)
동의어는 단방향 매핑이란 것이다.  
그래서 GUCCI를 검색했을 때도 구찌가 뜨게 하려면  

이렇게 한땀한땀 다 쳐줘야 한다.  

![](/images/img_0fdc972fde40cfcb.png)

![](/images/img_6c16c7c12115f35f.png)
