Publication은 PostgreSQL에서 자체적으로 제공하는 데이터 복제용 기능이다.  
WAL을 그대로 스트리밍할 수 있게 해주며, Replica 구성을 위한 복제나 CDC 기능 등에 활용할 수 있다.  

<br>

<br>

<br>

## 제한

DML, INSERT/UPDATE/DELETE에 대해서만 유효하게 동작한다. DDL 등에 대해서는 반응할 수 없다.  

<br>

<br>

<br>

## Publication 만들기

복제 경로를 뚫으려면 먼져 publication이라는 일종의 채널을 만들어야 한다.  
다음과 같이 만들 수 있다.  

```javascript
CREATE PUBLICATION 이름 FOR ALL TABLES -- 모든 테이블
```

```javascript
CREATE PUBLICATION 이름 FOR ONLY TABLE 테이블, 테이블2 -- 특정 테이블
```

```javascript
CREATE PUBLICATION 이름 FOR TABLE ONLY 테이블 WITH (publish = 'update'); -- 특정 테이블의 update 연산만
```

예를 들어 global_state라는 테이블에 대해 update/insert 기록을 조회하고 싶다면, 이런 식으로 만들 수 있다.  

![](/images/img_b4598f1756f692f5.png)

publication 목록은 이걸로 조회할 수 있고  

```javascript
SELECT * FROM pg_publication order by oid desc
```

![](/images/img_209e190acb09e00b.png)

publication에 속한 테이블 목록은 이걸로 조회할 수 있다.  

<br>

```javascript
SELECT * FROM pg_publication_tables where pubname = 'pub 이름'
```

![](/images/img_60be1cad90353171.png)

![](/images/img_22eec9752b05ed46.png)

삭제는 반대로 DROP 명령을 통해서 처리할 수 있다.  

![](/images/img_9230604f4f4cdf02.png)

생성 이후에 테이블을 추가하거나 제거하고 싶다면, ALTER PUBLICATION을 사용할 수 있다.  

![](/images/img_c71ca4cd76784f4a.png)
ADD TABLE이나 DROP TABLE을 사용하면 된다.

## Slot 만들기

변경내역을 조회하려면 슬롯이란걸 또 만들어야 한다.  
슬롯을 기반으로 해서 일종의 페이지네이션 조회를 할 수 있다.  

```javascript
SELECT pg_create_logical_replication_slot(슬롯이름, '출력 포맷');
```
출력 포맷은 응답이 어떤 형태로 나올지를 정의한다.  
pgoutput이 가장 일반적인 선택이고, 바이너리 형태로 응답을 반환한다. pgoutput을 쓸 경우에는 단순한 SQL 인터페이스로는 조회를 할 수 없다.  

<br>

![](/images/img_75441a6cd73a79bd.png)
이렇게 만들고 나면  

만들어진 slot은 다음 쿼리로 조회할 수 있고  

```javascript
select * from pg_replication_slots
```

![](/images/img_440fcc10f8003cbe.png)

슬롯을 지우고싶다면 이렇게 지울 수 있다.  

```javascript
SELECT pg_drop_replication_slot('슬롯이름');
```

![](/images/img_afe757e014e31ed3.png)

## 변경 데이터 구독하기

변경 데이터 구독은 pg_logical_slot_peek_binary_changes 혹은 pg_logical_slot_get_binary_changes을 사용해서 처리할 수 있다.  
peek은 데이터를 조회만 하고, get은 조회 후 소비까지 한다.  

<br>

```javascript
SELECT *
FROM pg_logical_slot_peek_binary_changes('슬롯이름', NULL, NULL, 'proto_version', '1', 'publication_names', 'publication_이름')
SELECT *
FROM pg_logical_slot_peek_binary_changes('슬롯이름', NULL, NULL, 'proto_version', '1', 'publication_names', 'publication_이름')
```

![](/images/img_3505cda87fd54311.png)
그럼 이런식으로 조회가 되는데, 문제는 저 데이터가 바이너리 인코딩이라서 해석하려면 디코딩을 해야 한다는 것이다. 이건 SQL 내에서 처리하긴 좀 까다로울 수 있고, 프로그램으로 해결해야 한다.

## 수동 슬롯 위치 조절

peek는 슬롯 위치를 이동하지 않고 데이터를 꺼내오고, get은 위치를 이동시키면서 꺼내온다고 했다.  

근데 그러면 데이터 가공을 완료한 뒤에 위치를 이동시키려면 어떻게 할까?  
get을 썼는데, 오류가 나면 그대로 데이터가 증발하는 셈이고, peek를 쓰면 위치가 이동되지 않는다.  

그럴때는 pg_replication_slot_advance라는 함수를 쓸 수 있다.  

```javascript
pg_replication_slot_advance('슬롯이름', '커서(lsn)')
```

![](/images/img_78b7e307bf93a6de.png)
만약 저기서 6번째까지 다 처리했다고 하면  

<br>

![](/images/img_6f3cc3f4cc71afce.png)
이렇게 슬롯 위치를 이동시킬 수 있다.  

<br>

![](/images/img_4a882b01ab97fb2d.png)
그러면 이후에 조회할 때는 그 로그부터 조회가 될 것이다.
