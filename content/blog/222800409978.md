## 함수 생성

함수 콘솔로 이동해 "만들기"를 클릭한다.  

![](/images/img_b36584114838333c.png)

구독과 리소스 그룹은 취향껏 넣어주고, 이름과 런타임을 결정한다.  

![](/images/img_7c6739b83791ffef.png)
이름이 도메인으로 박혀서 unique 검사가 까다로우니 감안을 좀 해야하고, OS도 선택할 수 있다.  

<br>

![](/images/img_33a47c4df7f6e315.png)
런타임은 그렇게 많지는 않다.  
이외에 필요한게 있다면 커스텀을 사용해야할 것이다.  

완료를 하면, 리소스가 생성될 것이다.  
세팅에는 시간이 좀 걸린다.  

![](/images/img_e4ade576b77bd28f.png)

## 코드 편집

Azure Function은 좀 불편하게도 콘솔에서 바로 코드를 보거나 고칠 수는 없다.  

빠른 시작에 보면 편집기를 세팅하고 배포하는 법에 대해 알려준다.  

![](/images/img_b35c1045afa69972.png)

![](/images/img_632013ce2c4f25d6.png)
나는 VSCode를 자주 사용하므로 VSCode 위주로만 다루겠다.  

Azure Function용 확장을 깔고  

![](/images/img_84760bb9cb4ad163.png)

npm으로 다음 모듈을 설치해준다.  

```javascript
npm install -g azure-functions-core-tools@4 --unsafe-perm true
```

functions용 커맨드툴도 깔아두는게 좋다.  
아래 링크에서 다운받거나, 다운받는법을 볼 수 있다.  
[https://docs.microsoft.com/en-us/azure/azure-functions/functions-run-local?tabs=v4%2Cwindows%2Ccsharp%2Cportal%2Cbash#v2](https://docs.microsoft.com/en-us/azure/azure-functions/functions-run-local?tabs=v4%2Cwindows%2Ccsharp%2Cportal%2Cbash#v2)  

배포까지 커맨드로 하려면 azure cli도 있어야 한다.  
다음 링크에서 플랫폼대로 설치하자.  
[https://docs.microsoft.com/ko-kr/cli/azure/install-azure-cli](https://docs.microsoft.com/ko-kr/cli/azure/install-azure-cli)  

확장을 깔았다면 아래와 같은 관리탭이 추가된다.  

![](/images/img_e086d588fd7f0b73.png)

그리고 폴더를 만들고 관리도구로 설정파일을 초기화해준다.  

```javascript
func init
```

![](/images/img_292e8a2f8db87f00.png)

그 다음에는 함수 코드를 추가한다.  

```javascript
func new --name HttpExample --template "HTTP trigger" --authlevel "anonymous"
```

그러면 다음과 같이 프로젝트 폴더가 구성될 것이다.  

![](/images/img_80cb0deb897d9cc0.png)

## 배포하기

먼저 커맨드라인으로 배포하는법부터 설명해보겠다.  
azure 로그인을 하고  

![](/images/img_1df3dd5699be9f84.png)

아래의 명령어를 실행한다.  

```javascript
 func azure functionapp publish 함수명
```

만약 다음과 같은 오류가 발생한다면  

![](/images/img_1dc23360a0980b0b.png)

"AzureWebJobsStorage": "UseDevelopmentStorage=true" 세팅값을 추가해준다.  

![](/images/img_852e2107938fa96b.png)

그리고 다시 시도하면  

![](/images/img_6460471692793d02.png)

이제 올라갈 것이다.  

![](/images/img_08eed1f40e5a7928.png)

관리용 VSCode 확장을 사용하면 이보다 더 간편하게 처리할 수도 있다.  
선택하고, 배포 클릭하고, 현재 로컬 디렉터리를 설정한다.  

![](/images/img_0e89d9c99ae6b252.png)
그거면 끝이다.  

똑같이 올라갈 것이다.  

![](/images/img_cf6021e12ce761e1.png)
