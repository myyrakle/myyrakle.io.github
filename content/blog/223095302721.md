![](/images/img_ca0ec981db294351.png)
envoy는 nginx와 비슷한 포지션에 있는 리버스 프록시 시스템이다.  

2016년에 나왔다. 메이저한 프록시 시스템들 중에는 가장 젊은 녀석이다.  

주된 사용처는 역시 로드밸런서를 만드는 것이다.  

보니까 GCP에서 제공하는 로드밸런서도 envoy로 만든거더라.  
프로덕션에도 의외로 많이 사용되는 편이다.  

장점은 풍부한 기능과 뛰어난 성능이다.  

MSA에 대한 지원도 빠릿해서, grpc, protobuf에 대한 기능도 지원한다.  

성능만 두고보면 envoy가 nginx보다 2배 정도는 빠르다.  

그리고 순수한 오픈소스란 점도 특기할만한 부분이다. 경쟁자인 nginx와 HAProxy도 오픈소스이긴 하지만, 기업체가 보유-관리하고 있으며, 오픈소스 버전과 상업용 버전의 기능 차별이 존재한다.  
그런 부분에서 envoy는 매우 독립적이고 깨끗한 편이다.

## 설치 및 실행

우분투에서는 다음 명령을 사용해서 설치할 수 있다.  

```javascript
sudo apt update
sudo apt install apt-transport-https gnupg2 curl lsb-release
curl -sL 'https://deb.dl.getenvoy.io/public/gpg.8115BA8E629CC074.key' | sudo gpg --dearmor -o /usr/share/keyrings/getenvoy-keyring.gpg
Verify the keyring - this should yield "OK"
echo a077cb587a1b622e03aa4bf2f3689de14658a9497a9af2c427bba5f4cc3c4723 /usr/share/keyrings/getenvoy-keyring.gpg | sha256sum --check
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/getenvoy-keyring.gpg] https://deb.dl.getenvoy.io/public/deb/ubuntu $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/getenvoy.list
sudo apt update
sudo apt install -y getenvoy-envoy
```

설치가 잘 됐다면 cli로 실행할 수 있다.  

![](/images/img_821cbedba3579dc3.png)

docker를 사용할 때는 이렇게 가져와서 쓰면 된다.  
latest 태그가 없어서 항상 버전을 명시해야 하고, nginx와는 다르게 기본동작이 없어서 설정파일을 넣어주지 않으면 뭐가 따로 동작하지는 않는다.  

```javascript
docker pull envoyproxy/envoy:v1.26-latest
docker run envoyproxy/envoy:v1.26-latest
```

## 프록싱해보기

다른 서버를 프록싱하는 간단한 로드밸런서를 구현해보자.  
일단 로컬 12345 포트에 간단한 서버를 띄워놨다.  

![](/images/img_d68fd5c17fea7010.png)
저걸 프록싱하게 하겠다.  

다음과 같은 형태로 yaml을 짜준다.  

```javascript
static_resources:
  listeners:
  - name: listener_0
    address:
      socket_address: { address: 0.0.0.0, port_value: 80 }
    filter_chains:
    - filters:
      - name: envoy.filters.network.http_connection_manager
        typed_config:
          "@type": type.googleapis.com/envoy.extensions.filters.network.http_connection_manager.v3.HttpConnectionManager
          stat_prefix: ingress_http
          codec_type: AUTO
          route_config:
            name: local_route
            virtual_hosts:
            - name: local_service
              domains: ["*"]
              routes:
              - match: { prefix: "/" }
                route: { cluster: some_service }
          http_filters:
          - name: envoy.filters.http.router
            typed_config:
              "@type": type.googleapis.com/envoy.extensions.filters.http.router.v3.Router
  clusters:
  - name: some_service
    connect_timeout: 0.25s
    type: STATIC
    lb_policy: ROUND_ROBIN
    load_assignment:
      cluster_name: some_service
      endpoints:
      - lb_endpoints:
        - endpoint:
            address:
              socket_address:
                address: 0.0.0.0
                port_value: 12345
```
리스너는 외부에서 트래픽을 받아서 처리하는 영역을 정의한다. 여기서는 0.0.0.0:80로 해서 80 http로 접근할 수 있게 했다. 그리고 밑에서 한 클러스터 설정으로 트래픽을 라우팅했다.  

리스너가 프론트엔드면 클러스터가 백엔드인 셈이다. 클러스터에서는 로컬에 떠있는 12345 포트 서버를 가리키도록 구성했다.  

저렇게 만들어놓고 cli로 실행을 해주면 설정파일이 지정됨과 동시에 재실행이 된다.
