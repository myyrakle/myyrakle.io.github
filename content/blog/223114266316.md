git submodule은 git 차원에서 지원하는 자체적인 모듈시스템이다.  
하나의 git 프로젝트에, 다른 프로젝트를 포함시키고 clone받아와서 사용하는 구조를 가진다.  

근데 이게 쓰다보면 히스토리 관리도 신경을 써야하고.. 여러모로 피곤한게 많아서 호불호가 갈린다.  

그래도 한정된 상황에서는 대체품이 없어서 종종 써야할일이 있다.  

<br>

<br>

<br>

## 서브모듈 등록

프로젝트에 새로 모듈을 등록하려면, submodule add 명령을 사용한다.  

```javascript
git submodule add 링크
```

만약 llvm 프로젝트를 모듈로서 추가하고 싶다면, 아래와 같이 그 github 링크를 박아주면 되는 것이다.

## submodule 재설치

git 코드를 다시 처음부터 clone받았다면, 서브모듈들은 그냥 텅 비어있을 것이다.  

그러면 초기화를 해줄때마다 .gitmodules를 기반으로 submodule을 다시 내려받아야 하는데, 그럴때는 다음과 같이 init 명령을 사용해주면 된다.

원본 모듈 레포지토리에 변경사항이 발생했고, 그걸 지금 다시 반영해야 한다면 다음과 같이 update 명령을 사용해준다.  

![](/images/img_ac8d3f76adef518d.png)

## submodule의 재귀적 참조

서브모듈에 또 서브모듈이 있을 경우에는 어떻게 처리를 해야할까?  
다행히 전용 명령을 제공한다.  

init을 재귀적으로 수행하려면 이렇게 날리면 되고  

```javascript
git submodule update --init --recursive
```

update만 하는거면 이렇게 하면 된다.  

```javascript
git submodule update --recursive
```
