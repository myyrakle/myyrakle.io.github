Agent Skills는 앤트로픽에서 공개한 에이전트 개발용 생산성 강화 수단이다.  
쉽게 말해서, 에이전트를 위한 라이브러리 같은 것이다.  

프롬프트를 입력으로 받아서 파워포인트를 만든다거나 pdf를 수정/생성한다거나 하는 파일 기반의 헤비하거나 복잡할 수 있는 작업들을 모듈의 형태로 제공하는 것이 특징이다.  
내세우는 주요 장점은 토큰 사용량을 줄일 수 있고, 규칙 기반의 점진적 처리를 용이하게 해준다는 것이다.  

앤트로픽을 구독해야만 사용할 수 있고, 현 시점에서는 안정화가 되지 않은 기능이다. 아직 베타 버전이다.  

그리고 특이하게도 Skill에 대한 처리 작업은 일종의 가상머신 위에서 실행되는 구조를 갖고 있다.

## 프리셋 Skills 사용해보기 (with Python)

LLM 환경들이 으레 그렇듯이, 이것도 환경이 Python 쪽에 몰빵되어있다.  
uv를 쓴다고 가정하겠다.  

skills를 쓰려면 다음 전용 모듈이 필요하다.  

```javascript
uv add anthropic
```

앤트로픽 클라이언트를 통해 앤트로픽에서 지원하는 skill 목록을 조회해보자.  

```javascript
import anthropic

api_key = "..."

client = anthropic.Anthropic(api_key=api_key)

# List Anthropic-managed Skills
skills = client.beta.skills.list(source="anthropic", betas=["skills-2025-10-02"])

for skill in skills.data:
    print(f"{skill.id}: {skill.display_title}")
```

![](/images/img_e6155cf7d7549cb3.png)
현 시점에서 제공되는 스킬 프리셋은 단 4가지 뿐이다.  
저 skill.id(pdf 등)을 사용하면 저 skill을 사용할 수 있다.  

간단한 프롬프트를 집어넣어서 pptx 파일을 만들어보자.  

```javascript
# Create a message with the PowerPoint Skill
response = client.beta.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=4096,
    betas=["code-execution-2025-08-25", "skills-2025-10-02"],
    container={
        "skills": [{"type": "anthropic", "skill_id": "pptx", "version": "latest"}]
    },
    messages=[
        {
            "role": "user",
            "content": "Create a presentation about renewable energy with 5 slides",
        }
    ],
    tools=[{"type": "code_execution_20250825", "name": "code_execution"}],
)

print(response.content)
```
이런 식으로 쏘면 된다.  
모델 정하고, 토큰 제한 걸고, messages로 요구사항 프롬프트를 지정해서 실행하면 그만이다.  

실행 시간 자체는 꽤 오래 걸릴 수 있다. 기본적으로 빠르게 되는 기능들은 아니다.  

![](/images/img_b8e2052ab7ee0c7f.png)
그럼 이런 식으로 출력이 나올 것이다.  
이 경우에는 pptx가 태그 기반 포맷이라 좀 어지럽긴 한데, 실제로 잘 만들어진 것은 맞다.

## 사용자 정의 Skill

Skill은 사전 제공하는 것 이외에도 직접 만들어서 쓸 수도 있다.  
이게 더 일반적인 사용사례일 것이다.  

공식 예제는 여기에 있다.  
[https://github.com/anthropics/claude-cookbooks/tree/main/skills/custom_skills](https://github.com/anthropics/claude-cookbooks/tree/main/skills/custom_skills)  

먼저 스킬을 정의해야 한다.  
강요되는 프로젝트 구조가 있다.

가장 필수적인 것은 SKILL.md라는 핵심 프롬프트 파일이다.  
저기 상단에 박혀있는게 최소한의 정보를 가진 메타데이터 섹션이고, 그 아래에 상세한 프롬프트가 나열된다.

## 감상

가끔 유용한 경우가 있을 것 같긴 한데, 글쎄... 지금으로서는 좀 계륵 같은 기능 같기도 하다.  
MCP, tools, 혹은 일반적인 개발 방법을 두고 왜 이걸 써야하는지에 대한 당위성이 충분하지 않다. 게다가 앤트로픽 환경에 지나치게 종속된다.  

이런 애매한 지점들 때문인지, 실제로 MCP에 비해서는 관심을 크게 받지 못하는 상황이다.
