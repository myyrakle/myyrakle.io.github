엘라스틱서치를 뭐때문에 쓰나? 검색하려고 쓰지.  

엘라스틱서치는 쿼리 API를 통해 인덱스에 저장된 값들을 빠르게 읽어올 수 있다.  
조건이나 정렬도 자유롭게 걸 수 있고, 어떻게 짜도 웬만하면 빠르게 결과를 받을 수 있다.  

이 포스트에서는 기본 쿼리 단위 몇가지를 먼저 소개하도록 하겠다.  
좀더 복잡하게 조합할 수도 있고 이런저런 커스텀도 많지만, 일단 기본부터.  
[https://www.elastic.co/guide/en/elasticsearch/reference/current/term-level-queries.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/term-level-queries.html)  

쿼리는 "GET /인덱스명/_search"나 "POST /인덱스명/_search"의 형태로 호출할 수 있다.  

Restful API를 사용하는게 정석이지만 간단하게 테스트를 할때는 키바나의 Dev Tool이 편리하고 괜찮다.  

![](/images/img_a8965afa18148d99.png)
탭 맨 아래에 있다.

## 전체 조회 쿼리: match_all

그냥 전부다 읽어오고 싶다면 그냥 이렇게 쏘면 된다.  
딱 이것만을 위한 기능이다.  

```javascript
POST products2/_search
{
  "query": {
    "match_all": {}
  }
}
```
기본적으로 query 파라미터로 조건을 전달한다.  

그럼 이렇게 잘 뿌려줄 것이다.  

![](/images/img_25e9b1647d162122.png)

## 단순 일치 쿼리: term, terms

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-term-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-term-query.html)  

term은 가장 기본적이고 단순한 조건절 중 하나다.  

딱 조건에 1:1로 일치하는 값만을 가져온다.  
가령 다음과 같이 쓴다면  

![](/images/img_f0d75e5394f5a1ee.png)
shop_id가 1인 데이터만 가져오는 것을 볼 수 있다.  

그리고 여러개를 지정하려면 term 대신 terms를 써서 배열로 값을 주면 된다.  

![](/images/img_e94d6bb377c580f6.png)

다만, 이건 그냥 저런 단순 식별값만을 매칭시키기 위한거지, 텍스트검색용은 아니다.  
텍스트에 띄어쓰기가 포함되기라도 하면, 기대한 것과 다르게 동작하기 시작한다.  

![](/images/img_13ab772645e54175.png)
분명히 '시원한 샴푸'가 있음에도 찾질 못하는 것을 알 수 있다.  

단어를 기준으로는 가져오긴 한다.  

![](/images/img_c1a8807cf664db4d.png)

## 텍스트 검색 쿼리: match

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-match-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-match-query.html)  

match는 텍스트검색에 최적화된 쿼리다.  

![](/images/img_1c1c8ad00405fa4e.png)
자. 전과 다르게 이제 "시원한 샴푸"로 검색을 해도 검색이 잘 된다.  
하지만 완전일치하지 않는 "공산당 샴푸"도 검색이 됐다!  

이건 match가 그냥 조건값을 그대로 보내는게 아니라, 내부적으로 analyze 단계를 거치기 때문이다.  
analyzer에는 여러가지 레이어가 추가될 수 있고, 그건 또 차후 자세히 다룰 예정이지만, 일단 지금에서는 문자열을 '공백'을 기준으로 잘라서 그 각각의 단어들을 검색에 사용한다는 것만 알아두자.  

그래서 '시원한 샴푸'를 '시원한'과 '샴푸'로 분리해서 검색에 사용한 것이고, '공산당 샴푸'의 '샴푸'와 조건이 일치해서 검색에 노출되게 된 것이다.

<br>

## 텍스트 검색 쿼리2: match_phrase

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-match-query-phrase.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-match-query-phrase.html)  

이건 좀더 정확도 높은 텍스트 검색을 위한 기능이다.  

그냥 삼푸를 쳐도 샴푸가 포함된 녀석들이 전부 나오는 것이나  

![](/images/img_7040bdbc989f606b.png)

띄어쓰기가 포함된 텍스트가 잘 나오는 건 match와 같다.  

![](/images/img_39c06cc3434edf6d.png)

하지만 match가 막 단어의 순서를 뒤섞어도 순서와 별개로 다 가져오는 것과 별개로  

![](/images/img_4f5a88c02f11f82c.png)

match_phrase는 가져오지 않는다.  

![](/images/img_8820e85d93526bcd.png)
이건 엄격한 의미에서의 문자열 포함만을 처리하기 때문이다.

## 텍스트 복합검색 쿼리: multi_match

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-multi-match-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-multi-match-query.html)  

이건 match랑 거의 똑같긴 한데, 하나의 값으로 여러 필드에 동시에 조건을 쏠 수 있다는게 편리한 점이다.  
fields가 조건으로 삼을 필드 목록이고, query가 검색할 값이다.  
이렇게 하면  

![](/images/img_7088932965339dd4.png)
'시원한'과 '인테리어'가 분리돼서 각각 조건으로 발사되고, product_name이나 category_names 중 하나만 일치돼도 검색결과에 포함이 된다.

## 텍스트 검색 쿼리3: query_string

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-query-string-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-query-string-query.html)  

이건 좀더 엄격한 형태로 여러 검색어 조건을 걸 수 있는 기능이다.  

일단 기본적으로 match와 동일한 방식으로 동작하지만  

![](/images/img_5ece60eaae1a592d.png)

검색 텍스트 내에 OR이나 AND 등의 키워드를 삽입할 수 있다는 것이 다르다.  

![](/images/img_2c1a49c6598dcc48.png)

![](/images/img_fba617770111ed3e.png)
OR나 AND 등의 구분자로 분리된 텍스트들은 별도로 분석과정을 거쳐서 전달된다.

## 텍스트 접두사 검색 쿼리: prefix

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-prefix-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-prefix-query.html)  

이건 특정 문자열로 시작하는 것을 조건으로 부여해주는 쿼리다.  
검색시에 자동완성같은걸 구현하기 괜찮다.  

그래서 다음과 같이 "시원한"을 전달한다면  

![](/images/img_b8cbc9f938501327.png)
"시원한"으로 시작하는 "시원한 샴푸"를 가져올 수 있다.

<br>

## 텍스트 와일드카드 쿼리: wildcard

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-wildcard-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-wildcard-query.html)  

이건 match 검색기능에 와일드카드 기능이 확장된 검색 쿼리다.  

다른건 거의 같지만, 단일 와일드카드 ?와 복수 와일드카드 *를 특수하게 처리해주는게 다르다.  

그래서 만약 "시?한"으로 검색을 시도한다면  

![](/images/img_209f5872ab1c2fbf.png)
가운데 뚫린 한글자가 "원"으로 "시원한"이 조건에 걸려서 "시원한 삼푸"가 검색되게 된다.  

복수 와일드카드 *를 사용하면 "시원한"과 "시니컬한"이 둘다 걸리게 된다.  

![](/images/img_4efad4a5c909ec0e.png)

유연한 검색기능이 필요하다면 유용할 것 같다.

## 범위 검색 쿼리: range

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-range-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-range-query.html)  

range는 말 그대로 날짜나 시간값에 대한 특정 범위를 조건으로 주는 것이다.  

gte(<=), lte(>=), gt(<), lt(>)로 범위값을 지정만 해주면 된다.  
다음은 product가 2 이상 3 이하인 것만 가져오는 쿼리다.

## 확인 쿼리: exists

[https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-exists-query.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/query-dsl-exists-query.html)  

이건 특정 필드가 존재하는지를 조건으로 걸게 해주는 기능이다.  
대단한건 아니고, null이나 빈 배열만 아니면 존재하는 걸로 친다.  

![](/images/img_ec520dfdc0ed1cf2.png)

부정 조건을 걸면 존재하지 않는 것만 가져올 수도 있다.  

![](/images/img_bc460dcf113b3dd6.png)
bool이나 must_not은 복합쿼리를 구성하기 위한 요소인데, 저건 나중에 설명하겠다.  

대충 기본은 이것만 알아도 될 것 같다.  
나머지는 천천히 살펴보자.
