with문은 쿼리의 결과를 임시로 묶어둘 수 있게 해주는 편리한 기능이다.  
이런걸 공통 테이블 식(Common Table Expression:CTE)이라고 부른다.

쉽게 말해, 다른 언어에서 변수를 선언하는 것과 동등하다.  
쿼리 결과를 일종의 변수로 묶어놓고 그 쿼리가 끝날때까지 재사용할 수 있는 것이다.

with 절은 아래의 구조를 가진다.

```javascript
WITH
이름1 AS
(
   쿼리1...
),
이름2 AS
(
   쿼리2...
)
...
```

아래와 같이 단순하게 SELECT 결과를 묶어다가 사용할 수도 있고

![](/images/img_71117673c1612a84.png)

INSERT나 UPDATE, DELETE의 결과도 반환을 받아서 묶어둘 수가 있다!

![](/images/img_c323cc8e2bf75040.png)
이를 응용하면 의존관계를 가진 여러 테이블의 수정을 보다 손쉽고 빠르게 처리할 수 있다.

잘 사용한다면 굉장히 편리하고 좋은 기능이지만, 역시 이것도 남용하지는 않는 것이 좋다.  
쿼리가 실행되는 동안에는 WITH으로 묶어놓은 값들이 메모리에 올라가 유지되기 때문에, 메모리에 부담을 줄 수도 있다.
