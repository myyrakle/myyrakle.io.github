HAProxy는 헬스체크를 통해 서버의 생존여부를 파악하는 기능을 제공한다.  
헬스체크에 실패한 서버는 실제 라우팅 대상에서 제외되고, 헬스체크가 다시 성공하면 라우팅 대상으로 돌아온다.  

헬스체크는 다음과 같은 형태로 정의할 수 있다.  
이 경우에는 "GET /" API를 헬스체크로 사용했다.  

```javascript
frontend test-front
    mode http
    bind *:8000
    default_backend test-back

backend test-back
    balance roundrobin
    option httpchk GET /
    server server1 0.0.0.0:8081 check
    server server2 0.0.0.0:8082 check
```

![](/images/img_8795f5ecf00260fe.png)
헬스체크는 TCP로도 가능한데, 여기서는 HTTP만 예시로 다룬다.  

만약 갑자기 server2 서버가 다운된다고 하면, 순간적으로 server2에 연결한 요청은 뻗을 수 있지만  

![](/images/img_ea1deb41525df592.png)

server2가 헬스체크에 실패해서 라우팅 대상에서 빠지게 되므로, server1에만 라우팅이 되게 된다.  

![](/images/img_a85d2c605d855b0c.png)
하지만 실패했다고 끝은 아니다. server2가 다시 실행되고, 헬스체크에도 성공하면 다시 라우팅 대상이 된다.

## 기타 설정

**disabled**  
특정 서버를 임의로 라우팅 대상에서 제외하고 싶다면, disabled 키워드를 쓸 수 있다. 이걸 쓰면 저 행 자체가 빠지는 것과 동일한, comment와 동일한 효과를 준다.  

```javascript
backend test-back
    balance roundrobin
    option httpchk GET /
    server server1 0.0.0.0:8081 check
    server server2 0.0.0.0:8082 check
```

**http-check**  
http-check 절과 expect를 사용하면 응답 상태 코드를 지정할 수도 있다.  
이러면 응답이 200으로 왔을 때만 헬스체크가 성공한 것으로 친다.  

```javascript
backend test-back
    balance roundrobin
    option httpchk GET /
    http-check expect status 200
    server server1 0.0.0.0:8081 check
    server server2 0.0.0.0:8082 check
```

![](/images/img_3b8a61d2634286fc.png)

**inter**  
inter 키워드를 사용하면 헬스체크에 대한 간격도 조정 가능하다. 다음 설정은 4초마다 헬스체크를 하게 한다.  

```javascript
backend test-back
    balance roundrobin
    option httpchk GET /
    server server1 0.0.0.0:8081 check inter 4s
    server server2 0.0.0.0:8082 check inter 4s
```
설정하지 않을 경우 기본값은 2초다.  

**fall**  
fall 키워드를 사용하면 실패에 대한 기준도 정할 수 있다.  
다음 설정은 5번 실패했을때부터 실패로 가정한다.  

```javascript
backend test-back
    balance roundrobin
    option httpchk GET /
    server server1 0.0.0.0:8081 check fall 5
    server server2 0.0.0.0:8082 check fall 5
```
fall의 기본값은 3이다.  

**rise**  
rise 키워드를 사용하면 죽은 상태에서 healthy한 상태로 다시 판단하는 성공 횟수를 지정할 수 있다.  
다음과 같이 지정한다면, 7번 실패하면 죽었다고 판단해서 뒤로 미루고, 그 상태에서 다시 8번 성공하면 살았다고 판단한다.  

```javascript
backend test-back
    balance roundrobin
    option httpchk GET /
    server server1 0.0.0.0:8081 check fall 7 rise 8
    server server2 0.0.0.0:8082 check fall 7 rise 8
```
rise의 기본값은 2다.
