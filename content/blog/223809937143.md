Active Record 패턴은 데이터베이스를 편리하게 다루기 위한 패턴 중 하나다.  
쉽게 말해 ORM 구현들에 사용한다.  

<br>

<br>

<br>

## 기본 원리

ORM을 자칭하는 대부분의 ORM 라이브러리들은 Active Record 패턴에 기반해 구현된다고 생각하면 된다.  
Active Record가 뭐냐하면 이런거다.  

```javascript
part = new Part()
part.name = "Sample part"
part.price = 123.45
part.save()
```
각각의 엔티티를 클래스-타입으로 정의하고, 클래스의 필드들을 통해 column 값을 제어한다.  

객체의 값을 수정 후에 저장 액션을 메서드를 통해 트리거(save)하면 수정된 값을 실제로 데이터베이스에 수정하는 쿼리를 날린다. 그리고 생성, 삭제 등도 마찬가지로 객체와 객체 필드들을 통해 제어한다.  

사실 뭐 복잡하게 설명할 것도 달리 없다.  

<br>

<br>

<br>

## 구현체

사실 대부분의 ORM이 Active Record를 지원한다고 봐도 된다.  
Python - sqlalchemy  
Java - JPA & @  
Rust - SeaORM  
Node - TypeORM, Sequelize, Prisma, Drizzle  

등등...  

사실 이런걸 직접 구현하는 것도 그리 어렵지는 않을 것이다.
