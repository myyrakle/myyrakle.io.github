Node를 사용하다보면 때로는, js 코드만으로는 부족할 경우가 있다.  
성능 때문이라든지, 다른 unsafe한 행동이 필요하다든지, C++의 특정 라이브러리를 쓸 필요가 있을 때 말이다.  

다행히도 Node는 C++용 인터페이스를 제공한다.  
애초에 Node 자체가 C++로 구현돼서 그런 것 같다만..  
사실 쓰기는 몹시 불편한 편이다.  
일단 해보자  

가장 먼저 필요한 것은 node-gyp이다. 빌드 관련 작업들을 전부 처리해주는 녀석이다.  

![](/images/img_7c198987ad548444.png)

그리고, C++ 컴파일러도 세팅된 상태여야 한다.  
윈도우라면 VSC, 리눅스면 GCC. 없으면 컴파일을 못한다.  
파이썬도 있어야 한다고 한다...  

다 갖췄다면 이제 소스를 작성해보자.  
간단하게 "world" 텍스트만 반환하는 함수를 구현한 것이다.  
겁나게 조잡하다.  

![](/images/img_2c4d4ee2634ad9b0.png)
node.h는 node-gyp에서 알아서 넣어주는 기본 헤더이니, 뭘 따로 찾을 필요는 없다.  
GetReturnValue에 Set으로 값을 넣어주면, 알아서 리턴값으로 인식하고 꺼내간다고 한다.  
코드다.  

```javascript
#include <node.h>

//내보낼 구현 함수
void Foo(const v8::FunctionCallbackInfo<v8::Value>& args) {
    v8::Isolate* isolate = args.GetIsolate();

    //반환값 설정
    args.GetReturnValue().Set(
        v8::String::NewFromUtf8(isolate, "world", v8::NewStringType::kNormal)
            .ToLocalChecked()
    );
}

//Foo 함수를 hello라는 이름으로 내보냄
void Initialize(v8::Local<v8::Object> exports) {
    NODE_SET_METHOD(exports, "hello", Foo);
}   

//Node gyp 규격으로 내보냄?
NODE_MODULE(NODE_GYP_MODULE_NAME, Initialize)
```

그리고 binding.gyp라는 특수한 설정파일을 루트 경로에 만든다.  
그리고 대강 아래와 같이 쳐준다.  

![](/images/img_560702bde1e855e7.png)
target_name은 방출될 모듈의 이름이고,  
source는 c++ 소스들이다.  

저렇게 해두고 node-gyp configure를 하면 이런저런 설정파일들이 생성되고  

![](/images/img_41027b2679816c0d.png)

build를 통해 c++ 소스들을 node에서 돌아갈 수 있게끔 만들어줄 수 있다.  

![](/images/img_bb2db75b8b19d7c3.png)

build가 문제없이 완료된다면, build/Release 경로로 애드온 모듈이 생겨있을 것이다.  

![](/images/img_3bcded7ae2faed61.png)
