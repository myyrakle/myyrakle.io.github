이번 포스트에서는 docker를 통해서 couchDB 서버를 띄워보고 사용하는 방법을 간단하게 다뤄보겠다.  

<br>

<br>

<br>

## Docker로 실행하기

다음과 같이 계정과 패스워드를 설정한 채로 컨테이너를 띄우면 테스트 수준의 세팅은 간단하다.  

```javascript
sudo docker run -p 5984:5984 -e COUCHDB_USER=admin -e COUCHDB_PASSWORD=q1w2e3r4 couchdb
```

![](/images/img_246048254a309c63.png)

그리고 열어둔 포트로 바로 접근이 되면 잘 실행된 것이다.  

![](/images/img_91354296e184c513.png)

<br>

<br>

<br>

## 데이터베이스의 조회와 생성

다음과 같이 _all_dbs를 조회하면 전체 데이터베이스 목록을 조회할 수 있다.  

```javascript
curl -X GET http://admin:q1w2e3r4@127.0.0.1:5984/_all_dbs
```

![](/images/img_2d809340169dcaf8.png)
아직은 아무것도 없다.  

PUT api를 사용하면 바로 데이터베이스를 만들 수 있다.  
다음은 users라는 이름의 데이터베이스를 만드는 명령이다.  

```javascript
curl -X PUT http://admin:q1w2e3r4@127.0.0.1:5984/users
```

![](/images/img_7b64bee652a70ba8.png)

이미 존재하는 DB명으로는 다시 생성 요청을 날릴 수는 없다.  

![](/images/img_4234676c63641513.png)

<br>

<br>

<br>

## 데이터베이스의 삭제

삭제는 restful하게 DELETE api를 통해서 수행한다.  

```javascript
curl -X DELETE http://admin:q1w2e3r4@127.0.0.1:5984/users
```

![](/images/img_4eb50f9207046362.png)
직관적이다.
