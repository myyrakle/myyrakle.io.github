인덱스는 엘라스틱서치에서 검색을 사용하는데 있어 가장 중심이 되는 요소다.  
말 그대로 검색을 위한 인덱스 구조를 정의하는 가장 큰 틀인 것이다.  

골자는 이렇다.  
검색용 데이터 구조는 인덱스를 처음 생성할때 지정해주고,  
검색용데이터도 그렇게 만들어둔 인덱스에 넣어두고,  
검색을 할 때도 인덱스를 통해서 조건을 걸고 꺼내오는 것이다.  

만약 어떻게든 엘라스틱서치 환경을 구축했다면, 대충 다음과 같이 접속 가능한 엘라스틱 링크가 있을 것이다.  

![](/images/img_2b8e9ac07594277c.png)
이 링크와 인증토큰을 통해서 Restful API로 엘라스틱서치를 제어할 수 있다.

## 인덱스 생성
인덱스를 생성하려면 "PUT /인덱스명"의 형태로 API를 호출해주면 된다.  
나는 평소에 즐겨쓰던 VSCODE 확장 Rest client로 API를 주로 쏘는데, 편한걸로 아무거나 쓰면 된다.  

```javascript
@host = 호스트...
@token = 위에 토큰...

###
PUT https://{{host}}/products
Content-Type: application/json
Authorization: {{token}}

{
    "settings": {},
    "mappings": {
        "properties": {
            "product_id": { "type": "integer" },
            "product_name": { "type": "text" },
            "shop_id": { "type": "integer" },
            "shop_name": { "type": "text" },
            "color": { "type": "text" },
            "brand_id": { "type": "integer" },
            "brand_name": { "type": "text" },
            "description": { "type": "text" },
            "category_names": { "type": "text" }
        }
    }
}
###
```
우선 Authorization 헤더에 인증토큰을 넣어줘야 한다.  
setting은 잡다한 인덱스 설정들이 들어가는 부분인데 그건 나중에 보자.  
mappings의 properties가 인덱스의 목록을 지정해주는 부분이다.  
저렇게 잡아놓은 속성들은 전부 검색조건으로 사용할 수가 있다.  

저렇게 해서 문제없이 호출된다면  

![](/images/img_d89dbb1435ac240c.png)
위와 같이 응답이 올 것이다.  

정상적으로 생성이 됐다면 인덱스 목록에서도 추가가 될 것이고  

![](/images/img_114b4887cc09560d.png)

"GET /인덱스명"의 형태로 접근해서 인덱스 정보를 조회하는게 가능해진다.

## 인덱스 재배포 (무중단)

인덱스에는 수정이라는 개념이 별로 없다.  
수정 기능 자체가 아주 제한적으로 제공되기도 하고 권장되지도 않는다.  

그래서 인덱스의 구성이나 형태가 바뀔 일이 있다면, 인덱스를 수정하는 게 아니라 인덱스를 새로 만든 다음에 교체를 하는 방법을 사용한다.  

큰 문제 없이 교체를 하려면 alias를 미리 붙여둬야 한다.  
클라이언트들은 실제 인덱스명이 아니라 alias를 바라보게 하고, alias가 가리키는 것만 바꾸면 되기 때문이다.  
alias는 "POST /_aliases" API를 통해 제어할 수 있다.  
다음과 같이 해두면 products라는 인덱스에 release라는 별명을 붙여주게 된다.  

![](/images/img_d293c62cc55d079e.png)

그럼 이제 저걸 기반으로 교체작업을 할 수 있게 된다.  
교체할 인덱스를 새로 만들고  

![](/images/img_4ee8ec3f5f25e10a.png)

필요하다면 기존 인덱스의 데이터를 reindex로 옮겨준다.  
products의 검색 데이터를 products2에 반영시키고 싶다면 다음과 같이 하면 된다.  

![](/images/img_1beba9ca3ec53db4.png)
이건 부하가 굉장히 많이 걸리는 작업이니 상황봐서 잘 해야한다.  
데이터가 많으면 이거 하다 뻗을 수도 있다.  

문제없이 인덱스 준비가 다 됐다면, 다음과 같이 교체를 해줄 수 있다.  
그냥 별거 없다.  
아까처럼 별명을 붙여주되, 기존 별명을 삭제하는 것과 거의 동시에 처리해주는 것일 뿐이다.  

![](/images/img_ebb505dedc90525d.png)
이러면 이제 release가 교체된 products2를 가리키게 된다.  

기존의 product 인덱스는 지우든 뭘 하든 맘대로 해도 된다.  

아 그리고 alias 목록은 /_cat/aliases로 볼 수 있다.  

![](/images/img_351f3687f6c4fbd3.png)
release가 product2를 가리키는 것을 볼 수 있다.  

<br>

이외에 인덱스를 종료시키는건  
"POST /인덱스명/_close"  

인덱스를 다시 실행시키는건  
"POST /인덱스명/_settings"  

인덱스를 삭제하는건  
"DELETE /인덱스명"  

등으로 처리할 수 있다.  
삭제는 한번 하면 돌이킬 수 없으니 신중하게 하길 바란다.  

다음 포스트에서는 인덱스에 데이터를 넣는 것과 검색법 등을 다뤄보겠다.
