git으로 작업을 하다보면, 이미 저장한 커밋들의 내용을 변경하거나, 쓸데없이 자잘하게 나뉜 커밋을 합치고 싶은 경우가 있다.  

그럴 때 유용한 기능 중 하나가 rebase다.  

<br>

<br>

## 커밋 합치기
지금 내 경우에는 배포가 자꾸 실패해서 커밋을 좀 많이 했었는데, 그것 때문에 쓸데없는 커밋이 잔뜩 쌓인 상태다.  

![](/images/img_670f58ce4063df1e.png)
어떻게 봐도 좋은 상태라고 하긴 힘들다.  

저기서 merge 이후부터 최근까지의 9개를 뭉쳐보겠다.  

git rebase -i HEAD~조회할 개수  
의 형태로 명령을 주면 된다.  

난 로컬에서 커밋을 추가로 한번 더 해서 10으로 해놨다.  

![](/images/img_0b1ba402c39c3ebd.png)

그냥 다 보고싶으면 ~~로 줘도 된다.  

![](/images/img_a7f70ceb0a6e22b3.png)
보기만 하고 무시해도 되니까...  

그럼 대충 이런 식으로 편집창이 뜰 것이다.  

![](/images/img_7fc5fc4d5e5a14f4.png)

저기 펼쳐보면  

![](/images/img_c6a616cbe65f45a4.png)
pick, rewor, edit, squash, fixup, drop 등의 항목이 있다.  
pick은 그냥 그대로 두는거고,  
우리가 지금 쓸 건 squash다.  
이건 지금 커밋으로 이전 커밋을 찍어눌러버린다.  

그래서 쭉 내리고  

![](/images/img_b77b52cd97398f1f.png)
오케이 버튼 누르면  

편집창이 뜬다.  

![](/images/img_4b562c894654f084.png)
저장하고 닫으면  

<br>

![](/images/img_cc3bb1fa1f6c6fad.png)
반영이 될 것이다.  

이대로 강제 푸시를 하면  

![](/images/img_4290b2457ad9f4dd.png)

![](/images/img_d4ffde0b9ec4b8b2.png)
짜잔.  
히스토리가 깔끔하게 정리된 것을 볼 수 있다.  

fixup도 squash와 비슷한 기능인데, 합치는 건 동일하지만 커밋 메세지는 합치지 않는다는 것이 다르다.  

<br>

<br>

## 커밋 메세지 수정
이외에도 다양한걸 할 수 있다. 커밋 메세지를 잘못 적었더라도 rebase와 push -f를 사용하면 입맛대로 바꿔쓸 수가 있다.  

![](/images/img_d362ff661a43ea24.png)
rebase를 켠 다음에  

<br>

![](/images/img_e2788da64e4682eb.png)
바꿀 항목을 reword로 선택한 뒤 start rebase 버튼을 누른다.  

그럼 편집창이 뜰 것이다.  

![](/images/img_94257bd9ab54f4be.png)

잘 바꾸고  

![](/images/img_be33d7a811c4ef01.png)
나간 다음에  

![](/images/img_ae03dd01b587329d.png)

강제로 푸시하면 된다.  

![](/images/img_d28b5fbc9b5a22a3.png)

![](/images/img_b559817e775eecb9.png)
그럼 다시 잘 바뀌어있을 것이다.  

<br>

<br>

## 커밋 삭제
특정 커밋들을 삭제하려면 rebase의 drop 기능을 사용하면 된다.  

![](/images/img_62631e39dfe7a2c6.png)
난 저기서 '실수' 커밋을 삭제하고 싶다  

그러면 rebase 창으로 진입해서  

![](/images/img_6e25ed28b6238331.png)

![](/images/img_e4b3f93d75c5e753.png)
drop으로 깔아주면 된다.  

<br>

![](/images/img_f066e109f015ac96.png)
그리고 강제푸시하면  

<br>

![](/images/img_9957071449b5a3cb.png)
잘 삭제될 것이다.  

<br>

<br>

## 커밋 수정하기
특정 커밋의 메세지뿐 아니라 실제 작업내용들도 수정을 하고싶을 수도 있다.  
그럴때는 edit 기능을 사용한다.  

![](/images/img_5cd4ff1b310cf54a.png)
그리고 그냥 오케이 버튼 누르면  

![](/images/img_a94291fb37e5b41f.png)

현재 커밋이 edit할 커밋으로 바뀐다.  

지금 상태에서 파일들을 적당히 수정하고, amend 옵션과 함께 커밋을 한다.  

![](/images/img_ab3293940882420d.png)

![](/images/img_1c43cfa5d7c72f0b.png)
수정할 파일들 주석 해제하고  

rebase --coninue 를 준다.  

![](/images/img_7fa50c7d898a9e14.png)
rebase로 인해 발생한 conflict를 해결하는 명령이다.  

그리고 강제푸시하면 끝이다.  

![](/images/img_e01e7935fa978b9e.png)

그럼 커밋 메세지 등은 유지된 채로 작업내용만 바뀌어있을 것이다.  

![](/images/img_a5d9f3f551031e5d.png)

그렇다.
