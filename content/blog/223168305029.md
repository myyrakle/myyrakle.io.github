## VS Kafka

메세지 브로커라는 점에서 카프카와 사용영역이 겹친다고 볼 수 있다.  
하지만 사용목적과 형태가 조금은 다르다.  

카프카는 대규모 메세지 처리에 주안점을 두고, 대규모 병렬 클러스터링이 가능하다.  
반면 RabbitMQ는 그렇지 않다. Rabbit은 대규모 처리보단 지정된 consumer에게 신뢰성있게 메세지를 주는데 중점을 둔다. RabbitMQ는 스케일아웃을 통한 수평 확장도 거의 고려하지 않는 편이다.  

그래서 중소규모 단위의 메세지 처리는 RabbitMQ가 적절하고, 대규모 단위의 처리는 카프카가 적절하다.  

그리고 Kafka는 대용량의 메세지를 장기보존하는데도 능한 편인데, RabbitMQ는 그런건 좀 힘들다. 바로바로 소비되는 형태의 데이터 처리에 적합하다.  

RabbitMQ가 Kafka에 밀리기만 하는 것은 아니다. RabbitMQ는 라우팅 처리 관련해서는 Kafka보다 나은 기능을 갖고 있다. 예를 들어, consumer에서 원하는 형태의 필터를 지정해서 받고싶은 것만 받을 수 있다.  
카프카는 어렵다.

## 설치

우분투에서는 이렇게 설치할 수 있고  

```javascript
sudo apt-get install rabbitmq-server
```

![](/images/img_d87f949100abba6b.png)

![](/images/img_9c04ceda4084afcd.png)

docker가 있다면 이미지를 통해 간단하게 실행해볼 수 있다. 초기 패스워드를 설정해준다.  

```javascript
docker run -d -p 5672:5672 -p 15672:15672 -e RABBITMQ_DEFAULT_USER=tester -e RABBITMQ_DEFAULT_PASS=q1w2e3r4 rabbitmq:3-management
```
