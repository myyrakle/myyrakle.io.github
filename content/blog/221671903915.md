insert는 테이블에 데이터를 삽입하는 쿼리다.  
대략 아래의 구조를 가진다.  

insert into 테이블명(속성명1, 속성명2, ...) values(값1, 값2, ...);  

따라서 저기다 데이터를 하나 더 넣으려면, 아래와 같이 할 수 있다.  

![](/images/img_f2c4850782c03ca0.png)
잘 들어갔다고 한다.  

<br>

![](/images/img_4873f0c22bcf6f7c.png)
그럼 이렇게 잘 추가되어 있을 것이다.  

NULL이 가능한 속성이라면 값을 아예 안 줄 수도 있다.  
맨 위의 속성 목록을 보면 알겠지만, student 테이블은 가본키인 _index를 빼면 전부 NULL이 YES로 되어있다.  
따라서 아래와 같은 쿼리도 잘 동작한다.  

![](/images/img_384a1fad12a9fb38.png)

그럼 이렇게 빈 공간은 NULL이 들어간다.  

![](/images/img_c014b4c4d067abeb.png)
그런데 NULL은 온갖 오류와 결함의 본산이 될 수 있기 때문에, 웬만하면 다 NOT NULL 속성을 주고, 값을 꼬박꼬박 다 채워주는게 좋다.  
자세한건 create나 alter 절에서  따로 다루도록 하겠다.  

이제까진 위에서 모든 속성 목록의 이름을 열거했지만, 모든 속성에 값을 넣을 것이라면 이름을 생략할 수도 있다.  

![](/images/img_e978407c49082b60.png)

그럼 잘 들어가는 걸 볼 수 있다.  

![](/images/img_a92e3b1766490728.png)
그런데 이 방식은 속성의 순서도 다 맞춰줘야하고 나중에 보기도 귀찮기 때문에, 경험상 쓰기가 그리 좋진 않다.
