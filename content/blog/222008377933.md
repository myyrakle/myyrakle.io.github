당연하지만, HTTP 웹서버만으로 모든 통신 서비스를 구현해낼 수는 없다.  

보다 즉시성을 가져야 하는 양방향 통신을 사용해야 할 경우에는 TCP 통신을 사용해야 할 경우가 빈번한데, 노드에서 이를 구현하기 적합한 모듈로 socket.io라는 친구를 제공한다.  

한번 갖다써보자.  
먼저 서버부터 구축해보겠다. socket.io 모듈을 설치한다.  

![](/images/img_463945348ee15da0.png)

http와 express도 필요하다.  
socket.io는 인터페이스만 제공하는 거라서, 엔진이 또 있어야 하기 때문이다.  

![](/images/img_92b09f6d0e072f5e.png)

![](/images/img_9b705b8e1d092784.png)

깔건 다 깔았으니 코드를 짜보자.  
먼저 express로 서버객체를 생성하고 클라이언트가 될 html을 파일을 쏴줬다.  
동일 호스트 내에서 접근하는 것이 CORS 문제도 없고 단순하기 때문에 이렇게 했다.  

그리고 http로 express 서버를 감싼 후, socketio 함수의 인자로 넘겨주면 이제 socket.io 핸들링 객체가 나온다.  
저기다가 .on으로 이런저런 이벤트를 등록하고 서버를 시작하면 되는 것이다.  

![](/images/img_38422d9cb75e9fbd.png)

그럼 이제 클라이언트를 보자.  
사실 html5의 웹소켓을 사용해도 되는데, 그건 브라우저 호환성 문제가 있을 수 있어 보통 socket.io 클라이언트 모듈을 사용하는 편이다.  
나는 cdn으로 모듈을 가져와 일단 접속만 했다.  

![](/images/img_df0880d48ccc5cad.png)

만약 모듈을 받아서 쓰고 싶다면 이렇게 socket.io-client를 받으면 된다.  

![](/images/img_3e0f8bf30cdfa6dc.png)

일단 기본세팅은 된 것 같다.  
이제 연결이 되는지 테스트를 해보자  
서버를 실행하고  

![](/images/img_21f872289232e3c9.png)

클라이언트 페이지를 불러와서 눌러주면  

![](/images/img_c963a8828fd243a6.png)

![](/images/img_f093dc721969d0e9.png)
연결됐다고 한다.  

그럼 이제 메세지를 주고받아보자.  
받을 때는 .on 메서드로 주제를 분리해서 핸들링하는 방식을 사용한다.  
아래는 "text"라는 주제로 들어온 메세지에만 반응해 출력하도록 한 것이다.  

![](/images/img_876cf7f1965dd08e.png)

메세지를 전송할 때는  
emit 메서드를 사용하면 된다. 마찬가지로 첫번째 인자로 주제를 적고, 2번째부터 실제 데이터를 쏴주면 되는 것이다.  

![](/images/img_d4dabfb619c076c7.png)

그럼  

![](/images/img_64b2ab2547e3dd04.png)
잘 날라올 것이다.  
마찬가지로 서버->클라이언트도 동일한 방식으로 사용할 수 있으니, 잘 응용만 하면 되겠다.
