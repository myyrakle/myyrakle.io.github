module type 스크립트는 비교적 근래에 추가된 javascript 임베딩의 한 형태다.  

기존에는 레거시한 형태의 script 임베딩만이 가능했다.  
이런 식으로 말이다.  

![](/images/img_304be0b6f4f39220.png)
코드를 script 태그로 가져오거나, 바로 정의해서 사용할 수 있었다.  
근데 여기에는 문제가 많았다. 글로벌 범위에 다 쏟아부어버려서 캡슐화가 전혀 되지도 않고, import/export 형태로 정의된 모듈 코드를 사용할 수도 없었기 때문이다.  

새로 추가된 module script를 사용하면 그러한 문제들을 해결할 수 있다.  

<br>

<br>

<br>

## 모듈 사용

<br>

![](/images/img_9c266fea82cd093f.png)
먼저 import/export 형태로 정의된 코드를 구현한다.  
export는 내보내는 함수다.  

<br>

![](/images/img_e36021c99005df32.png)
그리고 그걸 type="module" 형태로 모듈을 정의한 후에, 동적 import를 사용해서 모듈을 가져와서 쓰는 것이다.  
그러면 저기서 받아온 변수로 모듈이 감싸인 채로 사용할 수 있다.  

<br>

![](/images/img_b107bb508affe694.png)

![](/images/img_57b9d476a3fb160d.png)
그리고 모듈 코드는 모던 자바스크립트에서 하듯이 import, export 사이클로 소스 형태를 구조화할 수 있다.  
훨씬 관리하기 편하다.  

물론 요즘은 다들 프레임워크를 써서 말아올리니 직접 이렇게 할 일이 별로 없긴 하다만... 알아는 두면 좋다.
