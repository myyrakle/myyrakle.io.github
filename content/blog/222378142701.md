## Indexed DB란?
브라우저에서 기본으로 제공하는 데이터베이스 기능이다.  
JSON 형태로 데이터를 저장하며, 트랜잭션을 지원하고, Key-Value 단위의 데이터 관리 시스템을 가진다.  

그리고 도메인마다 독립적으로 구성되기 때문에, A 도메인에서 만든 IndexedDB는 B 도메인에서는 절대 접근할 수 없고, 서로 영향을 줄 수 없다.  

아마 웬만하면 쓸일이 없을 것 같긴 한데, 프론트에서 데이터를 복잡하게 다룰 일이 있다면 한번쯤 고려해볼만 한 것 같다.  

브라우저 지원 상태도 나쁘지 않다.  
IE도 불완전하게나마 지원되고 있는 상태다.  

![](/images/img_deae0a2265316dfc.png)

## 데이터베이스 열기
데이터베이스를 여는 것은 다음과 같이 할 수 있다.  
전역 객체에 존재하는 indexedDB 객체에서 open 함수를 호출하는 것이다.  
인자로는 데이터베이스의 이름을 지정한다.  

```javascript
<script>
        const request = window.indexedDB.open('test');
        const db = null;

        request.onerror = (event) => {
            console.log('DB 열기 실패');
        };
        request.onsuccess = (event) => {
            console.log('DB 열기 성공');
            db = event.target.result;
        };
</script>
```
그리고 콜백으로 상황을 처리하기 때문에, onsuccess에서 db 값을 반환받아 할당해주면 된다.

## 스토어 생성
RDB에서의 테이블과 매칭되는 것이 바로 Object Store라는 것이다.  
스토어는 키값을 포함하는 JSON 객체의 목록을 가진다.  

그리고 스토어는 기본적으로 onupgradeneeded에서 정의해야 한다. 안그러면 오류를 내더라.  
createObjectStore 메서드에 첫번쨰 인자로 스토어의 이름을, 그리고 두번째 인자로 이런저런 옵션을 넣어줄 수 있다. keyPath는 기본키로 사용할 값의 이름이다.  

![](/images/img_2d11e2e1ddcd13e6.png)
그리고 onupgradeneeded는 데이터베이스 버전이 올라갈때만 호출되는 녀석이라,  

스토어 정의를 변경할때마다 버전을 올려줘야 제대로 반영이 될 것이다.  

![](/images/img_d70ac44c98f5dfdd.png)

만약 키가 자동으로 생성되게 하고 싶다면 autoIncrement 옵션값을 true로 주면 된다.  

![](/images/img_a1483cb45924ab68.png)

## 데이터의 삽입
그럼 밑준비는 됐으니 데이터를 한번 집어넣어보자.  
IndexedDB는 기존의 DB들과 다르게, 읽기나 쓰기를 할때 항상 명시적으로 트랜잭션을 꺼낸 다음 처리를 해줘야 한다.  

먼저 가져올 스토어와 읽기&쓰기모드를 지정해서 트랜잭션을 열어둔 후에, 거기서 objectStore로 당장 작업할 스토어를 가져온다.  

여기까지는 스토어로 어떤 작업을 하든간에 공통적인 부분이다.  

그러면 이제 add 함수로 스토어에 삽입을 할 수 있는 것이다.  
여기서 삽입할 값을 오브젝트로 넘겨주기만 하면 된다.
