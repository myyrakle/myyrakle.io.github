컬렉션은 Mivus의 기본 테이블 단위다.  

컬렉션은 기본적으로 고유 ID와, 사전에 정의된 여러 필드들로 구성될 수 있다.  

![](/images/img_f3f29b4a0eda24de.png)
벡터 값도 일반적인 필드 중 하나로 들어갈 수 있다.  
벡터를 포함한 필드들에 대한 인덱스는 명시적으로 지정해야 한다. 자동으로 인덱스가 걸리는 구조는 아니다.  

<br>

<br>

<br>

## 컬렉션 생성

생성은 전용 create API를 통해서 수행할 수 있다.  
반드시 데이터베이스도 함께 지정해야 하고, 벡터 차원도 지정해줘야 한다.  

```javascript
### 
POST http://{{HOST}}:{{PORT}}/v2/vectordb/collections/create
Content-Type: application/json

{
    "dbName": "test_db", 
    "collectionName": "test_collection",
    "dimension": 8
}
###
```

![](/images/img_eafbffc851cad658.png)
GRPC 등을 쓰면 벡터를 여러개 만든다거나, 벡터 타입을 지정한다거나 하는게 되는 것 같지만, HTTP는 기능에 제한이 있는 것 같다.

<br>

## 컬렉션 정보 조회

목록 조회는 list로 조회할 수 있다. 마찬가지로 db를 지정해야 한다.  

```javascript
### 
POST http://{{HOST}}:{{PORT}}/v2/vectordb/collections/list
Content-Type: application/json

{
    "dbName": "test_db"
}
###
```

![](/images/img_3d85e8dcb936c860.png)

상세 조회는 describe API를 쓰면 된다.  
db 이름와 컬렉션 이름을 모두 전달받는다.  

```javascript
### 
POST http://{{HOST}}:{{PORT}}/v2/vectordb/collections/describe
Content-Type: application/json

{
    "dbName": "test_db", 
    "collectionName": "test_collection"
}
###
```

![](/images/img_960c60a292a333dd.png)
그럼 이런 식으로 컬렉션에 대한 전체 정보를 뿌려준다.  

<br>

![](/images/img_a942b20211a8b236.png)
그럼 이런식으로 자동 생성된 필드도 들어있다.  
id가 정수 기본키고, vector가 벡터 필드다.
