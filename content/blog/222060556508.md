관계형 DB들은 데이터 구조가 조금만 바뀌더라도 엄청난 수고가 동반된다.  
데이터별로  테이블을 분리하고... 키값을 통해 연관을 시키기 때문이다.  

하지만 PostgreSQL에서 제공하는 기똥찬 JSON 기능들을 사용하면 이를 조금 완화할 수 있다.  

먼저, PostgreSQL은 JSON과 JSONB라는 JSON 전용 타입을 지원한다.  
이 둘은 약간의 차이를 제외하면 거의 기능이 같은데, JSONB이 인덱싱도 되고, 되는게 더 많은 편이다. 특별한 경우가 아니면 JSONB를 쓰는 것이 권장된다.  

JSONB 타입 값은 다음과 같이 표현할 수 있다.  
JSON 텍스트 문자열에 캐스팅을 가하는 것이다.  

![](/images/img_f6e0063e80ca3cf4.png)

<br>

<br>

## 값 획득
json 값 접근 연산자로는 ->와 ->>가 제공된다. 오른쪽 피연산자에는 인덱스나 키값 텍스트가 들어간다.  

![](/images/img_c077172909fb2560.png)

->는 jsonb이나 json을 반환하고, ->>는 문자열로 반환한다.  

![](/images/img_5ff7c74da7b9ea9e.png)
->>는 숫자가 들어있어도 문자열을 반환한다는 점에 유의한다.  

<br>

<br>

## JSON 병합
문자열과 마찬가지로, || 연산자를 사용하면 JSON 타입끼리 합칠 수 있다.  

![](/images/img_1cc8548d093288bf.png)

<br>

<br>

## 값 삭제
값 삭제는 매우 쉽다.  
그냥 키값 텍스트로 뺄셈을 하면 된다!  

![](/images/img_92af7259cdb42eb5.png)

객체 구조 깊숙히 들어있는 값의 경우, #- 연산자를 사용, 키값을 중첩해 삭제를 한다.  

![](/images/img_ca06acde95f6f187.png)

<br>

<br>

## 값 삽입
값을 삽입할 때는 jsonb_set이나 jsonb_insert 등의 전용 함수를 사용한다.  
set은 덮어쓰기까지 가능하고, insert는 새 삽입에만 유효하게 동작한다.  
둘다 순서대로 json 값, 검색패턴, 삽입값을 매개변수로 받는다.  

![](/images/img_954df1ae3632c8bb.png)

![](/images/img_94011cbbfcf85f13.png)

그렇다.  

이외에도 여러가지 함수들과 기능들이 많지만, 일단은 여기까지만 정리한다.
