기본적인 사용법 정도만 다뤄본다.  

duckdb는 sqlite와 많은 부분에서 닮았다.  
RDB들처럼 중앙 DB 서버를 띄워서 쓰는 구조가 아니라, 필요할 때만 실행해서 쓰는 구조를 가진다.  

그래서 그냥 CLI로 실행해서 가볍게 쓸 수도 있고, 프로그래밍 언어 수준에서 import해 사용할 수도 있다.  
여기서는 언어에 독립적이고 간단한 CLI 사용법 정도만 다룬다.  

<br>

<br>

<br>

## CLI 설치

[https://duckdb.org/docs/installation/?version=stable&environment=cli&platform=macos&download_method=direct](https://duckdb.org/docs/installation/?version=stable&environment=cli&platform=macos&download_method=direct)  

설치는 가이드 문서가 있어서 꽤 간편한 편이다.

```javascript
curl https://install.duckdb.org | sh
```

![](/images/img_a494603e66aa266a.png)
그래서 이렇게 실행이 되면, 설치는 잘 된 것이다.

## 데이터 output 기능

duckDB의 편리한 부분 중 하나는, 다른 데이터 포맷들과의 연계가 쉬운 편이라는 것이다.  
쿼리 실행시에 .으로 시작하는 명령어를 적용할 수 있는데, 이러면 출력결과를 마크다운 파일로 만들겠다는 뜻이 된다.  

```javascript
.mode markdown
.output series.md
SELECT *
FROM generate_series(5);
```

![](/images/img_62fc4b291af2ce4c.png)

![](/images/img_60379d722815de5c.png)

csv도 되고  

```javascript
.mode csv
.output series.csv
SELECT *
FROM generate_series(6);
```

![](/images/img_6d94ea99c136a859.png)

![](/images/img_3f08b5b753c91c5d.png)
일반적인 포맷들은 다 된다고 보면 된다.

## 데이터 import

duckDB는 외부 데이터 포맷을 바로 데이터처럼 사용할 수 있는 기능을 제공한다.  
예를 들어 이런 csv가 있다면  

![](/images/img_91f1f2eb0fa6c114.png)

바로 쿼리를 통해서 로드해서 쓸 수 있는 것이다.  

```javascript
SELECT *
FROM read_csv('series.csv',
    delim = ',',
    header = true,
    columns = {
        'generate_series': 'VARCHAR'
    });
```

![](/images/img_d5d7b7d5f18baf28.png)
