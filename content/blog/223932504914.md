Single executable applications, SEA는 Node.js에서 제공하는 실험적 기능이다.  
js 코드나 종속성, config를 지저분하게 주렁주렁 달고다닐 필요 없이 단일 바이너리로 압축해서 서빙할 수 있게 해주는 기능이라고 할 수 있다.  

<br>

![](/images/img_b228c506804b1354.jpg)
[https://dev.to/this-is-learning/building-single-executable-applications-with-nodejs-16k3](https://dev.to/this-is-learning/building-single-executable-applications-with-nodejs-16k3)  
이걸 진짜 바이너리로 컴파일해서 최적화를 해주는 것은 아니고, 컨테이너로 말듯이 실행 단위를 격리해주는 것에 불과하다.  
그러니까, node.exe 실행파일에다가 소스코드 등을 억지로 우겨넣는 것이다. 그리고 파일시스템도 가상으로 모사해서 넣어주는 것으로 경로로 인한 문제도 없도록 해준다.  

사실, 서버 환경에라면 이런걸 쓸 필요도 없고 그냥 컨테이너로 말면 된다.  
하지만 클라이언트 환경이라면 배포가 용이해지는 면은 확실히 있을 것이다.  

<br>

<br>

<br>

## 제약

아직 기능적인 제한이 좀 있다.  
모듈시스템은 CommonJS만 사용할 수 있다. 그래서 ESM을 쓴다면 변환 과정이 필요하다.  

그리고 세팅하는게 좀 불편한 편이다.  

<br>

<br>

<br>

## 사용해보기 (in Linux)

리눅스를 기준으로만 정리한다. 이건 플랫폼마다 처리 방식이 다 다르다.  

node.js 22부터 실험적으로 제공하는 기능이다. 그 이전버전에서는 사용할 수 없다.  
버전부터 확인해보자.  

![](/images/img_b59bef0e3f03d368.png)

![](/images/img_2ea808479a875976.png)
이렇게 소스코드가 있다고 하면  

<br>

```javascript
{ "main": "main.js", "output": "test.blob" }
```

![](/images/img_9207f998d571c9a1.png)
이렇게 빌드를 위한 json 설정파일이 있어야 한다.  

그리고 전용 플래그를 줘서 코드를 blob 파일로 변환한다.  

```javascript
node --experimental-sea-config compile.json
```

![](/images/img_a4071a5462440c77.png)
저렇게 생긴 blob 파일은 그 자체로는 쓸모가 없다.  
이걸 이제 node 실행파일에 우겨넣어야 한다.  

기본으로 쓰는 실행파일을 변조할 수는 없으니, 복사해서 복제본을 하나 만들어두고  

```javascript
cp $(which node) node-app
```

![](/images/img_5c2d611fd4aa88c0.png)

다음 명령을 사용해서 우겨넣는다.  

```javascript
npx postject node-app NODE_SEA_BLOB test.blob --sentinel-fuse NODE_SEA_FUSE_fce680ab2cc467b6e072b8b5df1996b2
```

![](/images/img_fdeeaf2a85df5c97.png)
그러고 나서 실행파일인 node-app를 그대로 실행하면 되는 것이다.
