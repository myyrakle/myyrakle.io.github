## Redis 서버 구성 (Docket)

먼저 docker를 통해서 레디스 서버를 N개 띄워보자.  

```javascript
sudo docker network create redis-network

sudo docker run --name master -d --network redis-network -p 44444:6379 redis
sudo docker run --name slave-1 -d --network redis-network -p 44443:6379 redis
sudo docker run --name slave-2 -d --network redis-network -p 44442:6379 redis
```

## Replication 구성

replication 구성은 의외로 간단하다.  
slave 노드들에 들어가서, replicaof 명령을 치면 된다.  

```javascript
replicaof master 6379
```
그럼 저 노예들이 master 서버를 가리키게 된다.  

<br>

![](/images/img_4d485a6f4389e4bc.png)

![](/images/img_01e14aacbe9806c9.png)
하나씩 등록하고 좀 기다리면  

정보가 바뀔 것이다.  

![](/images/img_7dc9c8cd13090169.png)
master는 connected_slave가 2개가 뜨고  

<br>

![](/images/img_8bd56820d5b85516.png)

![](/images/img_48880a92b7a11a73.png)
slave들은 master가 연결되어있음을 보여준다.  

그럼 이제  

![](/images/img_2b3523677cb53063.png)
master에 데이터를 찍으면  

<br>

![](/images/img_851bb93249b8429d.png)

![](/images/img_2f64f047390d50ac.png)
바로바로 slave에 복제될 것이다.  

<br>

![](/images/img_d98774255459a6ba.png)
단, slave 노드들에서는 쓰기 작업이 불가능하다. 일관성을 위해서 읽기 전용으로만 동작한다.  

이 정도가 기본으로 제공하는 Replication 구성이다.  

자동 장애 복구나 규모 확장성을 늘려야 한다면 Sentinel이나 Clustering을 직접 구성해야 하고, 신경써야할게 곱절로 많아진다.
