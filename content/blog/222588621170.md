## 데이터의 삽입

measurement은 미리 정의해서 만들거나 하는 것이 아니라, 삽입과 함께 자동으로 생성이 된다.  

삽입 쿼리는 다음과 같은 형태를 가진다.  

```javascript
INSERT ﻿measurement명,태그컬럼1=태그값1 컬럼1=값1, 컬럼2=값2, ...
```
태그는 일반 컬럼과는 좀 다른 성질의 것인데, 저건 차후 다루도록 하겠다.  

만약 시간에 따라 늘어나는 코로나 확진자수를 저장하고 싶다면,  
이런 식으로 데이터를 넣을 수 있다. 한국과 세계 값 2가지를 넣었다.  

![](/images/img_92b52f0aa98b3d98.png)

## 데이터의 조회

데이터를 읽어올 때는 select 명령어를 사용하는데, RDB들의 select문과 거의 유사하다.  

```javascript
select 컬럼1, 컬럼2, ... from measurement명 where 조건...
```

![](/images/img_ee29c04ac7ef6d07.png)
그런데 방금 넣은 데이터를 보면 time이라는 값이 갑자기 튀어나온 것을 볼 수 있다.  

인플럭스는 값의 기본 키로 항상 "시간" 값을 사용하고, 이 시간 값을 서버 시간 기준으로 해서 항상 자동으로 넣어주기 때문이다.  

그리고 where 절도 SQL과 유사한 형태로 사용할 수 있다.
