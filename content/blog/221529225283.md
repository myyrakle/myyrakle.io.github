변수 선언은 뭐 자바스크립트에서 하던대로 해도 되긴 한다.  
그런데 그러면 이걸 쓰는 이유가 없지 않은가?  

TS에서는 var을 권장하지 않는다.  
let과 const를 사용하는 것이 좋다.  

그리고 타입을 표시하려면 이런식으로 한다.  
**let num: ****number**** = 33;**  

위에서 number가 바로 타입이다.  
이렇게 적어놓으면 타입 검사를 수행해서 숫자가 아닌 값이 들어간다면 오류로 잡아준다.  

함수도 별 차이 없다.  
number를 하나 받아서 number를 반환하는 함수다.  
**function inc(num: ****number****): ****number**  
**{**  
**    return num+1;**  
**}**  

기본 타입들론 대강 이런게 있다.  

number ->숫자형 타입  
string ->문자열 타입  
boolean ->논리 타입  
T[] ->T의 배열 타입  
[T,T2] ->T와 T2로 된 튜플 타입  
any ->타입검사 안함  
T | T2 ->T와 T2의 공용체 타입  
void ->반환 안함(함수 반환타입)  

<br>

그리고 선언과 동시에 값을 초기화하되, 타입을 명시하지 않으면 타입을 추론해서 부여해준다고 한다.  
근데 이러면 자스 완전 호환이 못되지 않나...? 에러날 수도 있을텐데.  
모르겠다.
