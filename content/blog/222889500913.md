카프카는 대용량 처리를 위한 고성능 큐 시스템이다.  
정확히는 메세지 브로커라고 표현한다.  

링크드인에서 개발되었으며, 2011년에 오픈소스화되어 이후에는 아파치 재단에서 관리중이다.  
스칼라로 작성되어 자바로 구동된다.  

여러가지 용도로 많이 사용된다.  

<br>

<br>

<br>

## 설치

카프카 브로커는 리눅스와 맥에서만 제대로 지원이 된다.  
윈도우즈에서 카프카를 돌려보고 싶다면, 도커를 올려서 사용하는 것을 권장한다.  

![](/images/img_5795d62fd0bbcb6a.png)
포트만 잘 열어주자  

우선 Java 11을 깔아야 한다.  

```javascript
apt-get install openjdk-11-jdk
```

![](/images/img_11db6ccf25ad4ece.png)

![](/images/img_da1823926b1fef61.png)

그리고 카프카를 다운받는다.  
버전은 바꿔도 무방하다.  

```javascript
wget https://archive.apache.org/dist/kafka/3.0.0/kafka_2.13-3.0.0.tgz
```

![](/images/img_81d15286bf8b39e8.png)

압축풀고, 그 안으로 이동한다.  

```javascript
tar xzf kafka_2.13-3.0.0.tgz
mv kafka_2.13-3.0.0
```

![](/images/img_a57c50e69223948a.png)

<br>

zookeeper라는 녀석을 먼저 실행시켜줘야 한다.  
이건 kafka의 클러스터 관리용 의존성이다.  
다음과 같이 실행시키면 되는데, 포그라운드로 돌리고 싶다면 -daemon을 빼면 된다.  

```javascript
./bin/zookeeper-server-start.sh -daemon ./config/zookeeper.properties
```

![](/images/img_4c5dca1fda01d74b.png)

그러면 이제 카프카를 실행시켜주면 된다.  

```javascript
./bin/kafka-server-start.sh ./config/server.properties
```

![](/images/img_09e04025523e5b10.png)

![](/images/img_f9e791f5cf7e560e.png)
이렇게 로그가 잔뜩 쏟아지면 성공이다.
