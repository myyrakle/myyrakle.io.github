도메인은 타입에 대한 보일러플레이트라고 할 수 있다.  
타입에 디폴트값, 제약사항 등을 묶어서 또 하나의 타입을 정의하는 것이다.

도메인의 기본 생성법은 아래와 같다.

```javascript
CREATE DOMAIN 도메인명 AS 기반타입 ...;
```

테이블 컬럼을 정의할때처럼 DEFAULT, NOT NULL, CHECK 등을 줄줄이 이어붙일 수 있다.

난 이걸 이용해서 UTC값을 정수의 형태로 저장하는 도메인 타입을 정의했다.  
NULL이 아니고, 현재 시간값을 UTC 정수로 기본저장하며, 0 이상이도록 했다.

![](/images/img_177bf7704aac950b.png)

그럼 그냥 테이블 컬럼 타입 대신 저걸 넣어줄 수 있다.

![](/images/img_16f3f0b1fc63c18f.png)
타입이 UTC_TYPE으로 표시되긴 하는데, 실제로는 그냥 INT8 타입이라고 할 수 있겠다.

아예 고유한 타입을 생성하는 CREATE TYPE과 다르게 DOMAIN은 고유한 타입은 아니고, 그냥 문법적 편의일 뿐이기 때문이다.  
사용법에 별달리 특이한 점은 없다.

![](/images/img_6731007d52dcc240.png)
