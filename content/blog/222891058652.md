github packages는 다양한 패키지 시스템과의 호환 서비스를 제공하는 기능이다.  
기본적으로는 private package를 사용하기 위한 것이다.  

Docker, Java Maven, 닷넷 Maven, Ruby, nodejs의 npm 등을 제공한다.  

![](/images/img_af829e1dd3e2c80c.png)
본 포스트에서는 docker와 npm을 사용하는 방법만 간략히 다뤄보겠다.

## Docker 배포

먼저 기본 도커허브의 사용법 등은 알아두는게 좋겠다.  
모른다면 먼저 알아보길 권한다.  

참조  
[https://blog.naver.com/sssang97/221996513575](https://blog.naver.com/sssang97/221996513575)  

먼저 로그인을 한다.  

깃허브에서 제공하는 도메인은 ghcr.io라서, 이렇게 로그인을 시도하면 된다.  

```javascript
docker login ghcr.io -u 닉네임
```

![](/images/img_23b602a531ba907a.png)
패스워드는 아까 받은 토큰을 넣어준다.  

ghcr.io/유저명/이미지명의 형태로 태그를 붙여주고, 푸시하면 끝이다.  
나는 그냥 우분투 이미지를 그대로 올려봤다.  

![](/images/img_eb20972360c2e627.png)
그럼 이렇게 잘 올라갈 것이다.  

올라간 패키지는 github 패키지 탭에서 확인 가능하다.  
그리고 도커의 경우는 프라이빗으로만 올라가는듯 하다.  

![](/images/img_08cde929f9c69b6a.png)

![](/images/img_86a8f00e8e04382c.png)

## Npm 배포

npm 배포 경험이 없다면 알아두는 것이 좋다.  

참조  
[https://blog.naver.com/sssang97/222023436461](https://blog.naver.com/sssang97/222023436461)  

프로젝트 루트경로에 .npmrc 등으로 인증값을 생성한다.  

```javascript
//npm.pkg.github.com/:_authToken=토큰값
```

![](/images/img_397eef475e54e257.png)

그리고 package.json을 좀 건드려야 한다.  
이름에 github 유저네임을 달고  

![](/images/img_26e49368a38f2333.png)

![](/images/img_fba541def76bc680.png)

저장소를 github 저장소로 바꿔준다.  

```javascript
"publishConfig": {
    "registry": "https://npm.pkg.github.com"
}
```

![](/images/img_9f5f75d087b827c1.png)

그러고 publish하면 끝이다.  

![](/images/img_ae475ca7b25544b7.png)

![](/images/img_86f0da51bf773ebb.png)

그럼 이렇게 올라와있을 것이다.  

![](/images/img_6190b22e8ce2c8c4.png)
