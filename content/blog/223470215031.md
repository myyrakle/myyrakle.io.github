CouchDB에서 Document는 데이터베이스에 들어가는 각각의 데이터 행을 뜻한다.  

여기서는 Document를 CRUD하는 기본적인 방법들만을 정리해본다.  

<br>

<br>

<br>

## 전체 데이터 조회

_all_docs를 사용하면 모든 도큐먼트를 조회할 수 있다.  

```javascript
curl -X GET http://admin:q1w2e3r4@127.0.0.1:5984/users/_all_docs
```

![](/images/img_e65a775debf20e63.png)
아직은 없다.  

이러한 list api는 각 리스트의 id만 주는 형식이라서, 실제 데이터를 보려면 별도 API를 사용해야 한다. 그건 아래에서 다룬다.  

<br>

<br>

<br>

## 데이터 삽입

POST API를 사용해서 json 데이터를 body로 넘기면 삽입이 된다.  
이런 식이다.  

```javascript
curl -X POST -H "Content-Type: application/json" -d '{"name":"John Doe","age":30}' http://admin:q1w2e3r4@127.0.0.1:5984/users
curl -X POST -H "Content-Type: application/json" -d '{"name":"Tom","age":-444, "foo":"bar"}' http://admin:q1w2e3r4@127.0.0.1:5984/users
```

![](/images/img_6612e994e80257db.png)
그럼 추가된 것을 리스트로 확인할 수 있지만, 아까 설명했듯이 데이터가 보이지는 않는다.  

PUT API를 사용해서도 삽입을 할 수 있는데, 그건 id를 직접 할당해서 쓰고 싶을때 쓰는 방법이다. path에 원하는 id를 넣으면 된다.  

<br>

<br>

<br>

## 데이터 상세조회

실제 데이터를 조회하려면 각 데이터의 id를 가져다가 GET API를 쏴야 한다.  

```javascript
curl -X GET http://admin:q1w2e3r4@127.0.0.1:5984/users/아이디
```

![](/images/img_921f02895bb621f1.png)

![](/images/img_d379637cc8b96647.png)
그럼 이렇게 잘 가져올 것이다.  

<br>

<br>

<br>

## 데이터의 수정

수정은 PUT API를 통해 처리할 수 있다. 여기부터 좀 번잡스러워진다.  

![](/images/img_99bb1ca1e497993d.png)

수정할때는, 값 자체의 id와 더불어서 가장 최근 수정 내역인 _rev 정보까지 줘야 한다. 낙관적 락으로 동시 제어를 하기 위한 수단이다.  

```javascript
curl -X PUT -H "Content-Type: application/json" -d '{"_rev":"1-3341f3f62d6207dd706f52302a0df182","name":"Sohn Doe","age":30}' http://admin:q1w2e3r4@127.0.0.1:5984/users/17b9028ed468102e8d3593c9b9000a3d
```
이런 식으로 수정할 값 json 안에 rev를 넣어주면 된다.  

<br>

![](/images/img_a8ef693e647a6862.png)
그러면 이렇게 값이 수정되고, 기존 rev도 갱신될 것이다. 1-이나 2- 같은 앞자리 숫자는 수정된 횟수를 뜻한다.  

<br>

<br>

<br>

## 데이터의 삭제

데이터의 삭제는 DELETE API를 통해 처리할 수 있다.  
이것도 수정과 마찬가지로 rev를 넣어줘야 한다.  

```javascript
curl -X DELETE "http://admin:q1w2e3r4@127.0.0.1:5984/users/17b9028ed468102e8d3593c9b9000a3d?rev=2-6425a2fc401007d2c5109c5bffbb78e7"
```

![](/images/img_e9dc04745590df99.png)
그럼 삭제가 잘 될 것이다.
