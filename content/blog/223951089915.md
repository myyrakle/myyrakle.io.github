## 다운타임 없는 alias 교체

alias는 서버가 가리키는 인덱스를 교체할 때도 용이하게 사용할 수 있다.  

elasticsearch를 쓰다보면 하위호환이 보장되는 상태로 인덱스를 새로 만들때가 많다. 컬럼을 추가한다거나 해도 인덱스를 새로 따야 하기 때문이다.  
그러면 매번 서버가 가리키는 인덱스를 바꿔서 배포를 해야하는데, alias를 쓰면 이걸 간단하게 해결할 수 있다.  

alias 처리 API가 원자성을 보장하기 때문이다.  
예를 들어, market_item가 가리키는 실제 인덱스를 market_item_v5.3.0에서 market_item_v6.0.0로 바꾸고 싶다면, 이렇게 할 수 있다.  

```javascript
{
  "actions": [
    {
      "remove": {
        "index": "market_item_v5.3.0",
        "alias": "market_item"
      }
    },
    {
      "add": {
        "index": "market_item_v6.0.0",
        "alias": "market_item"
      }
    }
  ]
}
```
그러면 다운타임 없이 순식간에 alias가 가리키는 인덱스를 교체할 수 있다.
