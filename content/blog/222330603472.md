서버 로직을 짜든, 기타 어떤 함수를 짜든간에 입력값에 대한 검증은 항상 빼놓을 수 없는 부분이다.  

만약 이름과 나이값을 받는 foo라는 함수가 있다고 치자.  
그렇다면 foo 함수는 들어온 이름값과 나이값이 유효한지를 기본적으로 검증해야 할 것이다.  
이름은 당연히 문자열이어야 하고, 나이는 당연히 숫자여야 한다.  

그런 로직 체크를 손으로 하면 대충 아래와 같을 것이다.  

![](/images/img_9d7ae4081afc910f.png)

일단 당연히 동작은 하지만, 깔끔한 방법은 아니다.  

![](/images/img_4c86f551779940e4.png)

![](/images/img_6cc4eab9dd6bfd6b.png)
값이 많으면 많을수록 코드가 조잡해질 것이고, 좀더 세심한 검증이 필요할수록 코드의 복잡도는 하늘을 뚫을 것이다.  

이런 검증과정을 좀 쉽게 해주는 모듈 중 하나가 joi라는 친구다. javascript object scehma의 축약으로,  
다른 것도 있지만 이게 좀 인기가 많은 편이다.  

설치하고,  

![](/images/img_2fef9c4ff9e2a02c.png)

다음과 같이 검증용 스키마 값을 생성한다.  

![](/images/img_f90111d035e18a45.png)
검증할 값이 위의 형태를 갖춰야만 함을 명시하는 부분이다.  
name과 age라는 키값을 갖는 객체이며.  
name은 문자열이며 필수항목이어야 하고,  
age는 정수이며 최소값이 0이어야 함을 지정했다.  

그럼 이제 저기에다가 검증할 값을 집어넣기만 하면 된다.  
validate 함수는 값을 받아서 value와 error쌍을 가지는 객체를 반환한다.  
검증에 실패했다면 error에 값이 들어있을 것이고, 성공했다면 value에 값이 들어올 것이다.
