PostgreSQL도 다른 RDB들과 마찬가지로 함수, 다른 말로 프로시저라 하는 재사용 단위를 만들고 갖다쓰는 것이 가능하다.  

그리고 함수 구문 내에서는 일반 프로그래밍 언어의 형식으로 코드를 작성할 수가 있다. 반복문이나 변수나 그런거 말이다. 이런걸 PL/SQL이라 부른다.  

함수의 장점으로는  
1. 편리한 재사용과 일관적인 관리  
2. 쿼리 해석 시간(계획시간) 절약  
3. 자유로운 형태의 코드 작성 가능.  
등이 있고.  

단점은 많아지면 많아질수록 관리비용이 아주 커진다는 점이 있다.  
그냥 쿼리는 쿼리만 보고 이해할 수 있지만, 함수가 섞인 쿼리는 그 함수들까지도 다 타고 들어가면서 이해를 해야 한다. 함수가 수십 수백개가 넘는 경우엔 개판이 벌어진다.  
그래서 함수, 프로시저의 사용은 약간 지양되는 추세다.  

그래도 적당히 잘 쓰면 좋다.

## 함수의 생성
생성은 create function 키워드를 사용해 수행할 수 있다.  
그 뒤에는 이름과 (매개변수), 리턴타입.. 등등이 들어간다.  

만약 숫자 2개를 더하는 함수를 만든다면 이렇게 만들 수 있다.  

![](/images/img_9e8fa44c40eca986.png)

```javascript
CREATE FUNCTION MY_ADD(lhs int8, rhs int8) 
	RETURNS int8
AS 
$$ 
BEGIN 
	RETURN lhs + rhs;
END; 
$$
LANGUAGE PLPGSQL;
```
위의 두 줄은. int8 타입의 매개변수 lhs, rhs를 받고. int8 타입을 반환한다는 뜻이고  
begin과 end 사이가 실질적인 실행구문이다.  
return으로 값을 더해서 반환한다.  

그럼 기본함수들처럼 내가 만든 함수를 호출해서 사용할 수 있다.
