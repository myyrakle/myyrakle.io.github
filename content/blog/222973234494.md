tag는 커밋에 태그를 붙여주는 기능이다.  

보통 커밋에 버전을 붙여주는 것에 주로 사용한다.  
태그를 만들고 싶다면, 태그를 붙이고자 하는 커밋 시점에서 다음과 같이 버전명을 쳐주면 된다.  

```javascript
git tag 버전명
```

![](/images/img_864a8939b8aec3ff.png)
그럼 git tag 명령을 통해 로컬에 존재하는 태그 목록을 확인할 수 있다.  

태그를 올리려면, 브랜치 대신 태그를 넣어서 push를 하면 된다.  

![](/images/img_abe4af11977105ea.png)
remote history에서 커밋과 태그가 조화를 이루게 하려면 태그와 커밋을 전부 다 push해야 한다.  

아무튼 태그가 올라가면 github 등에서도 노출이 된다.  

![](/images/img_77f9fa3eeba7659f.png)
저걸로 들어가 볼 수도 있고  

태그가 찍힌 커밋에도 태그 표시가 뜬다.  

![](/images/img_3b472f0069e29dcd.png)

그럼 이런식으로 애셋 방출도 된다.  

![](/images/img_519e7b4e58b66701.png)

막 엄청난 기능이 있는건 아니지만, 가끔 유용하게 쓰이는 요소다.  

특히 golang의 경우에는 git 프로토콜 기반으로 모듈 관리를 하기 떄문에, 이 기능을 적극 이용하는 편이다.
