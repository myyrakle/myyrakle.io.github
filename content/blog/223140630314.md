Google Custom Search는 Google에서 일부 검색결과 집합을 따로 모아서 사용할 수 있게 해주는 도구다.  

유의할 점은 직접 커스텀해주는게 아니라, 특정 경로 내의 검색결과만 보여준다는 것이다.  
만약 배포된 내 사이트가 있고 그게 구글의 검색 데이터에 들어있다면, 내 사이트의 링크만 집어넣어서 우리가 만든 검색 시스템인 것처럼 보여줄 수 있다.  

그래서 간단하게 검색기능을 만들때는 좋다.  
다만 내가 직접 원하는 우선순위를 지정하거나 고수준의 커스텀을 하는데는 한계가 있다.

## API로 조회

UI를 그대로 쓰는게 아니라 직접 구현하고 싶거나, 그 데이터를 추가 가공하고 싶다면 다음 API를 사용해야 한다.  

```javascript
https://customsearch.googleapis.com/customsearch/v1?q=검색어&cx=검색엔진고유번호&key=API키
```
단점은 무료 일일 사용량이 100번이라는 것이다. 비용을 내면 더 추가는 가능하다.  

고유번호는 엔진 만들때 주는 이거고  

![](/images/img_fb7e6fbe602239cb.png)

API 키는 여기서  
[https://developers.google.com/custom-search/v1/introduction?hl=ko](https://developers.google.com/custom-search/v1/introduction?hl=ko)  

![](/images/img_2f8e27aaded4a111.png)
키 가져오기로 생성해서 가져올 수 있다.  

그리고 호출하면, 잘 가져올 것이다.

## API 비용 결제

비용 결제는 GCP에서 관리한다.  
해당 계정으로 들어가서 Custom Search API를 활성화해주면 된다.  

![](/images/img_36464f6784493d4a.png)
비용은 쿼리 1000개당 5달러다.  
일반 API는 일일 10000건을 초과할 수는 없고, Restrict API는 제한이 없다.  

<br>

<br>

<br>

## Restrict API

호출량이 많다면 Restrict 버전을 사용해야 한다.  
이건 검색 타겟으로 지정하는 사이트가 10개 이하여야 한다는 제한만 빼면 그냥 API와 동일하다.  

사실 본인 서비스에 붙이는데면 아무리 도메인이 아무리 많아도 2-3개일텐데, 10개 조건이면 대부분 문제는 없을 것 같다.  

API 경로에 siterestrict만 붙이면 된다.  

![](/images/img_1a352201959b2369.png)
