Unit of Work(이하 UoW)는 서비스 레이어와 레포지터리 레이어 사이에 들어가는 또다른 레이어 패턴이다.

## 원자성 (Atomic)

UoW의 주요 목적과 용도는 I/O 작업에 대한 원자성에 있다.  

쉽게 말하면, RDB에서 단일 트랜잭션 단위에 여러가지 연산을 묶어서 사용할 때가 많을 것이다.  
특히 일부만 실패하더라도 그 수정사항 전체를 rollback해야 할 경우 말이다.  

UoW는 그런 트랜잭션 단위의 commit과 rollback을 처리하는 껍데기 객체를 만들어서 쓰는 것이다.  

그림으로 표현하면 이렇다.

## 사용례 (Node.js + Sequelize)

Nest.js + Sequelize에서 UoW 패턴을 구현해서 적용하는 예시를 보여보겠다.  
UoW를 사용하지 않는다면 다음과 같이 레이어를 구성할 수 있을 것이다.  

레포지토리가 있고, 서비스가 바로 레포지토리를 갖다쓴다.  

![](/images/img_d856e407bd491a01.png)
하지만 이 코드에는 원자성에 대한 반영이 되어있지 않다.  
저 서비스에서 하나라도 삐끗한다면 어쩌겠는가?  

저기서 그냥 트랜잭션을 추가한다면, 이런식으로 해볼 수 있을 것이다.  
레포지토리 메서드에서는 명시적으로 트랜잭션을 전달받고,  

![](/images/img_ee777f48d9da34b6.png)

서비스에서는 명시적으로 트랜잭션을 생성해서 주입한다.  

![](/images/img_936ab750252153e3.png)
이 방법은 그럭저럭 쓸만하지만, 그리 세련된 모양새는 아니다.  
트랜잭션 자체는 사실 데이터와 I/O에 직접적으로 연관된 기능인데, 그걸 서비스에서 하고 있으니 말이다.  

보다 바람직한 형태는 UoW 레이어를 추가해서 트랜잭션에 대한 처리를 추상화하고 분리하는 것이다.  
이런 식으로 만들 수 있다.  

![](/images/img_3852df3884ff82df.png)
커밋과 롤백에 대한 처리를 수행하고, 레포지토리를 트랜잭션 기반으로 생성해주는 유틸성 객체가 됐다.  

그래서 레포지토리는 트랜잭션 객체를 전달받아야 하고  

![](/images/img_45afa0aad17b58e8.png)

서비스는 이제 UoW를 통해 레포지토리에 액세스하게 된다.  

![](/images/img_239f860762766735.png)
조금 더 간결해질뿐만 아니라, 단위테스트를 구성할 때에도 UnitOfWork의 Fake Object 버전을 구현해서 교체하면 된다는 장점이 있다.  

아래는 UoW에 대한 부분코드다.  

```javascript
...
export interface IUnitOfWork {
  start(): void;
  complete(work: () => void): Promise<void>;
  getRepository<T>(R: new (transactionManager: any) => T): T;
}

@Injectable()
export class UnitOfWork implements IUnitOfWork {
  constructor(
    @Inject('SEQUELIZE')
    private sequelize: Sequelize,
  ) {}

  private transaction: Transaction | undefined;

  async setTransactionManager() {
    this.transaction = await this.sequelize.transaction();
  }

  async start() {
    await this.setTransactionManager();
  }

  getRepository<T>(R: new (transaction: Transaction) => T): T {
    if (!this.transaction) {
      throw new Error('Unit of work is not started. Call the start() method');
    }
    return new R(this.transaction);
  }

  async complete(work: () => void) {
    try {
      await work();
      await this.transaction?.commit();
    } catch (error) {
      await this.transaction?.rollback();
      throw error;
    }
  }
}
...
```
